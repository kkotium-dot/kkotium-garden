# PRODUCT_LIFECYCLE_FLOW — 상품 생애 흐름 (앱의 뼈대)

> 운영자 구술 확정 2026-07-23. **이 흐름을 모르면 어떤 화면 설계도 틀린다.**
> 설계·기획·구현 착수 전 필독. 상태 판정은 `docs/design/LIFECYCLE_BRIDGE_V2_2026-07-23.md`.

## 1. 한 장 요약 — 상품은 이 길을 지난다

```
[꿀통 꽃나들이]          [정원 창고]              [온실 아틀리에]         [꽃밭 돌보기]
 크롤링·소싱 발굴   →   발행 전 대기 보관   →   소싱작업 + 검수   →   발행 후 운영·관리
 (도매매/도매꾹)         (아직 스토어에 없음)      (씨앗심기·꽃단장)      (스토어에서 판매 중)
                                                       ↓
                                              운영자가 발행 결정
                                                       ↓
                                        스토어 업로드 + 꽃밭 돌보기에 자동 연동
```

## 2. 메뉴별 역할 (정확한 정의)

### 꿀통 창고 (미발행 영역)
| 하위 | 역할 |
|---|---|
| **꿀통 꽃나들이** | 크롤링으로 상품을 발굴하고 **소싱 작업**을 하는 곳 |
| **정원 창고** | **발행 전 상품이 모여 있는 보관소.** ① 크롤링했지만 아직 발행 안 한 상품 ② 운영자가 직접 찾아 올려두고 아직 발행 못 한 상품 |

> 정원 창고 = "스토어에 등록하기 전까지 보관되는 곳". 여기 있는 상품은 **아직 스토어에 존재하지 않는다.**

### 온실 아틀리에 (작업 영역)
| 하위 | 역할 |
|---|---|
| **씨앗심기** | 기본 상품정보 소싱 · 네이버 엑셀 정보 연동 · **발행** |
| **꽃단장 작업실** | **상세페이지 · 썸네일** 제작 |

### 꽃밭 돌보기 (발행 후 영역)
- **스토어에서 실제 판매 중인 상품** 관리.
- 대상: 리셋·튜닝이 필요하거나, 수정해야 하거나, 지켜봐야 할 상품.
- 앱에서 발행한 상품은 **자동 연동**되어 함께 관리된다.
- 기존 스토어 상품 전량 연동은 부담이 크므로, **필요한 것만 선택 연동**한다.

## 3. 발행 게이트 — 운영자 개입점 (#307)

```
정원 창고 → 온실 아틀리에에서 소싱작업 완료 → ★검수 완료 → ★운영자가 발행
   → 스토어 업로드 + 꽃밭 돌보기 자동 연동/동기화
```

- **준비도 %·발행 N/8은 "빠진 항목 없음" 체크섬일 뿐 발행 승인이 아니다.**
- 검수 화면이 이미 존재: `/products/[id]/preview` = **"발행 전 검수"**
  (대표이미지 1000px·단색배경·텍스트없음·단품구도 / 상세이미지 / 네이버 페이로드 / 준비도)
- **미발행 → 발행 = 사람 검수 필수.** 시스템이 발행을 재촉하는 UI를 만들지 않는다.
- **예외**: 이미 발행된 좀비 상품의 "일부 수정 + 튜닝 리셋 → 재활성화"는 자동화 가능.

## 4. 화면 성격 2종 (#301)

| 성격 | 화면 | 기준 |
|---|---|---|
| **보관함(위치 축)** | 정원 창고(미발행) / 꽃밭 돌보기(발행) | 발행 여부로 분리. 상품은 정확히 한 곳에 |
| **작업 큐(조건 축)** | 처분 결정 대기함 · 좀비 부활소 · 대시보드 할일 | 조건 충족 상품만 수집. **발행 전용** |

> 큐에서 항목을 빼면 **갈 곳을 반드시 만든다**(#56). 예: 부활소에서 뺀 미발행 상품 → 정원 창고 "검수 대기" 큐.

## 5. 이 흐름을 어길 때 생기는 대표적 오류 (실사고)

| 사고 | 원인 | 교훈 |
|---|---|---|
| 부활소에 미발행 2건 노출 | 부활소를 "발행 전용 큐"로 안 봄 | 발행 여부가 1급 축 |
| "발행 가능 → 발행하러 가기" 배지 제안 | 준비도를 발행 승인으로 오독 | 검수는 사람 몫 |
| 기간판매(saleStartAt/EndAt) 설계 | "화훼 이커머스"로 오인 | 도매매 드롭십 셀렉트샵 |
| 재입고(RESTOCK) 액션 설계 | 셀러가 재고를 보유한다고 가정 | 드롭십이라 재고 없음 → 대체소싱뿐 |
