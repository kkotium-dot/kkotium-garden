# PROXY 인시던트 런북 — Funnel 503 (serve 멈춤) (2026-07-08)

Authoring: DESKTOP (직접 진단·해결). 승인 원칙: 운영자가 "직접 해결 가능한 건 모든 커넥터·스킬로 직접 해결" 승인(2026-07-08·#205). 권위: 본 런북 = 프록시 장애 대응 표준.

================================================================
## 0. 인시던트 요약
================================================================
- 증상: 앱의 네이버 연동 전부 실패(주문·상품 동기화). 앱 에러 = `fetch failed (ECONNRESET)`.
- 초기 오진 가능성: "시크릿 불일치"(프록시 err log의 `secret mismatch`는 **공개 Funnel URL을 긁는 봇 노이즈**였음 — 실제 앱 요청은 프록시에 닿지도 못함).
- **진짜 근본원인**: **Tailscale Funnel의 serve 프록시 컴포넌트가 멈춰 503 반환** — Funnel은 인터넷에 열려있으나 로컬 프록시(127.0.0.1:3001)로 포워딩 실패. (컴퓨터/프록시 프로세스/시크릿 전부 정상이었음.)

================================================================
## 1. 진단 절차 (계층별·홈 Mac iterm)
================================================================
1. **Mac 깨어있나**: iterm 접속되면 OK(절전 아님).
2. **Tailscale**: `Tailscale status` -> Running/Online.
3. **Funnel 설정**: `Tailscale funnel status` -> "Funnel on" + 타깃 `http://127.0.0.1:3001`.
4. **로컬 프록시 생존**: `curl -m5 http://localhost:3001/` -> **401**이면 프록시 정상(인증 필요).
5. **외부 도달성(핵심 분기)**: 외부에서 `curl -m10 https://macbook-pro.tail36c35f.ts.net/`:
   - **401** = 정상(프록시까지 도달).
   - **503** = Funnel serve 멈춤(본 인시던트) -> section 2 해결.
   - **연결 실패/타임아웃** = Funnel/네트워크 다운.
6. (앱이 프록시에 닿는지) 프록시 로그 `wc -l /tmp/kkotium-home-proxy.err` 증가 여부 + `tail /tmp/kkotium-home-proxy.log` 200 여부.

================================================================
## 2. 해결 (503 = serve 멈춤)
================================================================
```
TS=/Applications/Tailscale.app/Contents/MacOS/Tailscale
$TS serve reset            # 멈춘 serve/funnel 설정 클리어
$TS funnel --bg 3001       # 재수립 -> "Available on the internet"
```
- 검증: 외부 `curl https://macbook-pro.tail36c35f.ts.net/` -> **401**로 복구 확인 -> 앱 호출 200 확인.
- 본 인시던트는 이 2줄로 즉시 복구됨(재부팅/프록시 재설치 불필요).

**보조 조치(필요 시)**:
- 프록시 프로세스 재시작: `launchctl kickstart -k gui/$(id -u)/com.kkotium.home-proxy`
- 시크릿 확인(별건): 프록시 기대값 = `.env.local` CRON_SECRET(현재 kkotium2026cron). 앱 전송값 = Vercel `PROXY_SECRET ?? CRON_SECRET`. 불일치 시 양쪽 일치시킴.

================================================================
## 3. 예방 -> PROXY-HEALTH 반영 (#204 강화)
================================================================
- PROXY-HEALTH 헬스체크는 **응답 코드로 계층 구분**해야 함:
  - **503** -> "네이버 연동 끊김 — Funnel serve 멈춤(serve reset 필요)"
  - ECONNRESET/타임아웃 -> "프록시/Funnel 불통(홈 Mac 확인)"
  - 401 -> "시크릿 불일치"
  - Naver 4xx -> IP/인증/레이트
- 관제탑 경보에 **구체 조치**까지 표시(단순 "오류" 금지).

## 작업 유의사항/원칙 (신규)
================================================================
- **#205** 직접 해결 가능한 인프라/환경 문제는 **모든 커넥터·스킬로 Desktop이 직접 해결**(운영자 승인). 단 비가역/네이버 쓰기/발행은 여전히 GO 게이트(#46).
- **#206** 프록시 장애 진단은 **계층 분리**(Mac->Tailscale->Funnel->로컬프록시->외부도달성) — err log의 secret mismatch는 공개 URL 봇 노이즈일 수 있으니 "앱 요청이 실제로 프록시에 닿는지"(로그 증가)로 확정. Funnel 503 = serve reset.
- **#207** 외부 도달성은 반드시 외부(컨테이너 curl 등)에서 검증 — 로컬 curl 정상이 외부 정상을 보장하지 않음(#200 연장).
