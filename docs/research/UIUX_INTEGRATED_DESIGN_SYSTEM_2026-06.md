# 꽃틔움 가든 통합 UI/UX 디자인 시스템 재설계 연구 보고서

> 작성일: 2026-06-01 / 출처: Desktop 리서치 (네이버 SEO + 셀러 백오피스 UX + 디자인 시스템 베스트 프랙티스)
> 목적: 아틀리에/씨앗심기 포함 앱 전체 화면의 디자인 조화·레이아웃·워크플로우 정합성 통합 재설계
> 활용: P0~P4 단계별 개선 작업의 권위 참조 문서. Code 착수 블록 작성 시 이 문서 기준.

## TL;DR
- **단일 디자인 시스템(토큰 기반) + 좌측 사이드바·12컬럼 카드 그리드 셸을 모든 화면에 강제 적용하고, 워크플로우는 "독립 탭 + 상단 단계 진행 레일(stepper rail)" 하이브리드로 통일하라.** 화면을 한 줄 흐름으로 강제하는 풀 위저드보다, 각 탭은 독립적이되 상단에 "소싱→진단→이미지→씨앗심기→발행" 진행 상태 레일을 공통 배치하는 방식이 1인 셀러의 비선형 반복작업과 맞다.
- **아틀리에(스튜디오)와 씨앗심기가 최우선 재설계 대상이다.** 인라인 스타일을 Tailwind v4 @theme 토큰으로 이관하고, 4개 작업 카드를 "작업벤치(2~3분할: 좌측 목록/중앙 캔버스/우측 컨트롤)" 패턴으로, 씨앗심기를 "실시간 점수 게이지가 달린 SEO 편집기"로 재구성하면 사용자가 지적한 부조화·저퀄리티가 구조적으로 해소된다.
- **브랜드 레드 #E62310는 "행동·강조 전용 10% 액센트"로 격리하고, 핑크 계열을 15~30% 보조(카드/배경 표면), 화이트/뉴트럴을 60~75% 바탕으로 운용하라.** 따뜻한 식물 성장 컨셉은 카피·아이콘·빈 상태(empty state)·완료 축하 모먼트에 담고, 데이터·표·폼은 차분하고 고밀도로 유지해 감성과 효율을 분리·양립시킨다.

## Key Findings

1. **현대 SaaS 백오피스의 합의된 구조가 명확하다.** Stripe·Linear·Notion·Vercel은 동일 골격으로 수렴: 좌측 240–280px 고정 사이드바 + 상단 4–6개 KPI 카드 스트립 + 하단 12컬럼 CSS Grid 카드 영역. 5~50개 기능까지 재설계 없이 확장되며 F자 스캔 패턴 부합. 카드가 일관된 모서리 반경(8–12px)·내부 패딩(24px)을 가질 때 "시스템"으로 읽히고, 제각각이면 "프로토타입"으로 읽힌다 — 사용자가 지적한 "개별 부품은 동작하나 통합 디자인 시스템 부재"의 핵심.

2. **색은 상태·긴급도·위계 전달의 가장 빠른 수단이나 남용 시 신호가 죽는다.** "빨강은 망가졌을 때만"(Green=정상, Amber=경고, Red=치명적). 브랜드 레드는 강렬하므로 CTA·치명적 경고·핵심 강조에만 희소하게. 60-30-10 규칙(10%를 5%+5%로 분할: 주 행동/위험 행동)이 표준 골격.

3. **정보 위계의 "5초 규칙".** 사용자가 5초 안에 가장 중요한 정보를 식별 못하면 위계 설계 실패. 핵심지표 상단, 추세·맥락 중단, 상세 분해 하단. 점진적 공개(progressive disclosure). "5~9개 요소" 가이드는 Miller's Law(작업기억 7±2)에서 온 것이나, 정보가 화면에 시각 제시되면 한계가 약화되므로 절대 규칙이 아닌 참고선으로만 사용.

4. **한국 셀러 환경은 "통합관리" + "키워드 데이터 기반 의사결정" 양대 축.** 사방넷·플레이오토 등 통합솔루션은 기능 방대하나 초기 세팅·UI 진입장벽 높음. 아이템스카우트·셀러라이프·판다랭크 키워드 툴은 검색량·상품수·경쟁강도·전환율로 의사결정 지원. 꽃틔움 차별점 = 워크플로우 자동화 + 키워드 시각화를 1인 셀러용 단일 흐름으로 묶는 것.

5. **네이버 SEO 규칙은 정량화 가능 → UI 시각화 적합.** 표준상품명: 50자 내외 권장(최대 100자), 조사·수식어 미사용. 동일 단어 3~4회 이상 반복 시 SEO 로직 위반 → 노출 이상 + 상품명 품질 체크에 '동일 단어 반복' 경고. 랭킹 = 적합도·인기도·신뢰도 3요소. 신뢰도는 페널티 게이트형(상품명 SEO 어기면 감점). 경쟁강도 = 상품수÷검색수(낮을수록 경쟁 낮음, 1 초과 시 공급>수요) → 골든키워드(수요 多 공급 少) 발굴 핵심. 실시간 게이지·길이 바·점수 뱃지로 시각화 가능.

6. **크리에이티브 툴·AI 생성 워크플로우 패턴이 아틀리에 청사진.** Figma/Canva = "좌 레이어/요소 + 중앙 캔버스 + 우 속성" 3분할 표준. AI 이미지 생성 = 프롬프트 입력→큐→생성→결과 검토. 드래그앤드롭(idle/dragging/uploading/complete/error 5상태), 프롬프트 복사, 외부 링크 연동, 작업 큐 상태 표시가 핵심.

## 통합 디자인 시스템 권장안

### 색상 토큰 (75/15/10 → 백오피스)
- **바탕 60–75%**: 화이트 #FFFFFF + 뉴트럴 그레이. 본문·캔버스·테이블 배경.
- **보조 15–30%**: 핑크 #fff0f5(연한 표면), #FFE0EC(카드/섹션 구분), #FFCCEA(선택·호버 강조). 사이드바·카드·뱃지 배경.
- **액센트 10%**: 레드 #E62310. 주 CTA·발행 버튼·치명적 경고·활성 단계 표시에만 희소하게.
- **상태색**: 정상=그린/경고=앰버/오류=레드(브랜드 레드와 톤 구분). SEO 점수 게이지에 그린→앰버→레드 그라데이션.
- **토큰 3계층**: 원시(--pink-100: #fff0f5) → 의미(--color-surface-subtle) → 컴포넌트(--card-bg).

### 타이포그래피 (Pretendard)
본문 최소 14–16px, 줄간격 1.5~1.6. 비례 스케일 ratio 1.2~1.25.
- Display 28–32 Bold / H1 24 SemiBold / H2 20 SemiBold / H3(카드) 16 SemiBold / Body 14–15 Regular(lh 1.5) / Caption 12–13 Medium
- 굵기 Regular/Medium/SemiBold/Bold 4단계로 제한.

### 간격 스케일
4px 베이스 8px 리듬: 4·8·12·16·24·32·48·64. 카드 내부 패딩 24px 기본, 카드 간 gap 16–24px. "임의 숫자 금지".

### 컴포넌트 토큰
- **카드**: 반경 12px, 패딩 24px, 보더 1px 또는 약한 그림자 택1(혼용 금지).
- **버튼**: Primary(레드 채움/발행), Secondary(핑크 표면/아웃라인), Tertiary(텍스트). 높이 36/40/44px. 호버 스케일 변형 금지.
- **뱃지**: 워크플로우 단계 상태 + SEO 점수 등급. 핑크 표면+텍스트 기본, 상태는 기능색.
- **폼/입력**: 일관 높이·포커스 링(레드 가늘게). 실시간 검증 피드백 슬롯 내장.
- **아이콘**: Lucide React만, 이모지 금지. 16/20/24px 토큰화.

### 반응형 그리드
12컬럼, 컨테이너 최대폭 1280px(고밀도 화면 더 넓게 허용). KPI 4개=각 3컬럼, 중단 차트쌍=6+6/8+4, 테이블=12컬럼 풀폭. 컨테이너 쿼리로 카드 내부 적응.

### 인라인 스타일 → 토큰 이관 전략
Next.js 14 → Tailwind v4 @theme 디렉티브. 색·간격·타이포·반경을 @theme에 정의 → 유틸리티 클래스 + CSS 변수 동시 노출 → 인라인/유틸리티가 같은 토큰 공유. 순서: (1)토큰 정의 → (2)공통 셸 → (3)P0 화면 → (4)나머지. 화면 단위 점진 이관.

## 핵심 워크플로우 화면 재설계

### 아틀리에 "작업벤치(Workbench)" 3분할
- **좌측 (280–320px, 축소 가능)**: 상품 목록 + 선택 상품. 항목별 이미지 작업 진행 뱃지(썸네일 완료/배경/누끼).
- **중앙 캔버스 (최대 영역)**: 작업 중 이미지 큰 미리보기. AI 생성·누끼·썸네일 시안 표시. 드래그앤드롭 5상태. before/after 비교.
- **우측 속성/액션 (320px)**: 4작업(진단/썸네일/상세/액션)을 세로 스택 아닌 탭/아코디언으로. AI 워크플로우 컨트롤: 프롬프트 표시+복사, "외부 생성 도구 열기" 링크, 결과 업로드 슬롯, 작업 큐 상태(대기/처리중/완료) stepper.

### 씨앗심기 "실시간 점수 편집기"
- **상품명 + 실시간 길이 게이지**: 50자 권장=그린, 50~100=앰버, 100 초과=레드. 글자수 카운터 상시.
- **중복 키워드 경고**: 동일 단어 3~4회 이상 인라인 경고 + 반복 단어 하이라이트.
- **검색량·경쟁강도 시각화**: 키워드별 검색량(막대), 경쟁강도(상품수÷검색수, 1 기준선) 바 차트. 파이차트 금지.
- **골든키워드 점수**: "검색량 高 + 경쟁강도 低"를 등급(최적/좋음/보통/나쁨) 뱃지. "낮은 경쟁강도 != 판매 보장" 주의 병기.
- **적합도·인기도·신뢰도 3축**: 신뢰도는 "통과/미통과" 게이트(가점 아닌 페널티 반영).
- **발행 준비도 게이트**: 상품명 길이/중복 없음/카테고리/태그/이미지 체크리스트+완성도 미터. 전 게이트 통과 시에만 "네이버 발행" 레드 CTA 활성. 미완 항목 클릭 시 해당 작업 이동. (Zeigarnik 효과 활용)

## 워크플로우 동선: 하이브리드 권장
- **풀 위저드 단점**: 1인 셀러 작업은 비선형 반복(일괄 소싱→일괄 진단→나중 이미지→나중 SEO). 강제 선형은 숙련자 속도 저하.
- **순수 독립 탭 단점**: 작업 연속성·진행 인지 약함("어디까지 했더라" 재확인).
- **권장 = 독립 탭 + 공통 진행 레일**: 각 화면 자유 이동 + 선택 상품에 상단 "단계 진행 레일"(소싱→진단→이미지→SEO→발행, 완료/진행/대기 상태) 상시 표시. 레일 클릭 이동 시 상품 컨텍스트 유지. 단계 3~6개 권장 범위 부합.
- **컨텍스트 비용 최소화**: 상품 선택 상태 전역 유지. 다음 추천 액션 상시 노출("다음: 썸네일 제작").

## 브랜드 컨셉 ↔ 효율 양립 (영역 분리)
- **감성**: 빈 상태·온보딩·완료 축하·마이크로카피·아이콘·로딩. 식물 성장 은유(씨앗심기/온실/아틀리에/가든) 집중. 예: 발행 완료 "씨앗이 꽃을 피웠어요", 성장 단계(새싹→파워셀러) 시각화.
- **효율**: 데이터 테이블·폼·KPI·점수 게이지는 차분·고밀도. 장식 색·애니메이션 배제.
- **균형**: 레드=행동 신호만, 핑크=표면 온기, 식물 은유=카피·상태. 데이터 영역엔 은유 미침범. 가이드에 명문화.

## 전략 변경 임계값
- 단계 레일 무시/혼란 → 레일 약화 또는 대시보드 "다음 할 일" 중심 전환.
- 캔버스 비동기 큐 지연 답답 → 스켈레톤 로딩(스피너 금지) + 낙관적 UI 강화.
- 핑크 30% 과함 → 핑크를 사이드바·강조 표면으로 좁히고 뉴트럴 비중 상향.

## Caveats (중요 — 작업 시 반드시 인지)
- **프로덕션 직접 검증 불가**: kkotium-garden.vercel.app 인증 게이트 뒤 → 진단은 화면 구조 설명 기반. 실제 코드/스크린샷 확인 후 우선순위 미세조정 필요.
- **네이버 알고리즘 불투명**: 적합도/인기도/신뢰도 가중치 비공개·수시 변경. "50자"/"중복 3~4회"는 공식 문구의 실무 해석값(확정 수치 아님). UI에 하드코딩 금지, 설정 가능하게.
- **경쟁강도 데이터 한계**: 상품수÷검색수는 포털 전체 검색 기반(쇼핑 의도와 괴리). 낮은 경쟁강도 != 판매 보장. 주의 문구 필수.
- **"5~9개" 규칙**: Miller's Law는 작업기억 한계 연구. 시각 제시 시 약화 → 절대 규칙 아닌 참고선.
- **트렌드성 수치**: 240–280px/24px/8–12px는 2026 관행이지 절대 규칙 아님. 브랜드 따뜻함 위해 반경 조정 가능.
