# 대시보드·앱 셸 재설계 리서치 — ADHD 집중·프리미엄 SaaS IA (2026-07-08)

Authoring: DESKTOP(심층 리서치). 권위: 대시보드/셸 리팩터 설계 근거. 방향: 통합(대시보드+셸)·셀러툴+범용SaaS 하이브리드·ADHD "하루의 시작". Claude Design 컨셉(Retro-Pop Garden) 조율 포함.

================================================================
## 0. 3대 핵심 결론
================================================================
1. **과밀은 미학 아닌 IA 문제** — 사이드바 16→기능그룹 5~6개, 대시보드 26카드→핵심 4~6개, 나머지는 점진적 공개(progressive disclosure). 시각 리팩터는 IA 재편 "이후". (순서 바꾸면 "예쁜 26개"가 됨.)
2. **"하루의 시작" = 단일 초점** — 최상단 "오늘 할 일" 단일 큐(주문·품절·CS) + primary action 1개. 그 옆 KPI 4카드(오늘매출·신규주문·정산예정·품절경보). 상단=행동·중앙=이해·하단=탐색.
3. **브랜드는 프레임에·기능은 콘텐츠에** — 정원테마·마스코트·분수대 유지하되 장식강도 3단계 토글(기본=차분·모션 최소·prefers-reduced-motion 존중). 은유 라벨은 기능 라벨 병기("화단·상품").

================================================================
## 1. ADHD·인지부하 설계 원칙 + 체크리스트
================================================================
- 단일 초점(한 화면 하나의 목표) · 점진적 공개(3단계 이하) · 작업 분절(청킹) · 명확한 시각 계층 · 모션/색 절제.
- **작업기억 근거**: 밀러 7±2는 작업기억이라 화면 표시 항목엔 문자적용 오용(Nielsen). 실 근거는 Cowan(2001) **4±1 청크** → KPI 4개 상한.
- **경보 4단계**: Critical(빨강·상시·소량) / Warning(주황·해제가능·2~3개 제한) / Info(파랑·자동소멸) / Success(초록·3~5초 소멸). 빨강 남용 금지.
- **체크리스트(적용 검증)**: ①첫화면 요소 7~8개 이하 ②"오늘 할 일" 최상단 ③primary action 화면당 1개 ④빨강은 실위급(품절·배송지연·클레임)만 ⑤모션 기본 최소+토글 ⑥상세는 클릭/확장으로 숨김 ⑦빈상태는 "다음 한 행동"만.

================================================================
## 2. "하루의 시작" 대시보드 우선순위 구조
================================================================
F패턴(좌상단 우선·Nielsen 2006) 전제.
- **상단(행동)**: 좌상단 "오늘 할 일 단일 카드"(주문·품절임박·미답변CS 우선순위 큐·"지금 N건"+primary 버튼 1개) / 우상단 KPI 4카드(오늘매출·신규주문·정산예정·품절경보 — 큰숫자+델타화살표+스파크라인·Stripe 패턴).
- **중앙(이해)**: 매출추이 라인차트(**분수대 위젯 여기 배치**=브랜드 존치) / 최근 주문·리뷰 피드(아이콘 대신 색점).
- **하단(탐색)**: 스토어 진단(랭킹하락·유입감소 상위 3개·스토어센터 패턴) / 상세리포트 링크(점진 공개).

================================================================
## 3. 프리미엄 SaaS 레이아웃·시선흐름 적용안
================================================================
- 골격: 좌 사이드바 240~280px(접힘 56~72px) + 상단 KPI 스트립 4~6 + 하단 12컬럼 그리드.
- 그리드: KPI 각 3컬럼 / 차트쌍 6+6 or 8+4 / 테이블 12컬럼.
- 카드: 모서리 8~12px·패딩 24px·1px 헤어라인 or 옅은 그림자·카드당 시각요소 1개.
- 간격: **8pt 그리드**(8·16·24·32·40·48) · 카드간격 16~24 · 섹션여백 크게.
- 여백/색 **60/30/10**: 중립 60·텍스트 30·강조 10%미만. **1 주강조색(브랜드)+시맨틱색(성공초록/경고주황/위험빨강)**.
- ★**브랜드 강조색은 초록 외로 분리**(정원테마 초록 ↔ "성공=초록" 충돌 방지 → 브랜드=꽃 핑크/코랄).
- 라벨 14~16px·짧게·설명은 툴팁. Linear 원칙: 네비는 recede·작업은 focus·구분선 남발 금지·브랜드색 소수 지점만.

## 4. 사이드바 16→5~6 그룹 IA 재편안
================================================================
업무(JTBD) 기준 5그룹(각 2~4 하위·2단계 이하):
1. **홈/오늘** — 대시보드(단일)
2. **상품(화단)** — 등록·목록·재고·품절관리
3. **주문·배송(수확·배송)** — 신규주문·발송·배송현황·클레임
4. **고객·리뷰(벌통)** — CS·리뷰·알림
5. **분석·정산(곳간)** — 매출분석·순위/키워드·정산
6. **설정** — (하단 분리·회색 축소) 스토어설정·계정·도움말
- 상단바=전역(검색·알림·프로필·장식토글) / 사이드바=1차네비(폭15~20%·접기가능·활성표시 필수) / 메인=콘텐츠.
- 뱃지(건수)는 주문·CS만·사용자 토글. 설정/도움말/로그아웃 하단 회색.

## 5. 브랜드 감성 vs 집중 조율
================================================================
- **장식강도 3단계**: 차분(기본)/중간/맥시멀. 기본=마스코트 정적·분수대 정지 or 매우느린 1회·장식 최소.
- **차분 모드에서 끄는 순서**: ①무한루프 애니(분수대 물흐름) ②자동재생·주변부 모션 ③마스코트 모션 ④장식 일러스트·배경 ⑤채도.
- **모션 기준**: prefers-reduced-motion 자동존중(WCAG 2.3.3) · 5초초과 정지버튼(2.2.2) · 이동 대신 페이드 · 무한루프 금지.
- 은유 라벨 기능 병기 필수(은유 단독 금지) · 브랜드 감성은 매출위젯/빈상태/온보딩 등 비핵심작업 영역 한정 · 작업영역(테이블·폼)은 중립.

## 6. 셀러툴 벤치마크 (정정 포함)
================================================================
- 아이템스카우트·판다랭크 = 키워드·상품 분석 특화(경쟁강도·검색량·예상매출 상단).
- ★셀러오션=커뮤니티·셀러고=품절/재고 솔루션 → **대시보드 벤치마크 부적합·제외**.
- ★**네이버 스마트스토어센터 홈(2025.5.29 개편)=최적 벤치마크**: 신규주문·정산예정·미답변문의·스토어진단(상위3상품) 우선. 주문단계별(결제대기→신규→발송→배송→구매확정) 건수 카운트 To-do. 모바일 뱃지=신규주문/문의/클레임.
- 공통: **행동필요(주문·품절·CS)=To-do 큐 / 모니터링(매출·순위)=KPI 카드** 분리가 정답.

## 7. 실행 로드맵 + 재조정 임계
================================================================
- 1단계(즉시·최대효과): IA 재편 먼저(사이드바 16→5~6·카드 26→4~6·"오늘 할 일" 신설). 시각 리팩터는 이후. 합격=첫화면 8개 이하·primary 1개.
- 2단계: 상단바/사이드바/메인 역할확정·KPI 4카드·8pt 그리드·60/30/10·시맨틱색 브랜드 분리·활성표시.
- 3단계: 장식강도 토글(기본 차분)·prefers-reduced-motion·분수대 루프정지·은유 병기·경보 4단계.
- 4단계: 체크리스트 7항목 자가검증·주1회 15분 리뷰로 안 보는 카드 제거.
- **재조정 신호**: ①사이드바 8개 초과 재증가 ②카드 8개 초과 ③빨강 상시 3개 초과 ④"어디서 시작할지 모름" → 즉시 재편.

## 작업 유의사항/원칙 (신규)
================================================================
- **#216** UI 리팩터는 **IA 재편(그룹핑·우선순위·점진공개) 먼저, 시각(색·폰트·모서리) 나중**. 순서 역전 시 과밀 유지. 정보밀도는 Cowan 4청크(KPI≤4)·5~7 네비 상한.
- **#217** ADHD/집중 UX = 단일 초점·primary 1개·모션 절제(prefers-reduced-motion 존중·무한루프 금지)·빨강은 실위급만·장식은 사용자 토글(기본 차분).
- **#218** 브랜드 강조색은 시맨틱색(성공초록)과 분리 · 은유 라벨은 기능 라벨 병기 · 브랜드 감성은 비핵심작업 영역 한정.
