## 3. 광고법 방어선 = 게이트5 legal-lint 사전 (실전판)

| 표현 패턴 | Risk | 대응 규칙 |
|---|---|---|
| "체온이 내려가요" 등 신체효과 | ⚠ 중간 | "덮는 순간" 등 조건부 표현이면 허용 |
| Q-max 등 미입증 수치 | 🔴 | 근거(시험성적서) 없으면 자동 삭제 |
| "에어컨 대체" 등 단정 | 🔴 금지 | 비교·대체 단정 표현 차단 |
| "단 하나의 선택" 등 최상급 | ⚠ | 최상급→일반 표현 치환 |
| 시험성적서·인증마크 | ✓ | 실물 스캔만. AI 생성 절대 금지 |

이식처: copy-writer.ts `legal-lint` 모듈.
- 조건부 표현 허용 로직 포함 (단정은 막되 "~하면" 조건부는 통과)
- Groq 생성 시 1차 필터 + 게이트5 2차 검수

## 4. AI 이미지 사용 5계명 = Firefly 연동 가드레일

대표님 외주 실전 검증 규칙:
1. 제품 자체 = 반드시 실촬영 (AI 제품이미지 = 광고법 위반 리스크)
2. AI = 배경·환경·레퍼런스만
3. 인물(얼굴·손) AI 생성 절대 금지 (왜곡 + 모델 동의 이슈)
4. 생성 후 원본과 조명·톤 매칭 보정 필수
5. 공적 서류(시험성적서·인증마크) = 실물 스캔만

이식처: 게이트3(광합성 샤워) art-director Firefly 호출 가드레일.
- 5계명 위반 프롬프트 차단 로직
- "제품 누끼 + AI 배경 합성"이 표준 패턴 (제품은 실물, 배경만 AI)

## 5. 컨펌 게이트(CP) = 우리 5게이트의 검증된 원형

| 외주 CP | 우리 게이트 | 공통 원리 |
|---|---|---|
| CP0 기획안 컨펌 | 뉘 집 씨앗인가 | 방향 확정 = 확정 원고 잠금 |
| CP1 컨셉 시안(3섹션만) | 화단 스케치 | 30% 지점 방향 리스크 해소 |
| CP2 전체 1차 시안 | 꽃 까꿍 | 미세 수정만 수용 |
| CP3 최종 납품 | 비밀 정원 입장 | 출구 (JPG/HTML) |

결정적 통찰 — CP1 "3섹션만 먼저" 전략:
- 11섹션 다 만든 뒤 "컨셉 다른데?" = 최악 (재작업)
- 작업 30% 지점(핵심 3섹션)에서 방향 먼저 확정 → 리스크 조기 차단
- 이식: 게이트2(화단 스케치)에서 *대표상품 핵심 3블록 먼저 컨펌* 후 전개

수스존은 4종 패턴마다 다른 보조색 (전체 화이트 그리드 안에서 패턴만 컬러풀 = 갤러리):

이식처: 꽃틔움 무드 팔레트의 *상품 변형별 보조색* 패턴.
구조만 차용 — 색은 꽃틔움 레드/핑크 코어 + 무드 팔레트로.

## 7. 타이포 믹스 전략 (구조 차용)

수스존: 한글 굵은 산세리프 + 영문 세리프/이탤릭 = "프리미엄 + 위트"
꽃틔움 변환: 한글 Pretendard + 영문 둥근 산세리프 = "친근 + 위트"
→ 영문 라인명을 포인트로 쓰는 *믹스 원리*는 동일, 폰트 선택만 꽃틔움 톤.

## 8. 운영 효율 데이터 (ROI 근거)

수스존 프로젝트 효율화 수치 (handoff 기준):
- Firefly 배경 5종 생성 = 110분 (촬영 공수 0원 대체)
- PS 누끼 + Hero 합성 = 20분
- Figma 토큰 세팅 = 30분
- 토큰 기반 = 수정 대응 시간 50% 단축

이식처: 꽃틔움 엔진 ROI 목표치.
- 외주 1건 = 수일 (수동)
- 꽃틔움 엔진 목표 = 게이트 컨펌 포함 수십분 (자동)
- "토큰 1곳 수정 → 전체 자동 반영" = 결 맞추기(Sync) 층의 검증된 가치

1. [엔진] Satori 렌더링에 모바일 430px / 데스크톱 860px 듀얼 기준 적용

6. [Sync] 토큰 1곳 수정 → 전체 반영 구조 (수정 50% 단축 검증됨)
