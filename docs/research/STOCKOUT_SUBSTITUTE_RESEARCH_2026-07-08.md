# 품절 → 대체 → 재소싱 실무 로직 & 앱 적용 리서치 (2026-07-08)

Authoring: DESKTOP(심층 리서치). 권위: SUBSTITUTE/PL-3 실전 운영로직·UX 기본값·문구·임계 근거. 출처: 스마트스토어 페널티 개편(2024·2025)·셀러 실무 블로그·프리미엄 툴(셀러고/아이템스카우트/판다랭크) 벤치마크.

## 0. 3대 핵심 결론
================================================================
1. **품절은 "삭제" 아닌 "방어"** — 주문 후 품절취소 = 건당 페널티 2점(선물하기 3점). 30일 10점+비율 40%↑ = 단계 제재. 드랍십 최우선 = 재고 소량노출 + 옵션단위 품절 + 7~10일 예약구매 전환.
2. **대체상품 = 기존 리스팅 재활용, 단 규칙 준수** — 리뷰·URL·판매지수는 상품ID 귀속(상세만 교체 가능). BUT 같은 카테고리 + 상품명 핵심키워드 유지 + 대표이미지 급변경 금지 + 구매지수 살아있을 때 수정. 안 지키면 로봇이 다른 상품군 재분류→순위 리셋.
3. **앱 = "감지→판단→실행" 반자동** — 감지·경보·대체후보 제시는 자동, 최종 전환·상품명·가격은 사람 체크포인트. 페널티·상품불일치 클레임 동시 예방.

================================================================
## 1. 품절·재고 운영 수치 기준표 (앱 기본값)
================================================================
| 항목 | 권장 기본값 | 근거 |
|---|---|---|
| 드랍십 재고 노출 | 실재고 연동 or 소량(30~50) | 999 무제한=과주문→품절취소→페널티 |
| 재고 임계 경보 | 재고 10개 or 5일치 판매량 中 큰 값 | 재주문점=일평균×리드타임+안전재고 |
| 안전재고(간이) | (일최고판매×최대리드) − (일평균×평균리드) | 통계식: 수요표준편차×Z(95%=1.65)×√리드 |
| 품절취소 페널티 | 2점(선물하기·정기구독 3점) | 익영업일 부과·고객변심 0점 |
| 제재 임계 | 30일 10점 & 비율 40%↑ | 주의→경고(7일 등록불가)→이용제한 |
| 품절 유지 허용 | 2~3일 유지 / 7~10일 예약전환 / 장기 대체·삭제 | |

**품절 방어 액션 순서(랭킹 영향 적은 순)**: ①옵션 일부 품절 ②재고보충/재소싱 ③예약구매 전환(노출·판매·클릭 유지) ④대체상품 전환 ⑤판매중지/삭제(최후·신규ID 재등록=어뷰징 패널티).

## 2. 대체상품 소싱·큐레이션 체크리스트
================================================================
**후보 선정**: □동일 카테고리(적합도 핵심) □가격 ±10~15% □동급이상 스펙(상위호환) □상세/썸네일 저작권 가부 □공급사 재고·배송 안정 □최저가 정책.
**백업 관리**: 대체 체인 A→B→C 사전확보·상품카드 저장·소싱URL/공급가/예상가/우선순위(1·2·3)/즉시전환 플래그.
**리스팅 재활용 규칙**: ✅카테고리 유지 ✅상품명 핵심키워드 유지(확장만) ✅대표이미지 급변경 금지 ✅구매지수 살아있을 때 ⚠️상위호환·동급 제한(불일치 클레임=리뷰블라인드·판매중단) ⚠️삭제후 동일 신규ID=어뷰징.

================================================================
## 3. 품절→대체→재소싱 워크플로우
================================================================
①감지(자동) ②분류(일시/장기/단종·반자동) ③1차방어(옵션품절 or 예약전환·사람) ④대체검토(앱이 백업후보 제시·반자동) ⑤전환실행(규칙준수·사람) ⑥재소싱·백업갱신(사람) ⑦네이버 반영(수동·확인절차) ⑧성과추적(자동).
- **자동화**: 품절/가격 감지·임계경보·대체후보 매칭·성과추적·체크리스트 알림.
- **사람 판단**: 대체 최종선택·상품명/상세 수정·가격/마진 확정·네이버 최종 반영승인.

## 4. 앱 UX 기본값·문구·입력필드 (바로 적용)
================================================================
**경보/안내 카드 문구(이모지는 앱에선 Lucide 대체·#3-1)**:
- 재고임박: "'{상품명}' 재고 {n}개 — 소진 전 재소싱/대체 준비"
- 공급사품절: "'{상품명}' 공급사 품절 — 주문취소 시 페널티 2점! 옵션 품절처리/예약구매 전환 검토"
- 대체후보: "저장한 대체상품 {n}개 — 카테고리·가격 확인 후 전환"
- 예약권유: "7~10일 내 재입고면 예약구매로 노출·순위 유지"
- 전환 전 확인: "같은 카테고리? 상품명 핵심키워드 유지? 대표이미지 급변경 안 함?"

**기본값**: 재고경보 10개/5일치·안전재고 미입력시 일평균×7일·소진임박 1인 1~2개 제한.
**대체 입력필드**: 대체상품명 / 소싱처 / 소싱URL·번호 / 공급가 / 권장판매가 / 재고상태 / 카테고리일치 체크 / 우선순위 / 상위호환 여부 / 메모(클레임·저작권).
**재고·가격 수동반영 주의**: "네이버 자동반영 안 됨 — 스토어센터에서 직접" + 체크리스트(판매가·옵션재고·마진·배송비·반영후 노출확인).

## 5. 파워셀러 확장 아이디어 + 벤치마크
================================================================
1. 품절 패턴분석 → 불안정 공급사 리스트 자동생성(소싱 회피)
2. 대체상품 성과추적(전환 전후 판매·순위·전환율)
3. 잘팔리는상품 예측(품절빈도=수요신호→사입후보 승격)
4. 재고회전 관리(죽은 리스팅 자동정리)
5. 시즌 수요예측(데이터랩 연동)
**벤치마크**: 아이템스카우트(경쟁강도=상품수÷검색수·적정재고예측) / 판다랭크(월검색량·경쟁·6개월판매·시즌성) / 셀러고(품절처리 특화·원클릭·검색→품절10초) / 셀러툴 API(재고 자동연동·품절 자동처리).

## 6. 앱 반영 로드맵
================================================================
- **1단계(MVP·지금)**: SUBSTITUTE 입력필드+전환 전 체크리스트 팝업 · PL-3 "자동반영 안 됨" 경고+5단계 체크 · 재고경보 10개/5일치 · 페널티(2점/선물3점) 안내배너.
- **2단계**: 대체 체인 A→B→C·백업 슬롯 · 예약구매 전환 가이드(7~10일) · 품절패턴/대체성과 대시보드.
- **3단계(파워셀러)**: 데이터랩·아이템스카우트류 지표 연동·시즌예측·사입후보 · 셀러툴 API 재고 자동동기화.

## 7. 주의사항 (Caveats)
================================================================
- 페널티 점수는 하드코딩 금지·안내문구+"최신 기준 스토어센터 확인" 링크(정책 수시 변동·2025.6 플러스스토어 경고↑ 상품 미노출).
- "리뷰 유지한 채 상품 통째 교체 금지" 공식 명문 조항 미확보 → 동급·유사·같은카테고리+핵심키워드 유지 보수 설계 권장.
- 안전재고·회전율 수치는 물리재고 소매 기준 → 드랍십은 판매속도·리드타임 경보로 응용.
- 옵션품절 방치·예약구매 활용은 셀러 경험담(공식 권장 아님) → "단기·명분 범위" 제한 안내.
