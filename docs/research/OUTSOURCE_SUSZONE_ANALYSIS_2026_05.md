## 0. 한 줄 결론

이 외주 파일은 단순 상세페이지가 아니라 **"AI 협업용 디자인 생산 프로토콜"**이다.
대표님이 무의식적으로 설계한 이 구조 = 우리 GREENHOUSE_ATELIER 5게이트의 실전 검증판.
→ 5개 핵심 자산을 게이트별로 이식한다.

## 2. 자산별 상세 — 무엇을 어떻게 이식하나

### 2-1. 13색 CSS 변수 시스템 → 색 비율 황금률
외주 구조:
```
Base 75%   : paper, breeze, teal, icy 등 (바탕·주조색)
Pattern 15%: berry, watermelon, cherry, lavender (강조·포인트)
Line 10%   : line (구분선·테두리)
+ Annotation: 작업 주석 전용색 (좌표/firefly/ps/ai/legal/dna)
```
이식: 꽃틔움 토큰도 **75/15/10 비율 구조**로 재편.
- Base 75%: 화이트/아이보리 + (무드별 베이스)
- Pattern 15%: 꽃틔움 레드/핑크 (코어 강조)
- Line 10%: 꽃 라인 디바이더 색
→ "역할별 색 변동"도 이 비율은 유지. 비율이 브랜드 안정감의 비밀.

### 2-2. DNA 8종 표 → 게이트1 "비즈니스 임팩트" 자동 생성
외주의 핵심 무기. 디자인을 *돈의 언어*로 번역:
```
DNA          | 출처              | 적용섹션 | 비즈니스 임팩트
레이어드타이포 | veware 화장품     | SEC03   | Brand Authority +15%
블록컬러충돌   | veware 하단       | SEC04   | 이탈률 -10%
선아이콘시그널 | 냉감이불 경쟁사   | SEC05/7 | 가독성 +20%, 모바일 스캔성↑
상표등록배지   | 실제번호 40-...   | SEC11   | 신뢰성↑·짝퉁방지
Folio시스템   | veware 구조       | 전섹션  | 카탈로그→아트북 전환
```
이식: 게이트1(씨앗 인터뷰)에서 상품 진단 시,
*"이 상품엔 이런 디자인 DNA가 맞고, 그 이유는 이런 비즈니스 임팩트"*를
자동 생성. 1인 셀러가 디자인 스튜디오급 전략 논리를 갖춤.
→ role-engine.ts + identity-extractor.ts에 "임팩트 서술" 출력 추가.

### 2-3. 법적 수정 5건 → 게이트5 표시광고법 자동 검수
외주가 DAY0에 처리한 표시광고법 제5조(실증책임) 리스크:
```
수정 전        → 수정 후         | 리스크 | 근거
"체온을 낮춰주는" → "열감을 분산시키는" | HIGH  | "체온"=의학·기능성, 실증의무
"단 하나의 선택"  → "선택"          | -     | 최상급 표현 규제
```
이식: 게이트5(비밀 정원 입장)의 네이버 정책 가드를 **표시광고법 검수까지 확장.**
- 금지 표현 사전(辭典): 최상급("단 하나","최고","1위"), 의학·기능성("체온","치료","항균"
  근거 없을 시), 절대성("100%","완벽") 등
- 카피 생성(Groq) 단계에서 1차 필터 + 게이트5에서 2차 검수
→ copy-writer.ts에 legal-lint 규칙 추가. 위반 시 대안 표현 제시.

### 2-4. 이미지 매핑 매트릭스 → 게이트3 근거 기반 배치
외주는 15장을 *섹션-역할-근거*로 매핑:
```
섹션      | 역할        | 이미지          | 선정 근거
SEC01 HERO| 메인비주얼  | IMG_3834        | 유일 Hero급 인물구도·감성톤 대표
SEC06 하단| 시그니처    | IMG_3812        | "I cherish you"·선물 SEO 유입 결정
SEC11 CTA | 최종플로팅  | IMG_3834(재활용)| 수미상관·Hero↔CTA 일치로 인상강화
```
핵심 통찰: **Hero 이미지를 CTA에 재활용(-2° 회전) = 수미상관 효과.**
이식: 게이트3(광합성 샤워)에서 곳간 이미지 선택 시,
*역할 + 선정 근거*를 함께 제시. "이 컷을 여기 쓰는 이유"를 명시.
→ lifestyle-picker.ts에 "배치 근거" 메타데이터 추가. 수미상관 규칙 내장.

### 2-5. 11섹션 + Folio → 게이트2 아트북 서사 구조
외주 11섹션 흐름:
```
01 HERO(3초 Hook) → 02 VERIFIED(상표등록) → 03 PRODUCT →
04 EMPATHY(페인포인트) → 05 USP(3그리드) → 06 PATTERN(★메인,A/B교차) →
07 DETAIL(비대칭그리드) → 08 MATERIAL(스펙표) → 09 CARE(4아이콘) →
10 SIZE(다이어그램) → 11 CTA(에디션·가격·수미상관)
```
+ Folio 페이지번호(01/11): "카탈로그 → 아트북" 격상 장치.
이식: 게이트2(화단 스케치)의 블록 골격에 이 서사를 옵션으로.
- 단, 꽃틔움은 위트 톤이므로 Folio는 "꽃 번호"로 변형 가능
- 12 슬라이스 카탈로그(기존 설계)와 매핑: HERO=hero_hook, EMPATHY=painpoint,
  USP=usp_badges, PATTERN=lifestyle_scene, MATERIAL=certification 등
→ section-builder.ts 골격에 "아트북 서사" 프리셋 1종 추가.

## 4. AI-Native 업그레이드 — 외주 대비 우리의 진화

외주(2026.04)는 Photoshop/Illustrator 수작업 전제.
우리는 **커넥터 + 클로드 디자인 + 자체 엔진**으로 자동화:

| 작업 | 외주 방식 | 꽃틔움 엔진 방식 |
|---|---|---|
| 배경 합성 | Photoshop Generative Fill 수동 | Firefly Object Composite (게이트3) |
| 카피 작성 | 수동 + 법적 검수 수동 | Groq 생성 + legal-lint 자동 (게이트5) |
| 레이아웃 | HTML 수작업 | 모듈 조립 엔진 (게이트2) |
| 디자인 완성 | Adobe 수작업 | 클로드 디자인 (게이트4) |
| 이미지 배치 | 수동 매핑 | 곳간 + 근거 자동 추천 (게이트3) |
| 대량 생산 | 1건씩 수작업 | Satori+Sharp 자동 (수확 후) |

→ 외주 1건 = 수일. 꽃틔움 엔진 = 게이트별 컨펌으로 수십분 목표.
