## 0. 한 줄 결론 (TL;DR)

하나의 적응형 파이프라인을 만든다:
**카테고리 인텔리전스(DNA) → 슬롯/퍼널 맵 → 전략 조립 → 그라운디드 생성 → 채점/학습/자산화**

이것이 전상품 공통일 수 있는 이유: 카테고리별 지식이 *코드(하드코딩)가 아니라 데이터(버전드 "카테고리 DNA 카드")* 에 산다. DNA 카드는 네이버 공식 DataLab 쇼핑인사이트 API가 채운다.

네이버 한정 핵심 3가지:
1. 대표 썸네일(대표이미지)은 2024-10-28 정책상 **텍스트 없는 깨끗한 단일 제품 사진**이어야 한다(가격/프로모션/배송 문구·무관한 소품 금지). 네이버 쇼핑 랭킹은 클릭률(CTR)을 보상하고 저품질/어뷰징 이미지를 처벌한다.
2. 상세페이지는 보편적 설득 시퀀스를 따른다: 훅 → 문제 → 해결/USP → 증거 → 기능 → 사용/크기 → 신뢰 → CTA.
3. 생성은 기본적으로 **Adobe Firefly Image 모델(상업적 안전 베이스 사진)**, 그리고 **Google Nano Banana Pro/Nano Banana 2(Gemini 기반, Firefly 내장)**로 편집·이미지 내 텍스트·다중 참조 일관성·검색 그라운딩을 라우팅한다. 시드 + 구조/스타일 참조로 재현성을 확보하고, 모든 (프롬프트·모델·설정·그라운딩·결과·평점·CTR/CVR) 튜플을 로깅하며, 최고 평점 프롬프트를 자동 디폴트로 승격해 프롬프트 라이브러리를 복리 자산으로 만든다.

## 1. 핵심 발견 (Key Findings)

1. **네이버는 "판매 티 나는" 썸네일을 처벌한다.** 2024-10-28부터 네이버 쇼핑 대표이미지 정책은 대표이미지에 가격/할인/혜택 문구, 배송/설치 정보, 프로모션 문구, 옵션 단독 상품, 무관한 상품, 장식 테두리/마크를 금지한다. 위반 시 미노출 + 클린 프로그램 제재. 이것이 글로벌 마켓(아마존은 인포그래픽 메인 이미지를 일부 허용)과 가장 다른 한국 특이점. 썸네일은 깨끗한 제품 사진이어야 하고, 모든 설득 텍스트는 상세페이지로 옮긴다.

2. **네이버 랭킹 = 적합도 + 인기도 + 신뢰도** (이제 트리 기반 ML 모델로 계산). 이미지는 랭킹에 *간접적이지만 강력하게* 영향: 높은 썸네일 CTR이 인기도를 먹인다(인기도는 최근 7일 클릭수·판매·리뷰수·찜·최신성 사용), 저품질/어뷰징 이미지는 신뢰도를 깎는다. 즉 이미지 최적화는 전환 지렛대일 뿐 아니라 랭킹 지렛대다.

3. **네이버 공식 DataLab 쇼핑인사이트 API가 카테고리 인텔리전스의 백본.** `https://openapi.naver.com/v1/datalab/shopping/` 하위 8개 POST 엔드포인트(categories, category/device, category/gender, category/age, category/keywords, category/keyword/device, category/keyword/gender, category/keyword/age). 카테고리/키워드별 상대 클릭비율 시계열을 기기(pc/mo)·성별(m/f)·연령 6구간(10~60)으로 세분 반환. 중요 함정: API는 "카테고리별 인기검색어 top-500" 리스트를 반환하지 않음 — 그건 웹 UI 전용. 따라서 골든키워드 발굴은 웹 UI 또는 sanctioned 스크래핑 필요.

4. **Firefly의 차별점 = 상업적 안전 + 제어 표면; Nano Banana의 차별점 = 추론·텍스트·그라운딩.** Firefly 네이티브 모델은 구조 참조(구도 일치, strength 1~100), 스타일 참조, 재현용 시드, 콘텐츠 타입, 커스텀 브랜드 모델 지원. Adobe는 Google Gemini 기반 모델(Nano Banana / Nano Banana Pro / Nano Banana 2)을 Firefly Generate Image·Boards·Photoshop Generative Fill에 통합 — 최대 14장 참조 일관성, 정확한 이미지 내 텍스트(한글 포함), 그리고 Google 검색 그라운딩 추가.

5. **그라운딩은 실세계 충실도를 돕고, 추상 무드는 해친다.** 검색/이미지 그라운딩은 실제 사물·실제 제품 지오메트리·실제 장소/종·인포그래픽·사실 staging의 정확도를 높인다; 추상/무드/예술 이미지엔 도움이 거의 없다(혹은 연산 낭비). 슬롯별로 전술적으로 사용.

6. **프롬프트-옵스(prompt-ops)는 이미 정립된 패턴이라 그대로 복제 가능.** LLM 업계는 수렴했다: 프롬프트를 코드에서 분리, 레지스트리에 저장, 불변 버전관리, 버전마다 메타데이터, 평가(eval) 게이트를 통과해야 환경 승격, 프로덕션 모니터링 피드백 루프. 동일 아키텍처가 창작 자산 프롬프트에 그대로 적용 — 인간 평점 + 하류 CTR/CVR을 eval 신호로.

## 2. Pillar 1 — 슬롯 & 퍼널 기반 이미지 전략 (카테고리 무관)

### 대표 썸네일 — 보편적 CTR 동인
- **~200px 가독성:** 빽빽한 그리드에서 1초 미만에 판정. 제품이 프레임을 지배(작업 휴리스틱 85~95% 점유), 높은 밝기/대비, 깨끗한 배경. 흐릿/저해상은 네이버가 자동 후순위화.
- **해상도:** 최소 160×160px(1:1), 권장 가로 ≥1,000px, 안전 목표 ~1,300px. 정사각 1:1 권장.
- **정책 가드(2024-10-28):** 가격/할인/카드혜택 문구 금지; 배송/설치 문구 금지; 프로모션 문구("무료배송","네이버 No.1") 금지; KC/원산지/AS 오버레이 금지; 기본가에 판매하는 그 제품만; 무관한 제품 금지; 인위적 테두리/도형/마크 금지. 소품은 판매 제품이 명확히 식별되는 한에서만 허용(소파=제품, 러그/램프/식물=소품).
- **엔진 시사점:** 썸네일 생성기는 *텍스트 없는 단일 제품* 이미지를 만들어야 한다. 가치제안/텍스트 오버레이는 상세페이지 슬롯에만.

### 상세페이지 — 보편적 고전환 섹션 시퀀스
한국 CRO 실무는 카테고리 무관한 설득 아크로 수렴; 각 슬롯의 *내용*만 바뀐다.

| # | 슬롯 | 전환 직무 | 콘텐츠 유형 | 톤/구도 |
|---|------|-----------|-------------|---------|
| 1 | 훅/히어로 | 스크롤 정지; "누구를 위한 + 무엇" 3초 안에 | 무드 또는 제품맥락 히어로 | 대담·열망형, 메인카피 10~15자 |
| 2 | 문제/공감 | 구매자의 고통/두려움/손실 끄집어냄 | 개념/감정 | 공감, 어두운 팔레트 경향 |
| 3 | 해결/USP | 제품을 답으로 포지셔닝; 단 하나의 차별점 | 제품맥락 | 자신감·깨끗 |
| 4 | 기능 클로즈업 | 기능 설명 → 부가가치 지각 | 설명/매크로 디테일 | 선명·리터럴 |
| 5 | 사용/라이프스타일 | 소유 상상 ("혜택" 프레이밍, 단순 누끼 아님) | 라이프스타일/실사용 | 따뜻·공감 |
| 6 | 크기/규격 | 사이즈 불확실성 제거; CS 부담↓ | 설명/도해 | 중립·고가독 |
| 7 | 신뢰/사회증거 | 리뷰·전후·인증·수치·판매량 | 스펙/증거 | 신뢰; 형용사보다 숫자 |
| 8 | 기프트/상황(선택) | 구매 이유 확장 | 라이프스타일 | 시즌 |
| 9 | CTA/마무리+브랜드스토리 | 욕망의 순간에 전환; CTA 반복 배치 | 브랜드/CTA | 결단형 |

한국 특이점(인코딩 가치):
- **2024년 한국 이커머스 거래의 약 75%가 모바일(약 2,300억 달러 시장, PCMI/Kfriday); Mordor Intelligence는 2025년 스마트폰을 거래액의 73.40%로 추정.** → 모바일 퍼스트 설계(본문 ≥14pt, 핵심 메시지를 첫 3초에, CTA 반복).
- 신뢰 요소(인증 마크, 누적 판매량, 전후)는 맨 아래만이 아니라 *스크롤 중간에 반복*.
- 신선식품은 배송/택배 정보를 상단으로.

### CRO + 이미지 SEO 과학 (2025–2026, 글로벌+네이버)
- 이미지 단독 A/B 변경은 보통 헤드라인/가격 변경보다 작은 상승(3~8%)이지만, 전 트래픽에 복리로 적용되고 고객 세그먼트를 왜곡하지 않는다. **배경/맥락이 가장 큰 지렛대 변수이니 먼저 테스트** — 라이프스타일 vs 흰 배경 스왑은 큰 폭으로 움직일 수 있다.
- **Wayfair의 AI 룸 시각화 "Discover" 탭이 라이프스타일 맥락 효과를 예시 — CTO Fiona Tan은 "Discover 경험에 참여하는 사용자는 훨씬 더 생산적이며, 즉 전환율이 훨씬 높다"고 언급(US Chamber CO—, 2025); Snap Inc. 연구는 쇼핑객의 80%가 AR 도움을 받은 구매에 더 큰 확신을 느낀다고 발견.** UGC 스타일 이미지가 정제된 스톡을 능가하는 경우 多.
- 승리 전략은 혼합형: 욕망을 만드는 라이프스타일 + 실무 질문에 답하는 제품-흰배경.
- 가구는 맥락-vs-깨끗 이미지와 페이지 품질이 얼마나 움직이는지 보여주는 캘리브레이션 포인트: **온라인 가구 CVR은 대략 Overstock 3.1% > Wayfair 2.9% > Ikea 1.7% (Similarweb/DFD News)** — 한 카테고리 안에서도 ~2배 차이, 상당 부분 이미지·페이지 실행에 기인.
- 광고 크리에이티브용 vs 상품페이지 이미지용 테스트는 분리(피드 스크롤에서 이기는 것과 페이지에서 이기는 것이 다름).
- 이미지 SEO 위생: 설명적 파일명 + alt 텍스트, 압축, CDN 전송, 일관된 사이징.

### 일반 상품 → 슬롯 플랜 자동 매핑
엔진이 제품 속성(카테고리, 가격대, 소재, 치수, DataLab 타깃 인구, USP, 기프트성)을 받아 **결정 테이블** 적용: 사이즈 불확실 카테고리(의류·가구)는 크기 슬롯 강제; 신뢰 민감 카테고리(식품·화장품·유아)는 인증+임상/숫자 슬롯 강제; 기프트 편향 카테고리는 기프트 슬롯 추가; 저관여 commodity는 아크 단축. DNA 카드(Pillar 2)가 카테고리-노름 오버라이드 공급.

## 3. Pillar 2 — 자동 경쟁/카테고리 인텔리전스 (sanctioned 방법)

### 네이버 DataLab 쇼핑인사이트 API — 정확 스펙
인증 헤더 `X-Naver-Client-Id` / `X-Naver-Client-Secret` (+ `Content-Type: application/json`). 8개 POST 엔드포인트(`/v1/datalab/shopping/`):
- `categories` (최대 3 카테고리; `category`는 `{name, param:[catId]}` 배열)
- `category/device`, `category/gender`, `category/age` (단일 카테고리 문자열)
- `category/keywords`, `category/keyword/device`, `category/keyword/gender`, `category/keyword/age`

요청 바디: `startDate`/`endDate`(yyyy-mm-dd; 2017-08-01부터), `timeUnit`(date/week/month), `category` 코드(예 50000000=패션의류, 50000002=화장품/미용), `keyword` 배열, `device`(""/pc/mo), `gender`(""/m/f), `ages` 연령코드 배열["10","20","30","40","50","60"]. 응답: `results[].data[]`에 `period`+`ratio`(0~100 상대), 세분 엔드포인트엔 `group` 필드. **함정:** 쇼핑 API 연령코드는 6구간(10~60)으로, 검색어트렌드 API의 1~11 코드와 다름. 일일 쿼터는 보통 앱당 ~1,000콜로 인용되나 개발자 콘솔에서 검증 필요(원문 스펙 페이지 접근 불가였음).

각 신호 → 이미지/컨셉 방향 변환:
- **카테고리/키워드 트렌드** → 타이밍(시즌)·수요 모멘텀 → 어떤 상품 우선·어떤 시즌 무드.
- **기기 분할** → 모바일 우세면 ~200px 가독성·세로 스크롤 우선 설계.
- **성별+연령 분할** → 톤·팔레트·모델 캐스팅·라이프스타일 시나리오. (예: 가구/인테리어는 모바일+여성 강편향.)
- **find_category 헬퍼**(커뮤니티 MCP 도구 naver-search-mcp / naver-api-mcp)로 자연어에서 카테고리 코드 해석 — URL 수작업 매핑 불필요.

### 골든키워드 발굴
API가 top-500 인기검색어를 노출 안 하므로 조합: (a) DataLab 웹 UI의 카테고리별 top-500, (b) 네이버 검색광고 키워드 도구(검색량+경쟁도), (c) 네이버 쇼핑 검색 API(실시간 결과 구성). 교차참조로 고검색량/저경쟁 "골든키워드" 찾고, 상품명(메인키워드+속성+브랜드 순서)과 이미지 컨셉 방향에 투입.

### 경쟁 썸네일/상세 캡처 — ToS 존중 접근
법적 지형(hiQ v. LinkedIn, Meta v. Bright Data, Reddit v. Perplexity/SerpApi): 공개·로그아웃 데이터 스크래핑은 대체로 방어 가능하나, ToS 위반은 계약위반 리스크, 인증/안티봇 우회는 CFAA/DMCA 노출. 베스트 프랙티스: robots.txt를 선의의 신호로 존중, 레이트리밋, 실제 user-agent, 로그인/캡차 우회 금지, PII 제외, 세션 로깅, 그리고 서버측 대량 스크래핑보다 **운영자 트리거 브라우저 캡처**(셀러가 정당히 브라우징하며 경쟁사 공개 썸네일/상세를 *분석용*으로 앱에 캡처) 선호. 결정적: 캡처는 **티어다운/학습용**(관례 추출), **경쟁 이미지 재게시 절대 금지** — 공개라도 저작권 보호. "재현 아닌 분석" 존 유지.

### 재사용 가능한 버전드 "카테고리 DNA 카드"
디자인 시스템(디자인 토큰 = "DNA"), 콘텐츠 템플릿, 경쟁 티어다운 프레임워크에서 직접 차용: 카테고리 판매 관례를 **한 번** 캡처·버전관리하고 그 카테고리 전 상품에 재사용. DNA 카드 = 구조화·불변버전 JSON:
- 썸네일 관례(배경 노름, 제품 각도, 소품 허용 여부, 지배 팔레트, 밝기).
- 상세 슬롯 시퀀스 + 이 카테고리 필수 슬롯.
- 톤앤매너(격식/발랄/임상), 전형 모델 캐스팅, 무드.
- 구도/배치 노름, 중요한 신뢰신호 유형(식품=HACCP, 화장품=임상데이터).
- DataLab 유래 인구통계(연령/성별/기기 편향).
- 출처 provenance + 신뢰도 + 버전번호 + 생성/갱신 타임스탬프.

디자인 토큰처럼 DNA 카드는 **상속·오버라이드**: 베이스 카테고리 카드를 서브카테고리/스토어 브랜드별로 특화, 한 번 갱신하면 그 카드를 참조하는 전 상품에 전파.

## 4. Pillar 3 — 생성형 AI 프롬프트 통제·그라운딩·자산화 (핵심 요청)

### 최신(2025–2026) 프롬프트 해부
Firefly·Nano Banana 가이드 공통 신뢰 구조: **주제 + 표면/세팅 + 빛 + 렌즈/카메라 + 마감/스타일**, 가장 중요한 요소를 앞에, 단순 쉼표 구분, 요소당 ≥3 서술어, 배경은 끝에. 제품 사진 템플릿 예: `[제품, 정확한 묘사], on [표면], [조명 세팅+방향+색온도], [카메라 각도+샷 타입], [마감/미감], clean [배경], professional product photography, sharp focus`.

### 긍정형 프레이밍 vs 네거티브 프롬프트
Firefly 네이티브 + Nano Banana는 **네거티브 프롬프트를 따르지 않음** — 원하는 것을 긍정형으로("no people" 대신 "empty background"). 이건 하드 설계 제약: 프롬프트 조립 시스템이 제외 항목을 긍정형으로 번역해야 함(우리 #86과 일치). (Firefly 네이티브는 별도 visual-intensity 컨트롤 + 구조/스타일/구도/카메라 슬라이더 노출; Nano Banana 등 파트너 모델은 Firefly 네이티브 슬라이더 미지원.)

### 예측성/재현성 제어 표면
- **시드:** Firefly는 출력마다 `seed` 반환; 같은 시드+프롬프트+설정 재사용 시 재현/고정 — 한 파라미터만 변주할 때 핵심(템플릿화 필수). (단 매우 길고 복잡한 프롬프트는 시드 결정성 저하 보고 있음 → 템플릿 프롬프트는 타이트하게.)
- **구조 참조**(`structure.strength` 1~100, 디폴트 50): 구도/윤곽/뎁스 일치 — 카테고리 표준 제품 배치 강제에 이상적.
- **스타일 참조:** 색/조명/질감/무드 일치 — DNA 카드 톤 강제에 이상적.
- **참조 이미지:** Nano Banana Pro는 **정확히 14장 참조 + 최대 5인 일관성 유지**(Google DeepMind 발표: "Nano Banana Pro로 그 어느 때보다 많은 요소를 블렌드 — 최대 14장 사용, 최대 5인의 일관성·닮음 유지"). Google Cloud는 "디자이너를 위한 few-shot 프롬프팅: 14장 참조 업로드로 로고·팔레트·캐릭터 턴어라운드·제품샷 포함 전체 스타일가이드를 동시 로드"라 표현 — 즉 **DNA 카드의 시각 참조 전체를 한 번에 로드하고, 같은 제품을 여러 상세 슬롯·각도에서 일관 유지** 가능.
- **비율/해상도:** 명시적 설정(썸네일 1:1; 상세 패널 세로/롱 비율; Nano Banana 2는 최대 4K + 극단 비율로 롱 스크롤 패널 지원).
- **Firefly 커스텀 모델 / 적응형 합성:** 실제 제품에 subject 모델 학습 또는 실제 제품 사진을 생성 씬에 합성 — 실제 SKU 최고 충실도 경로, 브랜드 일관 배경.

### 그라운딩 — 슬롯별 전술 가이드
- **그라운딩 사용:** 실제 제품 지오메트리/staging, 실제 장소, 실세계 사물, 사실 숫자가 든 인포그래픽/스펙 패널, 현지화/번역 이미지 내 텍스트. 프롬프트를 리서치 질문처럼 구성하면 잘 트리거.
- **그라운딩 미사용:** 추상 히어로 무드, 예술/감정 배경 — 이득 없이 지연/연산만 추가. Nano Banana의 상대적 약점도 추상/예술 생성(Midjourney급이 우위).
- **핵심 한계:** Nano Banana Pro 그라운딩은 대체로 *텍스트* 세계지식 기반; **Nano Banana 2(Gemini 3.1 Flash 기반)는 *이미지* 검색 그라운딩 추가** — Google 발표: "모델이 Gemini 실세계 지식 베이스에서 끌어오며, 웹 검색의 실시간 정보·이미지로 특정 주제를 더 정확히 렌더." 신규 SKU엔 그라운딩이 제품을 가져오지 못함 — **대신 참조 이미지 공급**. Thinking Mode는 기본 OFF, 복잡 인포그래픽/그라운딩+공간추론 시에만 ON.

### 예측가능·템플릿가능 프롬프트 조립
결정적 조립 시스템 구축:
- **변수**(제품명·소재·색·치수·DNA 카드의 인구/팔레트).
- **블록**(슬롯별 재사용 프롬프트 조각: 히어로 블록·기능매크로 블록·크기도해 블록).
- **결정 테이블**: (슬롯유형 × 카테고리 DNA × 그라운딩 ON/OFF × 모델 선택 × 비율 × 시드정책) → 완전 조립된 프롬프트 + API 파라미터.

이로써 프롬프트→결과가 템플릿·반자동화 가능할 만큼 재현적이 되고, 모델/그라운딩/시드가 추측이 아니라 명시적 컬럼이 된다.

### 학습 + 자산화 시스템 (프롬프트-옵스를 창작에 적용)
LLM 프롬프트 레지스트리 패턴 미러링:
1. **모든 생성을 불변 튜플로 로깅:** {프롬프트 템플릿+해석된 변수, 모델+버전, 그라운딩 ON/OFF, 시드, 구조/스타일 참조, 비율, 결과 URL, 인간 평점(엄지/별), 하류 성과(썸네일 CTR·상세 CVR·체류시간)}.
2. **불변 버전관리:** 프롬프트 버전을 절대 변형 안 함; 편집은 author/타임스탬프/근거 메타와 함께 새 버전. "환경"(draft/candidate/default)을 특정 버전에 핀; 승격은 코드 배포가 아니라 포인터 변경.
3. **평가 루프:** 인간 평점이 1차 eval; 하류 CTR/CVR이 ground-truth 보상(창작 자산에 적용한 RLHF식 인간 피드백 루프). 평점+성과로 인컴번트를 충분 샘플로 이기면 카테고리 자동 디폴트로 승격.
4. **복리 라이브러리:** (카테고리×슬롯)별 최고 프롬프트가 재사용 디폴트; DNA 카드가 현 승리 프롬프트 버전 참조. 시간이 갈수록 라이브러리가 해자 — 알려진 카테고리의 새 상품은 검증된 프롬프트에서 시작.

참조 아키텍처 도구(패턴 예시): PromptLayer, Maxim, Braintrust, MLflow Prompt Registry, Langfuse — 전부 레지스트리+버전관리+eval+모니터링. 필요한 부분을 Prisma/Supabase로 복제.

### 자연 개입점이 있는 반자동화 (Human-in-the-loop)
재작업 비용·모호성·브랜드 영향·비가역성이 높은 곳에 체크포인트; 저위험 단계는 무인 실행. 권장 개입점(비모달·편집가능·강제 없음):
- **무인 실행:** DataLab 풀, DNA 카드 갱신, 프롬프트 조립, 1차 배치 생성(슬롯당 N 변주를 저해상으로).
- **자연 체크포인트 #1 — 컨셉/DNA 리뷰:** 대량 생성 전 슬롯 플랜+톤 확인(여기서 바꾸는 게 가장 쌈).
- **자연 체크포인트 #2 — 변주 선택:** 생성 변주에서 선택/평점(이게 eval 신호 — 평점 클릭 = 선택 클릭 하나로).
- **자연 체크포인트 #3 — 발행 전 리뷰:** 특히 정책 준수(텍스트 없는 썸네일·금지 오버레이 없음) + 그라운디드 사실/숫자 검토.
- **무인 실행:** 업스케일, 포맷/비율 익스포트, 자산 버전관리, 로깅.

HITL 베스트 프랙티스: 승인자+편집+전후+신뢰도+소요시간 로깅(학습에 먹임); 신뢰도/위험 임계로 라우팅; 고무도장 방지 위해 로테이션; 체크포인트는 시간 아닌 초 단위.

## 5. 3축이 하나의 적응형 엔진으로 연결되는 방식

```
DataLab API + sanctioned 경쟁 캡처
      │ (카테고리 리서치·인구통계·관례)
      ▼
[카테고리 DNA 카드]  ── 버전드·상속·오버라이드 가능
      │
      ▼
[슬롯/퍼널 맵]  ── 결정 테이블: 제품 속성 × DNA → 필수 슬롯
      │
      ▼
[전략 조립]  ── 변수 + 블록 + 결정 테이블 → 프롬프트 + 모델 + 그라운딩 + 시드 + 비율
      │
      ▼
[그라운디드 생성]  ── Firefly 네이티브(안전 베이스) / Nano Banana(편집·텍스트·다중참조·그라운딩)
      │
      ▼
[채점/학습/자산화]  ── 튜플 로깅 → 인간 평점 + CTR/CVR → 승자 승격 → DNA+프롬프트 라이브러리 갱신
      │
      └──────────► DNA 카드 + 디폴트 프롬프트로 피드백 (복리 루프)
```

카테고리 인텔리전스 = *입력 계약*; DNA 카드 = *재사용 인터페이스*; 하류 전부 = 전상품 공통 기계장치. 새 카테고리 = 새 DNA 카드; 알려진 카테고리의 새 상품 = 검증 프롬프트로 즉시 웜스타트.

## 6. 권고 — 단계별 (원문 17주+ 로드맵을 우리 스택/레인에 맞춤)

**Stage 0 — 기반 (1~4주):** Prisma/Supabase 데이터 모델: `Product`, `CategoryDNA`(버전드), `SlotPlan`, `PromptVersion`(불변), `Generation`(로그 튜플), `Rating`, `PerformanceMetric`. DataLab 쇼핑인사이트 API(8 엔드포인트) + find_category 리졸버 연결. 2024-10 썸네일 정책을 하드 발행전 검증기로 인코딩.

**Stage 1 — 단일 카테고리 수직 슬라이스 (5~10주):** 한 카테고리 선택; DNA 카드 반수동 구축(DataLab 인구 + 소규모 운영자 트리거 경쟁 티어다운). 슬롯 결정 테이블 + 프롬프트 조립(변수/블록/결정 테이블) 구현. Firefly API(네이티브+시드+구조/스타일 참조) = 베이스 사진, Nano Banana(Firefly 내) = 편집/텍스트/그라운딩. 3개 인간 체크포인트를 비모달·편집가능 리뷰로 출시.

**Stage 2 — 학습 루프 (11~16주):** 전체 튜플+인간 평점 로깅 계측. 하류 CTR(쇼핑파트너센터/상품리포트) + CVR(스토어 통계)을 `Generation`에 역연결. 승격 로직: 평점+CTR/CVR로 인컴번트를 충분 샘플로 이기면 카테고리 디폴트.

**Stage 3 — 전 카테고리 일반화 (17주+):** DNA 카드 생성을 템플릿화(새 카테고리 = 커스텀 코드가 아닌 가이드 위저드). 네이버 쇼핑 검색광고로 썸네일 A/B(상품 랭킹 리셋 없이 이미지 테스트하는 안전한 방법). 복리 프롬프트 라이브러리 UI.

**계획을 바꾸는 벤치마크/임계:**
- 썸네일 A/B 상승이 일관되게 <3%면 상세 히어로 + 가격/오퍼 프레이밍으로 테스트 예산 이동(더 큰 지렛대).
- DataLab 일일 쿼터(~1,000콜)가 묶이면 카테고리 트렌드 캐시·주간 갱신.
- 그라운디드 생성이 슬롯 유형에서 평점상 비그라운디드를 못 이기면 그 슬롯 디폴트를 비그라운디드로(지연/크레딧 절약).
- Firefly 생성 크레딧 비용이 지배적이면 고볼륨/저위험 슬롯을 더 빠른/싼 Nano Banana 2로, 히어로/브랜드 핵심은 Firefly 네이티브/Nano Banana Pro 예약.
- 모델 선택 규칙: **Firefly 네이티브** = 상업 안전 베이스 사진 + 브랜드 커스텀 모델; **Nano Banana Pro** = 히어로 + 이미지 내 텍스트 + 사실 인포그래픽; **Nano Banana 2** = 고볼륨 카탈로그/변주 작업.

## 7. 주의사항 (Caveats)

- **네이버 정책은 움직이는 표적.** 2024-10-28 썸네일 룰·트리 기반 랭킹은 셀러/대행사 소스와 네이버 공지 기반 현 최선 이해; 네이버는 수수료/정책/노출 기준을 주기적으로 변경. 정책 검증기를 config로 두고 릴리스 전 판매자센터 공지로 재검증.
- **DataLab API 쿼터·top-500:** ~1,000콜/일은 널리 인용되나 원문 스펙 페이지 미확인(리서치 중 접근 불가); 개발자 콘솔에서 검증. top-500 인기검색어는 웹 UI 전용 — API 아닌 UI-보조/sanctioned 발굴 계획.
- **랭킹 가중치 비공개.** 적합도/인기도/신뢰도 세부(예 7일 클릭 윈도)는 대행사 역설계 + 네이버 고수준 공개에서; 공식 공식 아님. 방향성으로만 사용.
- **스크래핑 적법성은 맥락적·진화 중.** hiQ/Meta/Reddit 판례가 엇갈림; 본 문서는 법률 자문 아님. 공개 페이지의 운영자 트리거 캡처(분석 전용) 선호, robots.txt/ToS 존중, 프로덕션 규모 스크래핑 전 자문.
- **모델 능력·가용성 빠르게 변동.** Nano Banana 2/Pro/Firefly Image Model 5 기능·크레딧 정책·Firefly 내장 모델은 2025말/2026초 소스 기준; 일부 프로모션 조건(무제한 생성 윈도 등)은 한시적이었음. 하드 의존 전 모델 라인업·네거티브 프롬프트 지원을 모델별 재확인.
- **전환 벤치마크는 방향성, 목표 아님.** 가구 CVR(Overstock 3.1%/Wayfair 2.9%/Ikea 1.7%), Wayfair/Snap 참여 수치는 제3자 분석/벤더 리서치; 기대치 가늠용.
- **시드 재현성 불완전.** 길고 복잡한 프롬프트일수록 결정성 저하, 일부 비디오/파트너 모델 맥락에서 약함; 모델 버전 간 픽셀 동일 재현 가정 금지.

## 8. 우리 프로젝트(꽃틔움 가든) 반영 포인트

- 기존 6축 무드-카메라 시스템은 **본 엔진의 한 부분(전략 조립 단계의 무드/카메라 룩업)**으로 흡수 — 대체 아님, 위에 얹음.
- 기존 #86(긍정형 제외)이 연구로 재확인 — 전 모델에 하드 제약으로 유지.
- 명화 4컷(cut-1~4)은 "향 노트 섹션 배경" 슬롯에 정확히 해당 — 썸네일/히어로가 아님. 슬롯 매핑이 빠져 미스적용됐던 것.
- 신규 추가 필요: 슬롯/퍼널 맵, 카테고리 DNA 카드(버전드), 결정 테이블 조립기, 학습 튜플 로깅 + CTR/CVR 역연결, 네이버 썸네일 정책 발행전 검증기, 모델 라우팅(Firefly/Nano Banana 2/Pro).
- 정보수집 백본 = DataLab 쇼핑인사이트 API(합법) + 쇼핑검색 API + 운영자 트리거 경쟁 캡처(분석 전용·재게시 금지).
- 전상품 공통(#55)·additive·자연 개입점(#56)·오류는 전사 확장.
