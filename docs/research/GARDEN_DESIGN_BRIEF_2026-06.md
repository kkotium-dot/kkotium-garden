## 0. 작업 채팅의 오류에 대한 정정 [대표 확인]
- Claude Design 채팅의 `image.source.base64 ... 400` 에러들은 **커넥터(MCP) 연결 불안정 때문**이지 디자인 결함이 아니다.
- 즉 정원 일러스트/대시보드 디자인 방향 자체는 유효. 일러스트 에셋도 커넥터 정상화 시 완성 가능.

## 1. 원본 디자인 브리프 (대표가 Claude Design에 준 최초 지시문, 실측 인용)
- 대상: https://kkotium-garden.vercel.app/dashboard — 개발 중 개인용 앱, UI/UX 전체 개선 목적.
- 첨부 스크린샷은 UI 통일 전이며, 전반적 느낌과 특정 컨셉(꽃틔움 가든 아이덴티티)을 녹이는 참고용.
- **핵심 무드 (확정)**: "빈티지하면서도 팝(Pop)한 컬러감, 물결치는 형태(Scallop/Wavy), 감각적인 타이포그래피 배치, 그리고 아날로그 스티커를 붙인 듯한 레트로 감성까지. 이 아름다운 '레트로 팝 & 가든 판타지' 무드를 차분한 B2B 데이터 대시보드에 녹여내는 것이 이번 작업의 핵심."

## 2. 무드 키워드 → 디자인 토큰 함의 [설계 기준]
| 키워드 | 토큰/컴포넌트 함의 |
|---|---|
| **팝(Pop)한 컬러감** | 레드 #E62310 + 핑크 + 정원그린을 생기있게 운용. 무채색 과다 금지. 단 작업 영역은 밀도로 균형. |
| **물결 형태 (Scallop/Wavy)** | 카드 모서리·구분선·헤더에 스캘롭/웨이브 엣지. 단순 직사각형 지양. 컴포넌트 토큰에 scallop variant. |
| **아날로그 스티커 레트로** | 뱃지·아이콘·라벨을 스티커처럼(미세 그림자·약한 기울임·테두리). 평면 플랫 아님. |
| **감각적 타이포 배치** | 영문 손글씨/이탤릭 세리프 라벨(garden, Products) + 한글 Pretendard 혼용. 이미 화면 구현됨. |
| **가든 판타지** | 꼬띠(튤립 마스코트)·분수·정원 은유가 장식이 아닌 컨셉의 축. |

## 3. 리서치 보고서 교정 [중요]
- 기존 UIUX 리서치는 "Stripe/Linear식 차분한 SaaS"를 기본값으로 깔았으나, **원본 의도는 '레트로 팝 & 가든 판타지'를 B2B 데이터에 녹이는 것** — 차분함이 아니라 "감각적이되 일 잘 되는" 균형.
- **2-Zone 균형점을 팝 쪽으로 이동**:
  - 감성 Zone(대시보드): 레트로 팝 가든 판타지 만끽(분수·꼬띠·스캘롭·팝 컬러).
  - 작업 Zone(SEO/아틀리에): 회색 엑셀 금지. **팝 컬러감 + 스캘롭 디테일 + 스티커 뱃지 유지하되 데이터 밀도만 확보.** "생기있지만 일 잘 되는 화면".
- 비유: "조용한 사무실"이 아니라 "감각적인 편집숍인데 일도 잘 되는 공간".

## 4. 1단계 토큰 설계 추가 반영 사항
1. 컴포넌트 토큰에 **스캘롭/웨이브 엣지 variant** 포함(카드·구분선·헤더).
2. 뱃지/아이콘 **스티커 스타일** 토큰(미세 그림자·테두리). 플랫 금지.
3. 팝 컬러 운용: 레드는 여전히 action/critical로 절제하되, 핑크+그린의 채도를 죽이지 말 것(팝 유지).
4. 타이포 토큰에 영문 이탤릭 세리프 라벨 스타일 추가(한글 Pretendard와 혼용 시스템).
5. 작업 Zone에서도 위 디테일 유지 — 무채색화 금지, 밀도로만 차별.

## 5. 변하지 않는 확정 사항 (재확인)
- 꼬띠 = 튤립 마스코트(다람쥐 아님), 공식 안내자.
- 닥스훈트 배송·좀비소(좀비 보관소) = 컨셉 후보(미실현), 라우트 미생성.
- 정원 그린 보조색 공식 추가. 2-Zone 전략.
- 일러스트 에셋은 교체가능 슬롯(커넥터 정상화 후 완성).
- 채팅 기수정 반응형 교훈: word-break:keep-all / min-width 가로스크롤 방지 / 말풍선 min-width.
