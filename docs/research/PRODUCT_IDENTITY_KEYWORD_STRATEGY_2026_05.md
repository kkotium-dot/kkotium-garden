## TL;DR
- **결론 1**: 긴 상품명에서 "상품 정체성"을 추출할 때 단일 대표 키워드(예: "잠옷세트")로 시장을 판정하면 거의 항상 오판이 난다. 정확한 방법은 (1)형태소 분석(Mecab/Komoran 권장)으로 명사 후보군 추출 -> (2)각 명사를 네이버 쇼핑 API에 단독 검색해 1위 상품의 category4(leafCategory)를 회수 -> (3)본 상품이 속한 leafCategoryId와 카테고리가 일치하는 명사 집합을 "정체성 키워드", 그 외(개업선물·집들이·이사 등)는 "용도/시즌/타깃 수식어"로 분리하는 것이다. 이 2단계 카테고리 일치 검증이 네이버 쇼핑의 "필드 연관도 + 카테고리 선호도" 알고리즘과 동일한 논리이며, 매드워드(매드파이프)·아이템스카우트 확장프로그램·퍼센티 등 실무 도구가 모두 이 방식을 채택하고 있다.
- **결론 2**: 새싹 셀러용 "황금 키워드" 판정의 산업 표준 공식은 `경쟁강도 = 상품수 ÷ 월간 검색수`(아이템스카우트·셀러마스터·판다랭크 모두 공식 동일). 그러나 벤더가 발표한 5단계 수치 컷오프는 없으며(아이템스카우트 홈 데모 카드에서 경쟁강도 1.74가 "아주좋음"으로 표시 -> 카테고리 상대화 분류 시사), 커뮤니티 컨센서스는 **경쟁강도 < 0.1 = 황금(strict), < 1.0 = 황금(loose), 1~3 = 좋음, 3~10 = 보통, 10~30 = 나쁨, >50 = 아주나쁨**이다. 새싹 셀러 추천 필터는 **월검색량 1,000~10,000(중형), 상품수 <= 5,000~20,000, 경쟁강도 <= 1.0, 쇼핑성 키워드, 카테고리 일치도 100%** 5중 게이트다.
- **결론 3**: 4개 API를 조합하는 "정확한 기회 지수" 공식 — **검색광고 API(keywordstool)** = 수요(월간 PC/모바일 검색수, 경쟁정도, 클릭률) -> **쇼핑 검색 API(shop.json)** = 공급(total 필드 + 상위 40개 상품 mallName/maker/brand로 경쟁자 강도) -> **데이터랩 쇼핑인사이트 API** = 카테고리·연령·성별 트렌드 및 계절성 -> **커머스 카테고리 조회 API** = leafCategoryId 정합성. 4가지를 결합한 **기회지수 OI = log10(월검색수+1) × (1 / (경쟁강도+0.1)) × 카테고리적합도 × 쇼핑성지수 × 트렌드모멘텀**으로 최종 스코어링하면 단일 키워드 오판 문제가 해소된다.

## 1. 네이버 쇼핑 검색 알고리즘 — 3축(적합도·인기도·신뢰도) + 카테고리 선호도 가중치

네이버 공식 자료에 따르면 노출 순위는 (1)**적합도**(필드 연관도 + 카테고리 선호도), (2)**인기도**(클릭수 7일/판매실적 2·7·30일/리뷰수/최신성, 카테고리별 다른 가중치), (3)**신뢰도**(상품명 SEO 페널티 + 어뷰징 페널티)의 3축으로 결정된다.

가장 중요한 축은 **카테고리 선호도**다. 같은 키워드라도 검색자 다수가 클릭하는 leafCategoryId에 가중치가 부여되므로, 상품을 그 카테고리에 정확히 매칭하지 않으면 노출 자체가 안 된다. 네이버 쇼핑은 매일 2,000만 개 이상 등록되는 상품을 약 5,000개 카테고리에 자체 TensorFlow 모델로 분류한다 — 즉 셀러가 입력한 카테고리를 그대로 신뢰하지 않고 **상품명·이미지·속성에서 자체 추론한 카테고리와 비교**해 가중치를 매긴다.

SEO 페널티 규정: 상품명 50자 이내 권장(최대 100자), 셀러/쇼핑몰명 금지, ()-=[]/& 외 특수문자 금지, "할인/쿠폰/적립" 등 메타데이터 금지, 동의어·중복 키워드 금지.

## 2. 긴 상품명에서 "정체성 키워드" 분리 — 형태소 기반 5단계 알고리즘

예시: "디자인 복 달항아리 도어벨 개업선물 액막이 집들이 이사 결혼 신혼 인테리어"

- **Step 1. 명사 추출**: Mecab/Komoran으로 NNG/NNP만 추출. Mecab과 Khaiii가 가장 정밀, Komoran은 가장 러프. 상품명에는 신조어·복합어가 많아 **사용자 사전 추가 필수**(예: "달항아리", "도어벨" 등록).
- **Step 2. 명사별 카테고리 회귀**: 각 명사를 네이버 쇼핑 검색 API에 단독 검색 -> 응답 items[0].category4 회수.
- **Step 3. 자기 상품 leafCategoryId와 매칭**: "도어벨"(매칭 O), "달항아리"(매칭 △, 도자기), "개업선물"(매칭 X, 선물세트)로 분류. 매칭 O = 본질 키워드, 매칭 X = 용도/시즌/타깃 수식어.
- **Step 4. 본질 키워드 조합 산출**: 2~3 토큰 조합 -> 다시 카테고리 매칭 -> 매칭률 100% 조합이 정체성 키워드 후보군.
- **Step 5. 수식어는 별도 보조 키워드 풀**: "개업선물 도어벨", "집들이선물 문종" 등은 롱테일 보조 키워드로 상품명·태그·상세설명에 배분.

매드파이프 설명: "네이버 쇼핑은 상품명의 각 단어별 카테고리 작업을 해 주어야 노출이 잘 된다. 각 단어를 네이버 쇼핑에 쳐 보았을 때 나오는 카테고리를 전부 일치시켜 주는 것."

## 3. "황금 키워드" 판정 — 공식과 임계값

**핵심 공식(벤더 공식 인용)**
- 아이템스카우트: "(상품수) ÷ (검색수), 낮을수록 경쟁이 낮은 키워드"
- 셀러마스터: "상품수대비 키워드의 월간조회수, 낮을수록 좋음" — 동일
- 판다랭크: "최근 한달 검색량 대비 당일 전체 상품량. 수요 대비 공급이 적을수록 좋음" — 동일

**아이템스카우트 경쟁 종합 지표(가중치 미공개)**
1. 경쟁강도(상품수/검색수) 2. 실거래상품 비율(높을수록 좋음) 3. 묶음상품 비율(낮을수록 좋음) 4. 해외상품 비율(국내셀러는 낮을수록) 5. 1년내 게시 비율(높을수록 신규유리)

**커뮤니티 컨센서스 임계값**

| 등급 | 경쟁강도 | 근거 |
|---|---|---|
| 황금(strict) | < 0.1 | 브런치 정영민 2020 "경쟁률 1% 미만" |
| 황금(loose) | < 1.0 | 키워드마스터/dkcho.me 컨센서스 |
| 좋음 | 1~3 | 아이템스카우트 데모(1.74=아주좋음) |
| 보통 | 3~10 | 토스페이먼츠 "10:1 미만 경쟁력" |
| 나쁨 | 10~30 | 토스페이먼츠 "10:1~30:1" |
| 아주나쁨 | >50 | 토스페이먼츠 "50:1 기준선" |

**키워드 크기**: 소형 <=2,000 / 중형 2,000~10,000 / 대형 >=10,000

**새싹 셀러 추천 필터(블로그 합의)**: 월검색량 1,000~30,000 / 상품수 5,000~20,000 이하 / 경쟁강도 <=10(보수적 <=1.0) / 쇼핑전환 0.5+

**중요 경고**: 아이템스카우트는 5단계 라벨 수치 컷오프를 공개하지 않는다. 1.74가 "아주좋음"인 것은 라벨이 **카테고리 상대값**임을 시사. **진단 엔진은 카테고리별 percentile 라벨링이 정답.**

## 4. 4개 API 조합 — "기회 지수(OI)" 설계

| API | URI | 핵심 필드 | 용도 |
|---|---|---|---|
| 검색광고 API | GET api.searchad.naver.com/keywordstool | monthlyPcQcCnt, monthlyMobileQcCnt, compIdx, plAvgDepth | 수요 + 연관키워드 최대 1,000개 확장 |
| 쇼핑 검색 API | GET openapi.naver.com/v1/search/shop.json | total, items[].mallName/maker/category1~4 | 공급(total) + 카테고리매칭 + 경쟁자분포 |
| 데이터랩 쇼핑인사이트 | POST openapi.naver.com/v1/datalab/shopping/categories | data[].ratio(0~100 상대값), 성별/연령 | 트렌드 모멘텀 + 계절성 + 타깃검증 |
| 커머스 카테고리 조회 | GET api.commerce.naver.com/external/v1/categories | leafCategoryId, 4뎁스 트리 | 카테고리 ID 정합성 |

**인증**: 검색광고 API = HMAC-SHA256(X-Timestamp/X-API-KEY/X-Customer/X-Signature) / 데이터랩·쇼핑검색 = X-Naver-Client-Id/Secret 헤더

**기회 지수(OI) 권장 공식**
```
OI = log10(월간검색수 + 1)              # 수요 (로그 스케일)
   × (1 / (경쟁강도 + 0.1))             # 공급 역수 (0 division 방지)
   × 카테고리적합도(0~1)                # 1위 상품과 leafCategoryId 일치 여부
   × 쇼핑성지수(0~1)                    # 쇼핑성=1, 정보성=0~0.3
   × 트렌드모멘텀(0.5~2.0)              # 데이터랩 3개월 추세(상승=1.5+, 하락=0.7)
```

**보조 가드(필수 부적격)**: 월검색수 <100 자동탈락 / 상품수 >1,000,000 레드오션 / 1위 카테고리 != 본 상품 카테고리 탈락 / 상표권 키워드 탈락

**전략적 역할 판정 로직**

| 역할 | 월검색수 | 경쟁강도 | 트렌드 | 카테고리적합도 |
|---|---|---|---|---|
| 꿀통 | 1,000~10,000 | < 1.0 | 안정/상승 | 100% |
| 니치 | 100~1,000 | < 0.5 | 안정 | 100% |
| 트렌드 | 5,000~100,000 | 1~5 | 강한상승(+30%) | >=80% |
| 레드오션 | > 10,000 | > 10 | 정체/하락 | 변동 |

## 5. 연관 키워드 자동 확장 (롱테일 발굴)

1. **검색광고 API hintKeywords**: 최대 1,000개 연관키워드 + 검색수/클릭률/경쟁정도 회수
2. **자동완성·연관검색어**: ac.search.naver.com (비공식, robots 준수)
3. **상위 40개 상품명 토큰 빈도**: display=100 -> Mecab 분석 -> 2~3회 이상 반복 명사 채택 (1페이지 진입 상품의 실제 보조 키워드)
4. **카테고리 인기검색어(데이터랩)**: leafCategoryId Top 500 -> 코사인 유사도 상위 N
5. **조합 알고리즘**: 본질키워드 × 수식어 Cartesian -> 검색광고 API 재검증 -> 검색수 >=50 채택

정영민 사례: "홍삼"(35만) -> "거래처선물"(8천) -> "거래처명절선물"(4천) -> "거래처대표선물"(11) — 단계마다 상품수 절반 이하로 줄어 새싹도 즉시 1페이지.

## 7. 구현 청사진 (5개 모듈)

- **모듈 A — Identity Extractor**: 형태소분석 -> 명사별 쇼핑 단독검색 -> leafCategory 매칭 -> 본질/수식어 분리 -> 2~3토큰 조합 재검증
- **모듈 B — Opportunity Scorer**: 검색광고+쇼핑검색 결합 -> OI 공식 산출 -> 역할 분류 -> 카테고리 상대 등급
- **모듈 C — Role Classifier**: cat_fit/search_vol/competition/momentum 결정트리 -> 꿀통/니치/트렌드/레드오션
- **모듈 D — 카테고리 상대화 등급기**: 카테고리별 평균 경쟁강도 캐시 -> z-score percentile 라벨
- **모듈 E — 상품명 자동 조립기**: 정체성+롱테일 50자 이내 SEO 가이드 준수 조립

## 8. 단계별 실행 로드맵

**Phase 1 (1~2주, MVP)**: 검색광고+쇼핑검색+커머스카테고리 3개 연동. Mecab+사용자사전. 모듈 A+B 최소구현(카테고리일치+경쟁강도 2지표). 커뮤니티 default 임계값.

**Phase 2 (3~6주, 정밀화)**: 데이터랩 추가(트렌드 모멘텀). 카테고리 percentile 벤치마크 캐시. 쇼핑성/정보성 분류기. 연관키워드 5단계 확장.

**Phase 3 (3개월+, 차별화)**: sentencepiece BPE 자체 토크나이저. 카테고리 자동분류 모델. 매출 추정 모델. 상품명 자동생성기.

**임계값 수정 신호**: 꿀통 판정 키워드 1페이지 진입률 <40% -> 경쟁강도 1.0->0.5 강화 / "부적합" 비율 >50% -> 부분매칭 허용 / API 한도초과 -> 24h 캐시 + hintKeywords 5개 일괄

## 9. 한계 (반드시 인지)

1. **벤더 공식 컷오프 미공개**: 임계값은 커뮤니티 컨센서스. 카테고리 상대화가 더 정확. "default이며 재학습 가능" 명시 필요.
2. **알고리즘 수시 변동**: 분기 단위 업데이트. 빠른배송·저가가 인기도에 강하게 반영. 인기도 가중치는 카테고리별로 다름. 결과는 항상 "추정치".
3. **검색광고 API "< 10" 마스킹**: 월검색수 10건 미만은 문자열 "< 10" 반환. "데이터 부족" 별도 표시 필요.
4. **상품수 인플레이션**: total은 묶음·해외·중고 포함 -> 부풀려짐. 상위 40~100개 mallName 고유 셀러 카운팅 필요.
5. **카테고리 자동매칭 한계**: 복합용도 상품("달항아리 도어벨"이 도어벨 vs 도자기 vs 인테리어소품)은 모호. 이미지 임베딩(CLIP 코사인) 보조 신호 권장.
6. **형태소 분석기 오류**: Mecab도 "달항아리"->["달","항아리"], "도어벨"->["도어","벨"] 오분할. 사용자 사전은 운영 데이터로 자동 갱신 필수(최근 30일 상위노출 신상품 상품명 빈도 >=5 미등록 복합명사 자동후보화).
7. **법적 리스크**: 타 브랜드명·상표권 단어 페널티+분쟁. 상표권 블랙리스트(KIPRIS/퍼센티 약 7만 단어) 운영.
8. **단순 경쟁강도 함정**: "풀필먼트"는 클릭경쟁률 0.03이나 쇼핑상품 아닌 물류서비스. 쇼핑성 분류+카테고리적합도 게이트 통과 후에만 황금 인정.
9. **데이터랩 ratio 상대값 한계**: 절대 검색량 미반환(0~100 ratio만). 트렌드는 "시계열 변화율"로만. 키워드 간 비교는 검색광고 API 검색수가 정답.
10. **롱테일 4단계 효과 검증**: 검색수 11회는 매출 임계점 아래 가능성. "검색수 너무 낮음" 경고(월 50회 미만) 표시해 무의미 롱테일 방지.
