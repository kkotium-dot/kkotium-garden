# 네이버 스마트스토어 키워드 확장 엔진 설계 가이드 (2025-2026)

> 저장일: 2026-06-03 | 출처: Desktop 심층 리서치 | 권위 문서 (키워드 확장 엔진 baseline)
> 트리거: ai-generate route의 키워드 후보가 상품명 파생어로만 제한되는 구조적 한계 발견.
> 목적: 상품명 밖의 실제 시장 검색 키워드까지 끌어와 네이버 쇼핑 검색 노출(SEO)을 극대화.

---

## TL;DR
- **네이버 SearchAd API `/keywordstool`(RelKwdStat)은 seed 키워드 1개만 넣어도 그 자신과 연관 키워드(relKeyword)를 월간 검색량·경쟁도와 함께 한 번에 반환한다. 실측 사례에서 seed "원피스" 1개 단일 호출이 keywordList 길이 정확히 1,200개를 반환했다.** 이것이 "상품명 밖의 실제 시장 키워드"를 끌어오는 가장 강력하고 합법적인 단일 소스이며, 상품명 토큰 조합에만 의존하던 현재 시스템의 핵심 한계를 즉시 해결한다.
- 권장 아키텍처: **seed 추출 -> 다중 소스 확장(SearchAd 연관키워드 + 카테고리 인기검색어 + 자동완성 + LLM) -> 검색량/문서수 검증 -> 의도·경쟁도 스코어링 -> 최종 선정**의 5단계 파이프라인. LLM은 "검증 대상 후보 생성기"로만 쓰고, 실제 채택 여부는 SearchAd 실측 검색량으로만 판정해 환각(없는 키워드 지어내기)을 원천 차단한다.
- 2025년 네이버플러스 스토어 개편으로 **상품명-카테고리 일치(적합도)와 속성/태그 정확성**의 비중이 커졌고, 신뢰도(상품명 SEO 스코어)는 가점이 아닌 **강력한 감점 요소**다. 확장된 키워드는 카테고리 적합성·중복 페널티·구매의도를 함께 평가해 35-45자 상품명과 sellerTags에 전략적으로 배치해야 한다.

## Key Findings

### 1. /keywordstool 연관키워드 자동 확장은 실재한다 (최우선 사실 확인)

"seed 1개를 넣으면 수십~수백 개 연관 키워드를 검색량과 함께 반환하는가?" -> **명확히 그렇다.**

- **동작 방식**: `GET https://api.searchad.naver.com/keywordstool?hintKeywords={seed}&showDetail=1` 호출 시, 응답의 keywordList 배열에 seed 자신과 **연관 키워드들이 함께** 반환된다. "차량용방향제" 한 개를 넣으면 상품명에 없던 "차량용디퓨저", "송풍구디퓨저", "차량선물" 같은 시장 키워드가 검색량과 함께 후보로 들어온다.
- **hintKeywords 개수 제한**: 쉼표로 구분해 **최대 5개**까지 입력 가능.
- **반환 최대 개수**: 단일 호출에서 **최대 약 1,200개**까지 반환 (실측치, 공식 하드캡은 비공개).
- **응답 필드 스키마**(각 keywordList 항목): relKeyword(연관키워드), monthlyPcQcCnt(월간 PC 검색수), monthlyMobileQcCnt(월간 모바일 검색수), monthlyAvePcClkCnt, monthlyAveMobileClkCnt, monthlyAvePcCtr, monthlyAveMobileCtr, plAvgDepth(월평균 노출 광고수), compIdx(경쟁정도: 낮음/중간/높음 3단계). **검색수 10 미만이면 "< 10" 문자열로 반환되므로 정수 변환·치환 처리 필수**(예: '< 10' -> 10).
- **인증**: 헤더 4종 — X-Timestamp(epoch ms), X-API-KEY, X-Customer(CUSTOMER_ID), X-Signature. 서명 = base64(HMAC-SHA256(secretKey, "{timestamp}.{method}.{uri}")).
- **호출 제한(설계상 가장 중요)**: 키워드 도구는 다른 오퍼레이션 대비 호출 속도가 1/5~1/6 수준으로 강하게 제한. IP당 속도도 1/2 수준. 5개는 무난하나 6번째에서 429(Too Many Requests) 흔함. 공식 권고: 429 시 타 서비스 대비 5~6배 긴 sleep 후 재호출. -> **호출 큐 직렬화 + 결과 캐싱 + 지수 백오프 필수.**

> 적용 노트: 현재 이미 /keywordstool을 연동 중이므로 엔드포인트·인증은 유지하고, **호출 입력을 "상품명 파생 후보 15개"에서 "대표 seed 1~5개"로 바꾸고 응답의 연관키워드를 버리지 않고 후보 풀에 흡수**하는 것만으로 후보가 수십 개에서 수백 개로 확장된다.

### 2. 2025-2026 네이버 쇼핑(플러스 스토어) SEO 알고리즘

랭킹 = **적합도 x 인기도 x 신뢰도**의 곱. 2024말~2025 개편으로 **개인화 선호도**가 4번째 축 추가.

- **적합도**: 필드 연관도(상품명/카테고리/브랜드/속성·태그) + 카테고리 선호도. 2025 개편으로 상품명-카테고리 일치 중요도 상승. 브랜드는 상품명보다 브랜드 필드에 정확 매칭이 유리.
- **인기도**: 클릭수·판매실적·리뷰수/평점·찜수·최신성, 카테고리별 가중치 상이.
- **신뢰도**: 상품명 SEO 스코어 + 페널티. **가점이 아니라 감점 요소.** 페널티 트리거: **동일 단어 3~4회 이상 반복**, 혜택/수식 문구, 특수문자, 과도하게 긴 상품명, 동의어/유의어 중복(네이버가 자동 처리하므로 어뷰징 간주).
- **실무 합의**: SEO는 감점 방지, 실순위는 인기도(리뷰·구매·유입)가 좌우. 상품명 키워드 4~8개 권장, 강조어 앞배치. 띄어쓰기 주의("생 삼겹살" != "생삼겹살").

### 3. 키워드 확장 소스 다양화 — 공식 API vs 비공식

| 소스 | 합법성/가용성 | 검색량 | 용도 |
|---|---|---|---|
| SearchAd 연관키워드 (/keywordstool) | 공식 API | 절대값 O | 확장 엔진 핵심 축 |
| 데이터랩 쇼핑인사이트 (POST openapi.naver.com/v1/datalab/shopping/categories) | 공식 API | 상대 비율만 | 카테고리 시즌성/트렌드 |
| 쇼핑인사이트 인기검색어 TOP500 | 반공식(웹 제공, Open API 미제공) | X | 카테고리 시드 확보 |
| 네이버 자동완성 (ac.search.naver.com/nx/ac) | 비공식 | X | 실수요 변형어 |
| 연관검색어 | 비공식 | X | 보조 |
| 커머스 API 카탈로그/상품 조회 | 공식 | X | 카테고리·브랜드 검증 |

핵심: **검색량 절대값은 오직 SearchAd만 제공.** 데이터랩은 상대 비율(0~100)만, 자동완성·인기검색어는 텍스트만. 다른 소스는 "후보 발굴"에만 쓰고 "검증"은 항상 SearchAd로 회귀. 비공식 엔드포인트는 저빈도·소량 사용으로 IP 차단·약관 리스크 관리.

### 4. 카테고리 기반 키워드 확장
- 8자리 leaf 카테고리 기준 인기검색어/속성 키워드 도출이 2025 적합도 강화 흐름의 핵심.
- 커머스 API 상품 조회는 속성을 풍부히 반환 안 함 — 속성 사전은 상품 등록 흐름(originProduct.detailAttribute)에서 확인. 카테고리 속성 키워드는 (1) 카테고리 인기검색어 TOP500 + (2) 상위 노출 경쟁상품의 실제 카테고리·속성 역추적으로 도출.
- 2025 네이버쇼핑 139개 항목 카테고리 개편 -> 카테고리 코드 매핑 테이블 주기 갱신 필요.

### 5. 롱테일/황금키워드 산출
- **황금키워드 공식**: 비율 = 검색량 / 경쟁상품수(또는 문서수). **비율이 낮을수록(1 미만 이상적) 황금 키워드.**
- 실무: SearchAd로 검색량(monthlyPcQcCnt+monthlyMobileQcCnt)+compIdx 확보 + 쇼핑 검색 결과 상품수 결합. **compIdx '낮음' + 검색량 충분 + 경쟁상품 적음**의 교집합이 신규 셀러 진입점. 롱테일(3단어+)은 전환 높고 경쟁 낮아 우선 채택.

### 6. 의도 가중치 고도화
- 검색의도 4분류: 정보형 / 이동형 / 상업조사형(추천·비교·순위) / 거래형(구매·가격·할인). 커머스는 상업조사형·거래형이 전환 핵심.
- 현재 단순 화이트리스트(선물/집들이 x2, 인테리어/디자인 x0.3) -> 고도화:
  - **보조 시그널 사전**으로 동적 가중: "추천/비교/순위/후기"(상업조사), "최저가/할인/구매/주문/선물/세트"(거래) -> 상향. "뜻/방법/원리"(정보) -> 하향.
  - **시즌성**: 데이터랩으로 키워드 월별 추이 조회해 현재 시즌 부합 키워드 일시 가중.
  - compIdx·전환 신호 결합 -> "거래 의도 x 낮은 경쟁" 동시 만족 상위.

### 7. LLM + 검색량 결합 베스트 프랙티스 (환각 차단)
**LLM은 후보 생성기, 진실 판정자는 SearchAd.**
1. seed 추출: 상품명+카테고리명 핵심 명사 1~5개.
2. LLM 확장: "이 상품 살 사람이 검색할 키워드" broad 생성.
3. **SearchAd 검증 게이트**: LLM 후보 + 연관키워드를 SearchAd 조회. **실측 검색량 없거나 "< 10"이면 폐기** -> 가짜 키워드 자동 제거.
4. 재정렬: 생존 키워드를 (검색량 x 의도가중 x 황금비율 x 카테고리적합도)로 스코어.
5. 배치: 상위 4~8개를 35-45자 상품명에 중복 없이, 나머지는 sellerTags·속성으로 분산.

## 권장 키워드 확장 엔진 아키텍처 (단계별)

**Stage 0 — Seed 추출**: productName+카테고리 -> seed 1~5개. 기존 토큰화·2-gram은 seed 보강용으로만 유지(최종 후보 유일 소스 아님).

**Stage 1 — 다중 소스 확장(Fan-out)**: (A) SearchAd /keywordstool seed 콤마 5개 이하 호출 -> 수백 연관키워드+검색량/compIdx 일괄 (메인). (B) 데이터랩 카테고리 트렌드/시즌. (C 선택, 저빈도) 자동완성·카테고리 인기검색어. (D 선택) LLM broad. -> dedupe 병합.

**Stage 2 — 검색량/경쟁도 검증**: candidate pool 전체 SearchAd 검색량 정규화("< 10" 처리). 0/극저 폐기. 쇼핑 상품수 결합해 황금비율 계산. **LLM 환각·노이즈 거르는 게이트.**

**Stage 3 — 의도·경쟁도 스코어링**: score = log(검색량) x 의도가중(보조시그널+시즌) x (1/경쟁비율) x 카테고리적합도. compIdx 높음 감점, 낮음 가점.

**Stage 4 — 최종 선정·배치**: 상위 키워드를 (1) 35-45자 상품명(동일단어 3회+ 금지, 강조어 앞) (2) sellerTags(사전등록어만) (3) 속성으로 분산. 카테고리 적합도 낮은 키워드 배제.

**Stage 5 — Rate-limit·캐시 인프라**: SearchAd 호출 큐+토큰버킷 직렬화, 키워드 단위 캐싱(TTL 수일), 429 시 5~6배 지수 백오프. 데이터랩/커머스 API 별도 큐.

## Recommendations
1. **즉시**: /keywordstool 입력을 "상품명 파생 15개"에서 **"대표 seed 1~5개"**로 전환 + 응답 연관키워드 흡수. 검색량 0/"< 10" 폐기만으로 누락 키워드(차량용디퓨저 등) 즉시 후보화. **벤치마크: seed당 유효 후보 15개 -> 100개+ 증가.**
2. **단기**: 카테고리 인기검색어·자동완성 시드 보강, 황금비율(검색량/경쟁상품수) 재정렬. **벤치마크: 채택 키워드 compIdx '낮음' 비중 상승.**
3. **중기**: LLM 확장 후보 SearchAd 검증 게이트 + 의도 동적 스코어(보조시그널+시즌). **벤치마크: LLM 후보 폐기율 모니터링.**
4. **상시**: Rate-limit 큐·캐시·백오프 출시 전 필수(미구현 시 429로 파이프라인 붕괴). 카테고리 코드 테이블 분기 갱신.

**전략 전환 임계치**: 429가 큐로도 감당 안 되면(대량 처리) CUSTOMER_ID/IP 분리 또는 야간 배치로 사전 키워드 풀 구축 -> 실시간 호출을 캐시 조회로 대체.

## Caveats
- 데이터랩은 절대 검색량 미제공(상대 비율만) — 검증·랭킹 단독 사용 불가, SearchAd 병행.
- 자동완성·인기검색어 TOP500은 비공식/반공식 — 약관·IP 차단 리스크, 저빈도·폴백 설계.
- 단일 호출 ~1,200개는 실측치(공식 하드캡 아님) — 페이지네이션·캡 변경 대비.
- 여러 hintKeywords(최대 5개) 동시 입력 결과가 합집합/합산인지 공식 명시 미확인 — 정밀 검증 시 소규모 A/B 권장.
- SEO 가이드 다수는 네이버 공식 정리 실무 블로그(아이보스·아이템스쿨·윈들리) 기반. 3축(적합도·인기도·신뢰도)·페널티는 공식 근거, 구체 가중치는 비공개·수시 변경.
- 커머스 API 카테고리 속성은 상품 등록 흐름에서만 풍부, 조회 API는 제한적.
