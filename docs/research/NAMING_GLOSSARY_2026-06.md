> 코드/DB/AI 프롬프트의 내부 코드 값은 절대 변경 금지 — 표시 레이어에서만 한글로 변환.

## 1. 절대 순화 원칙
- ★내부 코드 값(`skeletonId="S6"`, `grade="L2"`, persona/emotionalTone 등)은 그대로 — 로직·DB·AI 프롬프트 모두 raw 리터럴을 읽는다.
- ★표시 레이어에서만 한글로 변환. 매핑에 없는 값은 raw 그대로 표시(비파괴 fallback).
- ★점수에는 척도 명시(0~100 등) + 한 줄 보조 설명(툴팁) 부착.
- ★정원 은유(씨앗심기·화단·온실)는 가볍게만 사용. 진단 결과 등 정확성 필요 영역에는 은유 금지.

## 2. 노출 위치 (현재 확인됨)

## 3. 매핑 표 (★단일 진실)

### 3-1. 골격(상세페이지 구성안)
내부 코드: `SkeletonId = 'S1' | 'S2' | ... | 'S12'`

| 내부 값 | 라벨 | 표시 값 | 비고 |
|---|---|---|---|
| `S1`~`S12` | 상세페이지 구성안 | `1번`~`12번` | 골격/스켈레톤 단어 노출 금지. 1~12번 척도 |

### 3-2. 처리 등급
내부 코드: `Grade = 'L1' | 'L2' | 'L3' | 'L4'`

| 내부 값 | 라벨 | 표시 값 | 의미 |
|---|---|---|---|
| `L1` | 이미지 가공 단계 | `1단계 (자동)` | AI가 자동 완성 |
| `L2` | 이미지 가공 단계 | `2단계 (보통)` | 자동 + 가벼운 검토 |
| `L3` | 이미지 가공 단계 | `3단계 (검토)` | 검토 후 자동화 |
| `L4` | 이미지 가공 단계 | `4단계 (디자이너)` | 디자이너 손길 필요 |

### 3-3. 타깃 연령 (persona)
내부 코드: `Persona = '20s' | '30-40s' | 'senior' | 'kidsmom'`

| 내부 값 | 라벨 | 표시 값 |
|---|---|---|
| `20s` | 타깃 연령 | `20대` |
| `30-40s` | 타깃 연령 | `30·40대` |
| `senior` | 타깃 연령 | `시니어` |
| `kidsmom` | 타깃 연령 | `아이맘` |

### 3-4. 말투 (emotionalTone)
내부 코드: `EmotionalTone = 'friendly' | 'professional' | 'sensory' | 'trust'`

| 내부 값 | 라벨 | 표시 값 |
|---|---|---|
| `friendly` | 말투 | `다정함` |
| `professional` | 말투 | `전문적` |
| `sensory` | 말투 | `감각적` |
| `trust` | 말투 | `신뢰감` |

### 3-5. 점수 (척도 명시)

| 내부 필드 | 라벨 | 표시 형식 | 툴팁 |
|---|---|---|---|
| `inferenceConfidence` (0~100) | AI 추천 정확도 | `56 / 100` | AI가 이 분석을 얼마나 확신하는지 0~100점으로 표시해요. |
| `qualityScore` (0~100) | 사진 품질 점수 | `56 / 100` | 원본 사진의 해상도·선명도·여백을 0~100점으로 평가해요. |

## 6. 데이터/로직 회귀 0 원칙
- DB에 저장되는 `Diagnosis.skeletonId` / `Diagnosis.grade` / `Diagnosis.conceptTone` 등은 raw 리터럴 그대로 유지.
- `gradeProduct()` / `decideGrade()` / `matchSkeleton()` / `inferConceptTone()` 등 분기 로직 입력·출력 불변.
- 신규 추가는 표시 레이어(formatter)에서만 발생.
- 매핑에 없는 신규 값은 raw 그대로 표시되어 절대 깨지지 않음(graceful fallback).
