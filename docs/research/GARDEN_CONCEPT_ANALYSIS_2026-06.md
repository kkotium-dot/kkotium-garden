# 꽃틔움 가든 정원 컨셉 디자인 — Claude Design 작업물 분석 부록

> 목적: 1단계(토큰+공통셸) 설계를 현재 진행 중인 정원 컨셉과 정렬시키기 위한 실측 기록

## 1. 마스코트 — 꼬띠 (튤립 캐릭터) [확정]
- **꼬띠 = 튤립 마스코트.** 다람쥐 아님. 반드시 튤립으로 표기.
- 역할: 앱의 공식 안내자 / 축하 화자 / 툴팁·빈상태 화자. 대표가 명시한 "중요한 마스코트".
- 말투 예시(실측): "꼬띠: 튤립향 디퓨저, 또 1등 했어요! 화단을 한 번 더 가꿔볼까요?", "오늘 매출이 어제보다 17% 올랐어요! 분수가 신나서 물을 더 뿜고 있네요"
- 정체성 일관성: 정원 컨셉의 마스코트가 정원의 꽃(튤립)이라는 점에서 브랜딩 완결성 높음.

## 2. 정원 은유 ↔ 기능 매핑 [실측 — 구조는 신뢰, 일부 미실현]
| 정원 라벨 | 영문 | 기능 | 상태 |
|---|---|---|---|
| 가든 홈 | Garden | 대시보드 | 동작 |
| 화단 | Products | 상품 관리 | 동작 |
| 벌통 | Orders | 주문 | 동작 |
| 버섯밭 | Reviews | 리뷰 | 동작 |
| 온실 | Stock | 재고 | 동작 |
| 우편함 | Inquiries | 문의 | 동작 |
| 가든 일지 | Journal | 일지/로그 | 동작 |
| **닥스훈트 배송** | Shipping | 배송 | **컨셉 후보(미실현)** — Claude Design 실험. 배송 추적 기능 붙일 때 네이밍 재고. |
| **좀비소(좀비 보관소)** | Returns | 반품 보관 | **컨셉 후보(미실현)** — 반품 관리 기능 붙일 때 네이밍 재고. |

→ 1단계 사이드바 토큰에는 **현재 동작 메뉴만** 포함. 닥스훈트/좀비소는 i18n에 "컨셉 후보" 주석으로만 보존, 라우트 미생성.

## 3. 정원 컨셉의 압권 — 매출 분수 시각화 [참고]
- "오늘 흘러든 물 W4,287,500" — 매출을 정원 분수에서 솟는 물로 시각화. 어제 대비 +17.8%.
- 감성(물/생명력)과 효율(정확한 숫자)을 동시 표현. 2-Zone 전략의 감성 Zone 모범 사례.
- "오늘의 목표 달성률 71%" 진행 링이 분수를 감쌈 = 게이지 + 은유 결합.

## 4. 작업 채팅 내역에서 얻은 교훈 [1단계 필수 반영]
좌측 작업 채팅 확인 결과:
- **현재 상태**: `image.source.base64 ... 400 (invalid_request_error)` 에러 다수 = 대시보드 일러스트(분수/튤립/정원 요소) 이미지 생성 반복 실패 중. 즉 **정원 대시보드는 발전 중 시안이며, 개별 일러스트 에셋은 미확정.**
- **이미 수정된 반응형 이슈(우리 앱도 동일 적용 필요)**:
  - 한글 줄바꿈 깨짐 → `word-break: keep-all` (한글 어절 단위 끊기)
  - 좁은 뷰포트 가로 스크롤 → 앱 `min-width: 1180px` + grid `minmax(0, 1fr)`
  - 꼬띠 말풍선 세로 쌓임 → 말풍선 `min-width: 200px`
- **Claude Design 제안 향후 인터랙션**: 온보딩 투어(꼬띠가 닥스훈트 타고 안내), 매출 목표 달성 시 분수 꽃 폭죽 이펙트, 페이지별 꼬띠 코스튬.

### 1단계 설계 보강 (채팅 교훈 반영)
1. **한글 타이포 기본 규칙을 토큰에 박기**: `word-break: keep-all` 전 컴포넌트 상속. 한글 가독성 기본값.
2. **공통 셸에 반응형 가드 내장**: 최소폭·grid minmax 보호로 좁은 화면 가로 스크롤 방지.
3. **일러스트는 "에셋 슬롯"으로 분리**: 토큰은 색·간격·구조만 확정. 정원 일러스트는 교체 가능한 슬롯으로 두어, Claude Design이 에셋 확정하면 끼워넣기.

## 5. 추출 디자인 토큰 (정원 컨셉 실측값)
- 바탕: 크림/아이보리 #FAF8F3
- 브랜드 레드: #E62310 (로고/분수 링/상단 상태 바)
- 핑크: 카드·말풍선·뱃지 표면 (#FFCCEA/#FFE0EC/#fff0f5 계열)
- 정원 그린: 자연 그린 (네비 선택 상태/월간목표 카드) — 보조색 공식 추가 [승인됨]
- 폰트: 한글 Pretendard + 영문 손글씨풍 이탤릭 라벨(garden/Products 등)

## 6. 2-Zone 전략 [승인됨]
- **감성 Zone** (가든홈/빈상태/완료축하): 정원 풀 적용 — 분수·꼬띠·일러스트·정원 은유.
- **작업 Zone** (씨앗심기 SEO/아틀리에): 절제 — 정원은 색·아이콘·라벨에만, 데이터 영역 고밀도 유지.
- 같은 토큰 공유하되 작업 Zone은 일러스트 빼고 정보 밀도 ↑.

## 7. 역할 분담 [확정]
- **Claude Design** (브라우저): 정원 비주얼·대시보드·일러스트 에셋 발전.
- **Desktop + Claude Code**: 그 디자인 언어를 실제 Next.js 앱의 디자인 토큰 시스템 + 공통 셸로 코드화. 전 화면 상속 구조 정착.
