## 1. 최신 웹 작업툴 IA·레이아웃 패턴 (2024–2026)

**지배적 공식: "최소 상시 UI + 맥락형 속성 패널"**
모든 선도 브라우저 작업툴이 ① 접이식 좌측 내비/에셋 패널 ② 시각적 주인공인 중앙 캔버스 ③ 선택·단계에 따라 내용이 바뀌는 우측 패널 — 이 구조로 수렴했습니다.

| 툴 | 핵심 구조 | 온실 아틀리에 시사점 |
|---|---|---|
| **Figma (UI3, 2024~)** | 도구를 **하단 플로팅 툴바**로 이동해 캔버스 공간 극대화. 우측 속성 패널은 **선택된 객체에만** 컨트롤 노출. `Shift+\`로 양쪽 패널 접기 | 우측 컬럼은 "볼 대상이 있을 때만" 뜨는 인스펙터로 |
| **Canva (2024 글로우업)** | 고정 툴바 폐지 → **맥락형 퀵액션 툴바**(이미지 선택 시 자르기·필터, 텍스트 선택 시 폰트·크기). "More"로 점진 공개. 목표가 명시적으로 "집중 강화·과부하 감소" | 단계 안내·온보딩 명료성의 레퍼런스 |
| **Adobe Express** | 좌측 탭형 진입점, 캔버스 상단에 문서단위 액션, **선택 요소별로 메뉴 동적 변경**. ⚠️ "요소 선택 시 패널이 떠서 페이지가 밀린다"는 불만 존재 | **선택 시 레이아웃 밀림(layout-shift) 금지** |
| **Framer ($20억 가치, 25.8월)** | 좌측 레이어·우측 속성·중앙 캔버스. **우측은 선택 요소별 속성 변경**. 하단 캔버스 툴바 | 디자인 우선 툴의 현 표준 IA |
| **Linear** | 캔버스 툴은 아니지만 **미니멀·고요한 밀도**의 정석. `Cmd+K` 명령 팔레트 중심, 차트는 별도 탭 뒤로 숨김 | "프리미엄·즉시 가독" 감성의 모델 |
| **Miro / tldraw** | 좌측 상시 생성 툴바 + **선택 객체 위에 뜨는 플로팅 맥락 툴바**. 우측 상시 속성 패널 없음 | 가벼운 캔버스 요소엔 플로팅 미니툴바도 대안. 단, SEO처럼 데이터 풍부한 패널은 도킹형이 깔끔 |

## 2. 인지 과부하 감소 / 점진 공개 (Progressive Disclosure)

> 대표님이 느끼신 "어디서부터 시작할지 모르겠다"는 혼란의 **직접 해법**.

- **점진 공개 3종**: ① 단계형(스테퍼) ② 조건형(요청 시 노출) ③ 맥락형(현재 작업 기준 노출). 온실 아틀리에는 셋 다 적용 — 스테퍼로 단계형, Firefly 프롬프트·시장DNA를 기본 접힘으로 조건형, 우측 인스펙터로 맥락형.
- **"하나의 명확한 다음 행동" 원칙**: 각 단계의 빈 화면(empty state)은 단 하나의 주요 CTA를 제시해야 함. (Dropbox: 중앙 업로드 영역 하나, Notion: "첫 페이지를 만들어보세요", Slack: 3개 시작 프롬프트)
  - 온실 아틀리에 예: 1단계 빈 화면 = "대표 썸네일 생성하기" 버튼 하나만 크게.
- **항목별 액션 밀도(=에셋 레일 핵심 수정)**:
  - NN/G: 케밥(⋮)은 **부차적** 액션용. 핵심 액션만 호버로 노출. 케밥엔 항상 라벨/툴팁.
  - PatternFly: "액션 2개 이하면 케밥 쓰지 말 것" / "공간 제약으로 다 못 보일 때 케밥" — **217px 좁은 레일이 정확히 그 경우**.
  - **권장 패턴**: 평소 = 아이콘 0개(깔끔한 썸네일) → 호버/포커스 = 주요버튼 1개(★ 대표지정) + 케밥(⋮: 추가이미지·아카이브·삭제) → 다중선택 = 하단 일괄 액션 툴바.
- **여백·위계·집중 모드**: 2025–2026 대시보드 합의 = "무지개 대시보드 은퇴". 중립 회색/흰색 기반 + 강조색 1~2개(브랜드색 + 의미색 녹색=성공/적색=경고). 화면당 핵심 요소 5~9개. 가장 중요한 요소는 좌상단. (Linear·Stripe가 정석)

## 3. 단계형(스테퍼) 위저드 UX

- **4단계 유지가 적정** (이상치 3~6단계). 썸네일→상세→SEO→발행 = 잘 잡힌 크기.
- **비선형 점프 허용 + 단계별 상태 표시 필수**: 완료(체크)·현재(강조)·다음·오류/미완(예: 정책 게이트 실패) 4상태를 명시. "상태 없는 스테퍼는 헷갈리는 메뉴일 뿐."
- 데스크톱은 **상단 수평 스테퍼**, 모바일은 점/카운터로 축약하거나 하단 탭으로 전환.
- **발행 검토 = 마지막 선형 게이트**: 발행만 차단(검증 통과 전), 앞 단계는 자유 이동. 단계별 인라인 검증 배지(예: SEO 단계 "키워드 3/5 매칭").
- **단계 옆 실시간 미리보기**: 캔버스가 결과를 실시간으로 보여주는 구조(=온실 아틀리에의 캔버스 중심 의도와 일치).

## 4. 데스크톱 멀티패널 → 모바일 단일컬럼 반응형 축소

- **분기 전략**: CSS `grid-template-areas`를 브레이크포인트별로 재배치. 데스크톱 3컬럼 → 태블릿 2컬럼(한쪽을 드로어로) → 모바일 1컬럼.
- **모바일 단계 전환 = 하단 탭바(4단계)**: 3~5개가 최적, 터치 타깃 44~48px. 하단 배치 이유 = 엄지존. (NN/G 인용: 자연 하단존 탭 정확도 96% vs 상단 스트레치존 61%, 상호작용 속도 267% 빠름)
- **오프캔버스 드로어**: 도구함·에셋 브라우저·SEO 관제탑은 모바일에서 스와이프 드로어로. 캔버스/미리보기가 기본 화면.
- **편집 숨기고 승인만 노출**: 모바일은 읽기전용 미리보기 + 소수 승인 액션(썸네일 승인·상세 승인·발행 검토 진입). 하단 고정 CTA로 주요 승인/발행.
- **터치 타깃**: 최소 44×44px(Apple HIG 44pt·Material 48dp), 하단 내비 3~5개.

## 5. 에셋/미디어 브라우저 UX

- **그리드 vs 리스트**: 시각 탐색은 그리드(썸네일), 순서변경·일괄관리는 리스트. 좁은 217px 레일은 **1열 또는 2열 썸네일 그리드 + 그룹 헤더**.
- **단계별 그룹핑**: 스테퍼가 이미 "현재 단계 자산" 라벨을 동기화 중 → 이를 정식화. 에셋 레일은 **기본적으로 현재 단계 자산만 필터**, "전체 보기" 토글 제공.
- **항목 액션**: 호버 툴바 + 케밥 + 다중선택 일괄 툴바 (2번 항목과 동일).
- **가상 스크롤(virtualized)**: 대량 자산 대비. **그리고 col0에 자체 내부 스크롤 부여** (현재 2220px까지 늘어나며 스크롤 없는 것이 주요 결함 — 각 패널이 고정 뷰포트 안에서 독립 스크롤되어야 함, Figma·Linear 방식).
- **상단 검색/필터**: 작은 검색창 + 카테고리 칩, 즉각 시각 피드백.

## 6. 맥락형 속성/인스펙터 패널

- **검색 생장 관제탑(SEO 관제탑)**은 상시 3번째 컬럼을 그만두고 **맥락형 우측 인스펙터**로: SEO 부스터 단계에서 전면 확대, 그 외 단계에서는 접힘/최소화. (Figma 인스펙터·Canva 맥락 툴바 모델)
- 단계별 우측 패널 변신: 1·2단계 = 가볍게(썸네일 품질/진단 요약·섹션 속성) → 3단계 = SEO 관제탑 전면 → 4단계 = 정책 게이트 체크리스트.
- 이로써 SEO 관제탑이 "상시 잡동사니"에서 "호출되는 맥락"으로 전환 — 과부하 불만 직접 해소.

## 7. 이커머스 / 네이버 스마트스토어 SEO 워크플로우 트렌드 (2025–2026)

- **2026 알고리즘 = AI 기반 의도 + 콘텐츠 품질 중심으로 이동.** 구식 전술(상품명 키워드 떡칠, 태그 10개 채우기, 느슨한 카테고리)은 더 이상 안 통함. 가중 요소:
  - **카테고리명 일치 = 1순위 신호** ("카테고리 틀리면 나머지 다 고쳐도 노출 안 됨")
  - 구조화·정규화된 옵션명 (예: "블랙 / L / 귀마개 포함", "고급형 블랙 대형 추천" 같은 잡음 금지)
  - 상세페이지 **체류시간·스크롤 깊이**, 능동신호(장바구니·찜)
  - **리뷰 콘텐츠**(구체적 사용맥락) > 리뷰 개수
- **적합도·인기도·신뢰도 3축** 유지. 상품명 SEO는 **감점 축** — 네이버 상품명 가이드 위반 시 랭킹 페널티. 상품명은 ~50자 내외, 중복 키워드·특수문자 금지, 구체적(롱테일) 키워드로 경쟁도↓.
- **필드별 적합도 가중**: 키워드마다 가중 필드가 다름(예: "나이키"는 상품명보다 브랜드 필드가 더 중요). SEO 단계는 "이 키워드를 어느 필드에 넣어야 하는지" 안내해야 함.
- **국내 셀러툴**(아이템스카우트·판다랭크·사장님닷컴·셀러라이프·엠디몽·톡키AI): 키워드 분석 → 아이템 발굴 → 순위 추적 → 추천 상품명/카테고리/태그 순으로 구성. 신형 AI툴은 상품명 생성 + SEO 진단 + 카테고리 매칭 + 이미지/상세 점검을 단일 "AI 매출비서" 플로우로 묶음 = **온실 아틀리에가 만들고 있는 통합 파이프라인과 동일 방향**.
- **AI 상품 이미지 주류화**: "2026 고품질 AI 이미지는 실사 대비 전환율 차이 거의 없음." Firefly/AI 이미지 기능 정당성 확인. 단, 썸네일은 고해상도 + 상품 정직 표현 필수(네이버도 선명한 고해상 썸네일 권장).
- **SEO 부스터 단계가 보여줄 것**: 키워드 목록만이 아니라 → 카테고리 일치 상태, 상품명 SEO 점수/페널티 체크, 구조화 옵션·속성 완성도, 태그 잡음 제거, 상세 체류시간·콘텐츠 품질 넛지.

## 8. 종합 실행 권고

### 레이아웃 모델: 3컬럼(1087px) → **2컬럼 코어 + 접이식 맥락형 우측 인스펙터**

| 영역 | 폭 | 동작 |
|---|---|---|
| **좌측 레일** | ~240–280px, 접이식 | 도구함 + 에셋 브라우저 **통합**. 현재 단계 자산 필터. **자체 내부 스크롤**. 아이콘 레일로 축소 가능(Figma 방식) |
| **중앙 캔버스** | 유동(주인공) | 가장 지배적 영역, 넉넉한 여백, **현재 단계 섹션만** |
| **우측 인스펙터** | ~300–340px, 맥락형·접이식 | 3단계=SEO 관제탑, 4단계=정책 게이트, 1·2단계=가볍게/접힘. 토글 가능 |

### 4단계 기능 배치도

| 단계 | 들어갈 기능 | 우측 인스펙터 |
|---|---|---|
| **1 썸네일 랩** | 대표+썸네일 4변형, 무드 카메라, **Firefly 프롬프트(기본 접힘 "고급")**, AI 진단(생성 썸네일에 인라인 결과로, 상시 섹션 X) | 가볍게(썸네일 품질/진단 요약) |
| **2 상세 캔버스** | 5섹션 상세 조립, **9슬롯 퍼널 보드 이곳으로 이전**, 시장 DNA(접힌 참조로 이곳 또는 3단계) | 섹션/레이아웃 속성 |
| **3 SEO 부스터** | SEO 관제탑이 우측 인스펙터 전면. 키워드 매칭·카테고리 일치 상태·상품명 SEO 점수·구조화 옵션/태그·필드 적합도 힌트 | SEO 관제탑 전면 |
| **4 발행 검토** | 정책 게이트 체크리스트, 이미지 저장/발행 고정 CTA, AI 진단 최종 프리플라이트 | 정책 게이트 체크리스트 |

- **에셋 브라우저**: 좌측 상시 레일, 단계 필터.
- **SEO 관제탑**: 더 이상 상시 컬럼 아님 → 우측 인스펙터, 3단계에서 정점.

### 에셋 액션 밀도 수정 (구체 권고)
평소 = 깔끔 썸네일 아이콘 0개 → 호버/포커스 = 오버레이 미니툴바(★ 대표지정 1개 + ⋮ 케밥: 추가이미지·아카이브·삭제) → 다중선택 = 상/하단 일괄 액션 툴바. 호버 시 선택 체크박스 노출.

### 모바일 단일컬럼 패턴
1컬럼 + **하단 탭바=4단계**. 각 단계 = 읽기전용 미리보기 + 승인 액션. 도구함·에셋·SEO 관제탑은 스와이프 드로어. 단계별 하단 고정 주요 CTA. 편집 비활성 + "데스크톱에서 편집" 안내.

### 시각 위계·타이포 (프리미엄 감성)
- 중립 기반(흰색/연회색) + **강조색 최대 1~2개**(브랜드색 + 의미색 녹/적). 무지개 크롬 은퇴.
- 명확한 타입 스케일, 8pt 간격 그리드, 두꺼운 구분선보다 미묘한 보더(Linear·Stripe).
- 각 영역 좌상단에 가장 중요 요소, 화면당 핵심 5~9개.
- **Cmd/Ctrl+K 명령 팔레트**(단계·에셋·액션 횡단 이동, Linear 모델) — 매일 쓰는 툴에 고효율.
- 피드백 마이크로인터랙션(호버 음영, 패널 전환 150~250ms), **선택 시 레이아웃 밀림 금지**(Adobe Express 반면교사).
- 단계별 빈 화면 = 단 하나의 주요 CTA + 첫 실행용 가벼운 온보딩 체크리스트.

## 9. 3단계 실행 로드맵 (우선순위)

| 단계 | 작업 | 통과 기준 |
|---|---|---|
| **Stage 1 — 구조(최우선)** | ① 각 패널 고정 뷰포트 내 자체 스크롤(2220px col0 결함 즉시 해소) ② 에셋 레일 깔끔 썸네일 + 호버툴바·케밥·다중선택(상시 아이콘 100~160개 제거) ③ 스테퍼가 캔버스뿐 아니라 **우측 패널·에셋 필터까지** 진짜로 게이팅 | 어느 단계든 "다음 행동"을 스크롤 없이 5초 내 찾음 |
| **Stage 2 — 레이아웃** | ④ 2컬럼 코어 + 접이식 맥락형 우측 인스펙터 / SEO 관제탑→인스펙터(3단계 정점), 퍼널보드→2단계, 시장DNA→2·3단계, AI진단→1단계 인라인+4단계 프리플라이트 ⑤ 단계별 빈 화면 + 단일 CTA | 한 단계가 핵심요소 5~9개 초과 노출 안 함 |
| **Stage 3 — 모바일+폴리시** | ⑥ grid-template-areas 반응형 축소 → 1컬럼 + 하단 4단계 탭, 읽기전용/승인, 드로어 패널, 하단 고정 CTA ⑦ 중립+1~2강조 시각 시스템, 8pt 그리드, 타입 스케일, Cmd+K 팔레트 | 모바일은 편집 컨트롤 0 노출로 상품 검토·승인 완결 / 데스크톱은 Linear·Canva급 여백·가독 |

### 계획을 바꿔야 할 임계 조건
- SEO 관제탑 데이터가 대부분 세션에서 캔버스 편집과 **동시에** 필요하다면(3단계만 아니라) → 단계종속 인스펙터 대신 "상시-접이식 우측 컬럼"으로 유지.
- 자산이 상시 수백 개 초과 → 시각 폴리시보다 가상 스크롤·서버 검색 우선.
- 비선형 점프를 거의 안 쓴다는 데이터 → 더 유도적인 선형 플로우 + 큰 "다음 단계" 버튼.

## 10. 주의·한계 (검증 정직성)

- Adobe Express 패널 명칭·Canva 툴바 동작 등 일부 세부는 벤더 도움말·3자 리뷰 기반이라 잦은 UI 업데이트로 탭 명칭이 바뀔 수 있음. 단, **패턴 자체(맥락형 속성·점진 공개)는 다출처로 안정 확인**.
- 일부 출처는 디자인 에이전시 블로그·SaaS 벤더 콘텐츠(홍보성). 핵심 권고는 NN/G·Material·Figma/Canva 공식 문서·다출처 수렴 주장 우선 채택.
- **네이버 "2026 AI 알고리즘" 주장은 셀러툴 블로그·커뮤니티 기반**(공식 문서 아님). 방향(카테고리 일치·콘텐츠 품질·체류시간·구조화 옵션)은 다수 독립 출처에서 일관되나 **구체적 가중치는 비공식 = 합의된 추정**으로 취급. "아이템스카우트 셀러 40~50% 점유" 같은 수치는 미검증 → 사실로 인용 금지.
- 본 문서는 UX/IA 리서치이며 코드 명세가 아님. Next.js/Tailwind 구현(grid-template-areas·container queries·가상 리스트)은 패턴에서 직접 도출되나 본 리서치 범위 밖.
