## 2. 디스코드 알림 업그레이드 (운영자: "크롤링 정보만·실익 없음")
- 진단: 현재 알림이 **소싱(크롤링 상품) 중심** → 운영 실익 낮음. 운영자 지적대로 등록/연동 상품이 늘고 **운영 이벤트**에 연결돼야 유용.
- 제안 — **운영 파이프라인 연동 알림**(신규 앱 기능에 맞춤):
  | 채널/알림 | 트리거 | 실익 |
  |---|---|---|
  | 발행 준비 완료 | getPublishReadiness ready=true 전환 | "OO 발행 가능" 즉시 인지 |
  | 부활 후보 발견 | revival-score grade S/A 신규 | 되살릴 상품 알림 |
  | 주문/배송 이벤트 | 신규 주문·발송·배송완료(배송추적 연동) | 주문 누락 방지·CS |
  | 재고 품절/저재고 | stock-check 품절 전환 | 기회손실·대체 트리거 |
  | 마진 경고 | 마진 임계 이하 | 손실 상품 조기 발견 |
  | 좀비 경고 | product-lifecycle ZOMBIE 전환 | 정리/부활 결정 |
- 원칙: 소싱 알림은 유지하되 **운영 이벤트 알림을 추가**해 "실제 행동 유발". 각 알림에 꼬띠 한마디(세계관)+바로가기 링크(#56 개입점).
- 연동상품 유무: 주문/배송/리뷰 알림은 **실제 판매·연동 상품이 있어야 실데이터**. 지금은 명화(등록)만 → 연동/발행 늘면 가치↑. 단 발행준비·부활·재고·마진 알림은 지금도 유효.

## 3. 꼬띠 추천 상품 고도화 (황금/니치/시즌)
- 현재: daily-slots(꼬띠 추천 라벨·점수) 기반. 운영자: 더 디테일·네이버 최적화·황금/니치/시즌 센스.
- 제안 — 추천을 **유형 분류 + 근거**로:
  | 유형 | 정의(신호) | 데이터 |
  |---|---|---|
  | 황금상품 | 검색 상승세 高 + 마진 高 + 경쟁 低 | category-score(SEO×ROI·#249) + 트렌드 |
  | 니치상품 | 경쟁 低 + 안정 수요(틈새) | 트렌드 + 경쟁 근사 |
  | 시즌상품 | 지금 시즌 급상승 | 트렌드 캐시 시즌성 |
- 각 추천에 "왜 이 상품인가"(검색 상승·마진·경쟁) 셀러 언어로(#233). category-score(#249) 재사용 → 일관.
- 확장: 소싱(도매매) 후보를 이 유형으로 태깅해 꼬띠가 "이번 주 황금상품 3·시즌 2" 식 큐레이션.

## 4. 상품 연동 UI 개선 (운영자: "한 줄로 쭉·보기 힘듦")
- 현재: 번호 입력 + 단일 라인 목록.
- 제안 — **카드 그리드 + 체크박스 선택**:
  · 썸네일 + 상품명 + 상태(판매중/품절/중지) + 부활등급(S~C) 배지 카드.
  · 다중선택 체크박스·상단 "선택 N건 연동" 액션바.
  · 필터(부활 후보만·품절·중지)·정렬(부활 시급순·최신). 
  · 번호 입력은 "빠른 추가" 보조 옵션으로 유지(둘 다 지원).
  · ADHD: 카드 스캔·시각 청킹(#233)·한 줄 텍스트 나열 지양.

## 5. 우선순위 (Code)
1. ★꼬띠 오타 수정(즉시·1줄) + 유니코드 한글 스캔.
2. 상품 연동 UI 카드+체크박스(운영자 직접 체감·독립).
3. 디스코드 운영 이벤트 알림(발행준비·부활·재고·마진·좀비 — 지금 유효분부터).
4. 꼬띠 추천 유형 분류(category-score 재사용).
5. 배송 이벤트 알림(연동/발행 늘면).
