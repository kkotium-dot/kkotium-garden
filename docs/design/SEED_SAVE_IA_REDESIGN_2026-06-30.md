## 0. 유료 솔루션 참고 패턴 (web 확인 — 사방넷/플레이오토/샵링커)
================================================================
| 패턴 | 솔루션 근거 | 우리 적용 |
|---|---|---|
| 수집 레이어 ≠ 상품 레이어 분리 | 사방넷 "판매상품/출고상품 이원화"(관리 관점별 별도 엔티티 + 연결코드) | 꿀통(crawl_logs)=수집 ≠ 창고(Product)=상품. 분리 유지(이미 그러함) |
| 일방향 상품화 + 소스 상태전이 | 수집→상품화 시 소스 '처리완료' 표기 | crawl_log SOURCED→PLANTED + productId FK |
| 소프트 비활성화(하드삭제 금지) | 사방넷 "삭제 않고 비활성화, 비활성화는 별도 조회/집계 제외" | 꿀통 PLANTED 기본 숨김 + "처리완료 보기" 토글 |
| 상태 파이프라인 = 단일 진실원천 | 출고요청→지시→작업중→완료 일관 노출 | 수집→작성중→발행대기→발행됨→품절 단계 뱃지(전역 공통) |
| 중복 가드 | 동일 항목 재수집/중복주문 방지 | 크롤 시 "이미 상품화됨" 경고 + 창고 역링크 |
핵심 교훈: 유료 솔루션은 수집/상품을 **의도적으로 분리**하고 **연결(FK)+상태전이+소프트삭제**로 단일 진실원천을 만든다. 우리도 분리는 유지하되 "연결+전이+숨김"만 추가하면 중복 착시 소멸.

## 1. 꿀통/창고 재설계 (각 기능 보존)
================================================================
### 꿀통 꽃나들이 (수집 사냥터) — 기능: 후보 발굴·마진비교·선별
- 기본 뷰 = **미상품화(SOURCED)만**. 마진/재고/판매가 비교로 "올릴 것" 선별.
- 카드마다 단계 뱃지 + [씨앗 심기] 액션(= 상품화, 창고로 승격).
- 상품화된 항목은 **기본 숨김** + "처리완료 보기" 토글(소프트 비활성화) → 양쪽 중복 노출 소멸.
- 중복 가드: 이미 상품화된 URL/상품번호 재크롤 시 "이미 OO으로 상품화됨" 경고 + 창고 링크.

### 정원 창고 (상품 보관·발행 관리) — 기능: 상품 생애주기 관리
- 상태 섹션 그룹핑: **작성중(DRAFT) → 발행대기(READY) → 발행됨(ACTIVE) → 품절**.
- 각 상품에 출처(소스 crawl_log) 역링크 표시 → 추적성("이 상품 어디서 왔나").

### 다리 = 씨앗 심기
- 꿀통 후보 prefill 진입 시 crawlLogId 동반. 저장 성공 시 서버가 crawl_log.productId 연결 + SOURCED→PLANTED 전이(원자적).

## 2. ★ 임시저장 vs 발행준비완료 — 확실한 재정의 (운영자 요청: 구분 유지)
================================================================
이전 rev1의 "2버튼 폐기(autosave 단일화)"는 **철회**. 운영자는 명확한 2단계 구분을 원함. → 저장 위치(임시 vs DB)가 아니라 **완성도/상태**로 구분. **둘 다 DB에 전 필드 영속**(저장 위치 차이 아님).

| | 임시저장 → "작업 보관" | DB저장 → "발행 준비완료" |
|---|---|---|
| 의미 | 미완성 OK · 이어서 작업하려 보관 | 필수항목 충족 · 발행 가능 상태로 확정 |
| 검증 | 없음(상품명만) | 6필수(상품명/카테고리/판매가/공급사/옵션/대표이미지) |
| status 결과 | DRAFT (작성중) | READY (발행대기) |
| 창고 표시 | "작성중" 섹션 · 회색 뱃지 | "발행대기" 섹션 · 초록 뱃지 |
| 전 필드 영속 | O (버그 수정 후) | O |
| 다음 행동 | 이어서 편집 | 네이버 발행 |

### 핵심 UX 변경
- 라벨: "DB저장" → **"발행 준비완료"**(또는 "상품 확정")로 변경. **"DB"라는 기술 용어는 사용자 화면에서 제거.** "임시저장"은 유지하되 툴팁 "입력한 그대로 창고에 보관 · 미완성 OK".
- 시각 구분: 임시저장=보조 버튼(아웃라인/회색), 발행준비완료=주 버튼(브랜드 레드 채움).
- 안전망(권장): 백그라운드 자동저장(필드 blur)으로 작업 유실 0 — 단 명시 2버튼은 그대로(자동저장은 침묵, 상태칩 "저장됨 HH:MM").
- READY 추가: Product.status enum에 READY(발행대기) 신설. DRAFT→READY→ACTIVE 파이프라인.

## 4. 공통 status 택소노미 (전역 통일 #55)
================================================================
- crawl_logs.sourcingStatus: SOURCED → PLANTED (소프트 비활성화, 삭제 금지).
- Product.status: DRAFT(작성중) → READY(발행대기) → ACTIVE(발행됨) → OUT_OF_STOCK/INACTIVE.
- 영어 상수만. 한글은 표시 레이어 매핑. 단계 뱃지 = 전역 공통 컴포넌트.
