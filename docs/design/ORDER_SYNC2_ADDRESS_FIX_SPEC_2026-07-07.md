## 0. 증상 / 데이터 감사 결과
================================================================
운영자: "연동 주문관리가 여전히 제대로 동기화 안 된 것 같다."
Desktop 데이터 감사(Supabase 실측, 전 24건):
| 필드 | 상태 |
|---|---|
| 고객명·전화·금액·상품명·결제일·상태 | ✅ 정상(누락 0) |
| **배송지(shippingAddress)** | 🔴 **24/24 전건 빈값** — 활성 SHIPPING 주문(오현석)조차 빈값 |
→ 주문 자체는 동기화되나 **배송지가 전건 누락 = 발송 실무 불가**. "제대로 안 됨"의 실체.

================================================================
## 1. 근본원인 (코드 실측 확정)
================================================================
naver/orders/route.ts:96 —
```js
const shipping = (el.shippingAddress as ...) ?? (el.delivery as ...) ?? {};
const shippingAddress = shipping.roadAddress ?? shipping.baseAddress ?? '';
```
- `customerName = order.ordererName`은 정상 작동 → `el.order` 접근 정확.
- 그러나 배송지는 **`el.shippingAddress`(최상위 없음)·`el.delivery`(경로 불일치)**에서 못 찾음 → 전건 빈값.
- 3-endpoint 재작성 시 응답이 평면 `{order, productOrder, delivery, currentClaim}`로 바뀌었는데 **배송지 경로만 이 구조에 맞게 갱신 안 됨**.
- ★교훈: customerName이 된다고 전 필드가 되는 게 아님 — **중첩 필드는 각각 라이브 응답으로 실측 필요**(#181).

================================================================
## 2. 수정 (verify-first #181 — Code 라이브 실측 필수)
================================================================
1. **라이브 응답 실측**: naver-doctor 또는 임시 debug 프로브로 `POST /v1/pay-order/seller/product-orders/query` 응답의 **실제 배송지 경로** 확인(추측 금지). 유력 후보:
   - `el.productOrder.shippingAddress { name, tel1, baseAddress, detailedAddress, zipCode }`
   - 또는 `el.order.shippingAddress` / `el.delivery.shippingAddress`.
   - 활성 주문(SHIPPING/PAID) 1건으로 확인(취소건은 주소 부재 가능).
2. **매핑 수정** (실측 경로 기준):
   - `shippingAddress` = baseAddress + ' ' + detailedAddress (합성) → Order.shippingAddress.
   - `shippingZipcode` = zipCode → Order.shippingZipcode(기존 필드 존재).
   - 수령인 = shippingAddress.name / tel1 (주문자 ordererName과 별개 — 필요 시 별도 표기).
3. **재동기화 백필**: 수정 배포 후 `manual=1&hours=720`(30일) 재실행 → 기존 24건 upsert로 배송지 채움.
4. (권장·별건) **정렬 개선**: /orders 목록이 `createdAt`(동기화일) 아닌 **`payment_date`(실주문일) DESC** 정렬·표시인지 확인. createdAt은 동기화 시점이라 목록이 "오늘 다 들어온 것"처럼 보임(distinct 8일). 실주문일 기준 정렬로 교정.
