# SLOT FUNNEL BOARD — Phase 3 슬롯 조립 UI 스펙 v1 (2026-06-30)

Authoring: DESKTOP. 권위: 본 문서(스펙) + DETAIL_PAGE_PLAYBOOK(7섹션·규격) + OPERATOR_SYSTEM_BLUEPRINT #56(개입) + asset-taxonomy.ts(슬롯 코드 진실). 구현=Code. 원칙 #55/#56/#45/#46.

================================================================
## 0. 목적 (왜)
================================================================
스튜디오(S2 이미지 생성/가공)와 발행(S5) 사이의 **수작업 Figma/Canva 조립**을 앱 내부로 흡수.
상품별 가공 자산(stage)을 **7섹션 860px 상세페이지**로 조립하는 단일 보드. 표준 케이스를 1인 셀러가 드래그만으로 양산 → 파워셀러 속도.

- 파이프라인 위치(OPERATOR_SYSTEM_BLUEPRINT §1): 4(이미지)→5/6(SEO/속성)→**조립**→7(검수)→8(발행).
- 출력 배관은 이미 존재(IMAGE-SPLIT): detail_images(jsonb) + buildDetailContent HTML. 본 보드는 그 입력을 **사람이 조립**하는 UI.
- 데이터 입력원: GET /api/products/{id}/assets — stage별(source/cutout/plate/composite/thumbnail/detail/archive) 구조 + publicUrl (Desktop 실측 검증완료, storage 직독).

## 1. 데이터 모델
================================================================
- **입력(읽기)**: assets API의 stage별 파일 목록(publicUrl). 변경 0.
- **슬롯 정의(고정)**: DETAIL_PAGE_PLAYBOOK §2의 7섹션 = 보드의 7개 슬롯 그룹(순서 고정):
  1 후킹 · 2 가치제안 · 3 상세스펙 · 4 사용법/활용 · 5 신뢰요소 · 6 CTA · 7 공통안내(=store_settings 상/하단, 이미 구축 → 읽기전용 핀).
- **슬롯 배정(신규 영속)**: 어떤 자산이 어느 섹션 슬롯에 들어가는지 + 순서. ★영속 방식은 OPEN(아래 §5 Q2) — 신규 경량 컬럼 vs 기존 detail_images 순서 재사용.
- ★Code 필수 확인: 실 슬롯/스테이지 택소노미는 **asset-taxonomy.ts**가 진실 — 본 문서의 stage 목록과 대조 후 정합(불일치 시 코드 진실 우선, 본 문서 갱신).

## 2. 화면 (IA·UX) — #26 Sidebar 기준
================================================================
- 진입: 온실 아틀리에(스튜디오) 내 탭 "상세 조립" 또는 정원창고 행 액션 "상세 조립". (Q1: 독립 라우트 vs 스튜디오 탭 — OPEN)
- 3분할 레이아웃(유료 SaaS급):
  - **좌: 자산 트레이** — stage 탭(누끼/합성/썸네일/상세/아카이브)별 썸네일 그리드. 자산 없는 stage는 "생성 필요" 안내 + 스튜디오 딥링크(#56 개입).
  - **중: 조립 보드** — 7섹션 슬롯을 세로로(860px 비율 미리보기). 각 슬롯에 자산 드롭 + 섹션 카피(우측 연동). 빈 섹션은 점선 플레이스홀더 + 가이드 카피.
  - **우: 섹션 인스펙터** — 선택 섹션의 카피(SEO 엔진 연동), 자산 교체, 캡션(명화=작품 출처).
- 라이브 미리보기: 모바일 폭(360) 토글 + 860px 데스크톱. 로딩 3초·48px 버튼 등 규격 체크 인디케이터.

## 3. 단계 증분 (SEED-SAVE식 검증가능 단위)
================================================================
- **SF-1 (읽기 전용 보드·독립)**: 7섹션 슬롯 + 상품별 자산 트레이를 assets API에서 렌더. stage별 그룹·존재 여부 표시. DnD/영속 없음. → 검증: 보드 렌더, 자산 stage 그룹 정확, 빈 stage 안내.
- **SF-2 (슬롯 배정·영속)**: 자산→섹션 슬롯 배정(DnD 또는 클릭) + 영속(Q2 결정 후). 자동저장(C-1 패턴 재사용). → 검증: 배정→재오픈 복원, 중복/순서 정확.
- **SF-3 (섹션 카피 연동)**: SEO 엔진/템플릿 카피를 섹션별 슬롯에 프리필(무료 우선·#155). → 검증: 카피 채움, 편집 영속.
- **SF-4 (860px 내보내기)**: 조립 보드 → 860px 분할 이미지 산출 + detail_images/detailContent 반영(기존 배관 재사용). 네이버 규격 게이트(세로 5000px·20MB·JPEG90). → 검증: 분할 산출물, 상세 HTML emit, 규격 통과.
- **SF-5 (#56 개입 통합)**: 조립 미완 = Operator Action Queue "결정" 카드로 surface + 1클릭 진입. → 검증: 큐 카드 노출·딥링크.

각 SF는 독립 검증·배포. SF-1은 데이터 무변경이라 즉시 착수 가능.
