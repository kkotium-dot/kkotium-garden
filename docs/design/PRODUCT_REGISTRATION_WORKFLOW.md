## 0. 한 줄 / 왜 이 문서

공급사(도매꾹/오너클랜) 원본을 네이버 스마트스토어에 등록할 때 **원산지·옵션 재고·고시 규격·이미지**가 추측이나 디폴트로 오염되지 않도록 강제하는 전상품 등록 표준. 합성/이미지 단계는 `ADAPTIVE_COMPOSITE_ENGINE.md`(v8)를 참조한다. 핵심 원칙: **확정값만 등록, 미확인은 보류 큐**(임의 디폴트 금지).

---

## 1. 등록 파이프라인 (표준 7단계)

```
STEP 1  인제스트     : 공급사 원본 크롤링 -> DB 적재(원본 보존)
STEP 2  제품정체 확정 : 공급사 실상세 육안 확정(#58) -> 상품현실시트 작성
STEP 3  원산지 흡수   : 공급사 원산지 표기 -> 원산지코드 룩업(국산 디폴트 금지, 2장)
STEP 4  옵션·재고 정합: 옵션 트리 + 옵션별 재고 일치 검증(3장)
STEP 5  고시·규격     : 상품고시정보 + 고시 사이즈 -> 스케일 앵커(4장)
STEP 6  이미지        : v8 참조 드롭 하모나이즈 스테이지(5장)
STEP 7  검수·발행     : 페이로드 전체 재구성 검증 -> 대표 GO -> 네이버 발행(비가역 #46)
```

미확인 필드가 하나라도 있으면 STEP 7 진입 금지 -> **보류 큐**로 빠진다.

---

## 2. 원산지 정합 (공급사 권위 -> 룩업, 국산 디폴트 금지)

**원칙: 원산지의 권위 소스는 공급사 표기.** 코드는 그 표기를 원산지코드 테이블로 흡수만 한다. 표기가 없거나 모호하면 **국산(00)으로 자동 디폴트하지 않고 보류 큐로 보낸다.**

- 룩업 테이블: `src/lib/naver/naver-origin-codes.ts`(518 코드, 로컬). 원산지코드.xls 기준.
- 매핑 규칙:
  - 공급사 "중국" / "수입" + 중국 표기 -> `0200037`(중국)
  - 공급사 "국산" / "국내산" / "한국" -> `00`(국산)
  - 공급사 표기 없음 / 모호("기타", "해외") -> **보류**(operator 카드: `origin_unconfirmed`)
- **금지**: 표기 부재 시 국산(00) 자동 채움. 과거 사고 = 미확인 상품에 국산 디폴트가 박혀 원산지 허위 표기 리스크.
- 검증 케이스: 명화(cmpnooli40001f0gveaxr8iim) = `originCode=0200037` / `naver_origin=중국` (교정 완료, 가역).

> 제조사·수입사: 공급사/상세페이지 표기를 권위로 흡수, 없으면 '꽃틔움'(메모리 `naver-manufacturer-importer-policy`, 대표 확정).

## 3. 옵션·재고 정합

- 옵션 트리(조합형/단독형)는 공급사 원본 구조를 보존하되, 네이버 옵션 규격으로 정규화.
- **옵션별 재고는 공급사 재고와 1:1 일치 검증.** 옵션 수 != 재고 행 수면 보류.
- 재고 수정은 네이버 v2 **FULL REPLACE** 규칙(CLAUDE.md §3-7) 준수 — 부분 PUT 금지, GET-merge 경로만.
- 폴링: `src/lib/dome-inventory-poller.ts`(Sprint 6-A, Option C 하이브리드)와 정합.

---

## 4. 고시·규격 (고시 사이즈 -> 스케일 앵커)

- 상품고시정보(전기용품/화장품/생활화학 등 카테고리별 고시 항목)는 공급사 표기 흡수 + 미확인 시 보류.
- **고시 사이즈(실측 치수 W×H×D, 용량 ml) = 합성 스케일 앵커.** 상품현실시트(`ADAPTIVE_COMPOSITE_ENGINE.md` §11)의 실측 치수 필드로 직결되어 이미지 단계의 과대·축소를 차단한다.
- 예: 명화 15ml 걸이형 -> 고시 치수가 누끼 프레임 점유율 가이드(무드 히어로 ~50~55%)를 결정.

## 5. 이미지 스테이지 (v8 참조 드롭 하모나이즈)

이미지 생성·합성은 `ADAPTIVE_COMPOSITE_ENGINE.md`(v8) 권위를 그대로 따른다. 등록 관점 요약:

- 대표(1번): 흰배경 단일 본품 누끼(REP_FINISHING §9 규격, 네이버 대표이미지 규정 — 텍스트·가격·라인업 금지, CLAUDE.md §3-6).
- 추가(2~9번)·상세 히어로: **참조 드롭 하모나이즈**(Firefly 참조 슬롯 -> Nano Banana 2) -> 형태 가드 -> 실패 시 결정론 폴백(sharp).
- 1 누끼 -> 4컷 기본/6컷 확장, 무드 ≥2(동적 도출, 한국 SEO/ROI 배경).
- 모든 이미지 적용은 가역(extra_images 슬롯, 이전 자산 보존). 발행 PUT만 비가역(#46).

---

## 6. 보류 큐 (미확인 필드 게이트)

STEP 3~5에서 확정 불가한 필드는 발행을 막고 Operator Action Queue 카드로 노출(product-agnostic):

| 보류 사유 | 카드 | 해소 조건 |
|---|---|---|
| 원산지 표기 부재/모호 | `origin_unconfirmed` | 공급사/대표 확정 -> 룩업 흡수 |
| 옵션-재고 불일치 | `stock_mismatch` | 옵션 트리/재고 행 정합 |
| 고시 필수항목 누락 | `notice_incomplete` | 고시 항목 보강 |
| 실측 치수 부재 | `dimension_missing` | 치수 입력(스케일 앵커) |

해소 전 발행 GO 금지. 게이트 통과 후에만 STEP 7.

---

## 7. 발행 (비가역 #46)

- 네이버 v2 등록/수정은 **전체 페이로드 재구성**(`buildNaverProductPayload`) — 부분 PUT 금지(CLAUDE.md §3-7).
- 실 PUT은 `confirm:true && !dryRun`에서만. dry-run으로 전체 payload 미리보기 후 대표 GO.
- 발행 후 적용 현황 명시(#54), 가역 자산은 스토리지 보존.
