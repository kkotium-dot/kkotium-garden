# 정원 대시보드 — 디자인 빌드업 원칙 (v3 = 베이스라인, 최종본 아님) (2026-07-10)

Authoring: DESKTOP. 권위: 운영자 방향 확정. 원칙 #234/#235/#237. 상태: 빌드업 진행 기준 문서.

## 0. 핵심 방향 (운영자 확정 — 오해 금지)
- v3(정원 컨셉 융합 시안)은 **"복붙 이식할 최종본"이 아니라 "더 나은 디자인으로 빌드업할 출발 베이스라인"**이다.
- 목표: **현재 앱 기능은 100% 유지** + v3의 정원 컨셉/감성 유지 + **디자인을 한 단계 승화**(더 세련·트렌디·완성도↑).
- 즉 "v3를 그대로 구현" ≠ 목표. "v3를 딛고 더 좋게" = 목표.

## 1. 빌드업 3축 (무엇을 유지/올릴지)
| 축 | 유지(불변) | 빌드업(향상) |
|---|---|---|
| **기능** | 현행 앱 전 기능·데이터 파이프라인·6청크 구조·6그룹 IA | (기능 삭제 없음·실데이터 바인딩만 정교화) |
| **컨셉** | 정원 컨셉·꼬띠 세계관·분수대/화단/온실/우체통/망루 은유·레트로팝 | 은유의 완성도·일관성·위트를 더 세련되게 |
| **디자인** | 밝은 빨강+핑크+크림·손그림·스캘롭·스티커 | 타이포 위계·여백 리듬·그림자/곡률 정제·모션 품질·트렌디함·프리미엄감 |

## 2. 빌드업 대상 (v3 대비 개선점 — 수석 디자이너 관점)
v3의 좋은 점은 유지하되, 아래를 끌어올린다:
- **시각 위계 정교화**: 분수대(중앙)·화단(오늘 할 일)·KPI·요약 3구획의 시선 흐름을 더 또렷이(primary 1·secondary·tertiary 명도/크기 대비 강화).
- **데이터 가독성 ↑**: 숫자/지표는 또렷한 데이터 폰트·정렬·단위 처리(감성 폰트가 데이터 가독성 해치지 않게).
- **여백·그리드 정제**: 8px 그리드·카드 간격·섹션 리듬을 프리미엄 SaaS 수준으로.
- **모션 품질**: 분수대 물 흐름·상태 반응 오브제(품절=시든 꽃·문의=벌·성공=나비)를 절제되고 목적 있는 마이크로 인터랙션으로(#217/#228·reduced-motion).
- **컴포넌트 일관성**: 카드/배지/버튼이 하나의 디자인 시스템으로 통일(마스터 12색·스캘롭 규격·그림자 토큰).
- **트렌디 악센트**: 그레인/유기형태/표현적 헤드라인 등 2026 트렌드를 브랜드에 맞게 소량(#234 4개 한정).
- **실데이터 정합**: 매출/주문/굿서비스 등은 실제 앱 값·미제공 지표(굿서비스 API 없음)는 표기 방식 정직하게.

## 3. 유지해야 할 앱 기능 정합 (기능 불변 체크)
- KPI = Order 직접집계 실값(로컬 실매출)·시계열 14일(#223).
- 요약 3구획 = 실 컴포넌트(ProductHealthSummary·InboxSummary·ControlTowerSummary) 매핑.
- 사이드바 = 현행 6그룹 IA(홈/상품/주문/고객/공급사/설정) 유지(#233). v3의 고객·공급사 분리는 현행에 맞춤.
- 꼬띠 = 등장 규칙(#228·빈상태/성공/오늘할일 안내만·해제가능·reduced-motion 정적).
- 발행/네이버 쓰기 = GO 게이트 불변(#46).

## 4. 빌드업 절차 (반복)
1. Claude Design v3 → **개선 프롬프트로 빌드업**(위 §2 향상점·감성 유지·기능 정합)·Tweaks로 팔레트/폰트/density 실험 → v4.
2. 확정 시안 → Desktop 토큰/스펙 추출(색·폰트·간격·곡률·그림자·모션) → globals.css/컴포넌트 매핑.
3. Code 단계 구현(저위험 순: 색·폰트·장식 → 레이아웃 → 데이터영역 무접촉)·기존 토큰/컴포넌트 재사용·hex 하드코딩 금지.
4. Desktop prod 검증(밀도·시선흐름·대비 AA·회귀0·실데이터 표시). 반복.

## 작업 유의사항/원칙 (신규)
- **#237** 운영자가 제공하는 디자인 시안(v3 등)은 "그대로 이식할 최종본"이 아니라 "빌드업의 베이스라인"으로 취급. 현행 앱 기능은 불변(삭제 금지)·컨셉/감성은 유지·디자인은 한 단계 승화(위계·가독성·여백·모션·일관성·트렌디·프리미엄감). 시안의 임의 수치/구조가 현행과 다르면 현행 기능·실데이터·IA에 맞춰 정합. "제공물 복붙"과 "제공물 딛고 향상"을 구분해 항상 후자로 진행.
