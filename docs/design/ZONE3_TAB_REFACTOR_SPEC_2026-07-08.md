================================================================
## 0. 문제 진단 (실측)
================================================================
현재 존3(우측 diff 슬라이드 패널)에 3개 관심사가 세로로 쌓임:
① 동기화 diff(네이버 vs 앱·drift 배지) ② 반영 push(가격/재고·PL-3) ③ 품절 안전망(대체상품+전환 체크리스트).
→ 한 화면에 "볼 것/할 것"이 뒤섞여 "어디서부터?" 혼란. 스크롤 길고 시선 분산.

================================================================
## 1. 해법 — 존3를 3탭으로 분절
================================================================
존3 상단에 **탭 바**, 한 번에 하나의 작업만 표시(progressive disclosure):

| 탭 | 내용 | 성격 | 탭 배지 |
|---|---|---|---|
| **동기화** (기본) | 네이버 vs 앱 diff·drift 필드·sync 상태 | 조회(read) | drift N필드(주황) |
| **반영** | 가격/재고 push(PL-3)·dry-run 미리보기·경고·체크리스트 | 실행(write·GO) | 반영 대기 시 점 |
| **품절대체** | SubstituteEditor·전환 전 체크리스트·재소싱 | 대비(safety) | 미등록 시 "!" |

- 기본 탭 = **동기화**(먼저 "무엇이 다른지" 파악 → 그 다음 반영/대체 결정 = 자연스러운 작업 흐름).
- 탭 전환 시 해당 관심사만 렌더 → 시선 집중·스크롤 단축.

================================================================
## 2. 프리미엄 SaaS 레이아웃 규격
================================================================
- **탭 바**: 패널 상단 고정. 언더라인형 탭(active=브랜드레드 언더라인+진한 텍스트, inactive=회색). 탭 우측에 배지(숫자/점/!). 높이 44px·좌우 패딩 16px.
- **패널 헤더**(탭 위): 상품 썸네일(40px)+상품명(1줄 말줄임)+상태 배지. 닫기 X 우상단.
- **콘텐츠 영역**: 탭별 단일 관심사. 섹션 헤더(13px·세미볼드·회색)+본문. 섹션 간 24px 여백(밀집 금지).
- **1탭=1주요액션**: 각 탭 하단에 primary 버튼 1개(동기화=새로고침 / 반영=반영하기[GO] / 품절대체=저장). 부차 액션은 secondary.
- **여백·비율**: 패널 폭 desktop 420~480px. 콘텐츠 좌우 패딩 20px. "정보 밀도 높되 숨 쉬는" 간격(라인하이트 1.5·필드 간 12px).
- **색·타이포**: 브랜드레드는 active/primary에만(과용 금지). 본문 뉴트럴 그레이. Pretendard. 이모지0·Lucide.
- 반응형: 좁은 뷰포트(<768px)는 탭 바 가로스크롤 or 풀스크린 모달 전환.

## 3. 재사용 패턴 (systemic·#55)
================================================================
- 이 "탭 분절 + 1탭1액션 + 배지" 패턴을 **공통 컴포넌트 `PanelTabs`**로 만들어 다른 과밀 화면(관제탑 상세·스튜디오 배양실 등)에도 확장 적용.
- 원칙: **한 패널에 3개 이상 관심사가 쌓이면 탭/아코디언으로 분절**(시선 흐름 통제).
