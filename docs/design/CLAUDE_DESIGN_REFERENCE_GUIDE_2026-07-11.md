# Claude Design 레퍼런스 전달 가이드 (Desktop 수신 최적화) (2026-07-11)

Authoring: DESKTOP. 계기: 운영자 "레퍼런스 어떻게 전달하는 게 최적인가". 권위: 실무. 상태: 운영자 안내 확정.

## 0. 최적 전달법
- **이미지 파일(PNG/JPG)을 이 채팅(Desktop)에 직접 업로드** — Desktop이 이미지 네이티브 처리. PDF(v6처럼)도 가능.
- 순서: **Desktop(이 채팅) 먼저 → 그다음 Claude Design.** Desktop이 먼저 봐야 v7 프롬프트를 레퍼런스에 맞춰 정교화 가능.

## 1. 준비 요령
| 항목 | 권장 |
|---|---|
| 포맷 | PNG/JPG(또는 PDF) 직접 업로드 |
| 개수 | 1~2장(많으면 무드 희석) |
| 해상도 | 선명(팔레트·질감·패널 추출) |
| 메모 | "이 이미지에서 [색감/카드·패널/일러스트 화풍/여백]을 가져오고 싶어" 한 줄 |
| 웹사이트 | 스크린샷 업로드(URL은 로그인/동적 실패 가능) |
| 다중 | 이미지별 역할 지정(1=색, 2=레이아웃) |

## 2. 우리 케이스 최적 레퍼런스
- "감성 배경 위에 데이터가 또렷" 공존 화면(정원 감성+숫자 가독성 해법). 파스텔/일러스트 배경 + 선명 KPI 대시보드/앱.

## 3. 수신 후 Desktop 작업 흐름
1. 이미지+메모 수신 → 2. 디자인 언어 추출 → 3. v7 개선점 결합 → "레퍼런스 기반 v7 프롬프트" 전달 → 4. 운영자가 Claude Design에 (이미지+프롬프트) → 5. 결과 PDF/스샷 수신 → Desktop 검증.

## 작업 유의사항/원칙 (재확인)
- **#246(보강)** 레퍼런스는 Desktop(채팅)에 이미지로 먼저 전달 → Desktop이 디자인 언어 추출+v7 결합 후 Claude Design용 프롬프트 정교화. 1~2장·추출 축 메모·선명 해상도·픽셀복제 금지·저작권/사진 회피.
