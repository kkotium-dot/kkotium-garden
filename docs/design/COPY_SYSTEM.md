## 1. 상태 배지 카피 (7종 · 상태 축)

| 상태 | 배지 라벨 | 색 강도 | 한 줄 의미 |
|---|---|---|---|
| `DRAFT_INCOMPLETE` | 작성 중 | info(회색) | 아직 등록 준비가 안 끝났어요 |
| `READY_UNPUBLISHED` | 발행 대기 | info(회색) | 준비 끝났어요. 아직 스토어엔 안 올렸어요 |
| `SOURCE_GONE_RESOURCE` | 공급처 단절 | critical(빨강) | 공급처가 끊겼어요 (지킬 판매 이력 있음) |
| `SOURCE_GONE_DELETABLE` | 공급처 단절 | warn(주황) | 공급처가 끊겼어요 (판매 이력 없음) |
| `SUSPENDED` | 판매중지 | warn(주황) | 스토어에서 내려둔 상태예요 |
| `OUT_OF_STOCK` | 품절 | warn(주황) | 지금 재고가 없어요 |
| `ON_SALE` | 판매중 | none/success | 정상 판매 중이에요 |

## 2. 처분 권고 카피 (5액션 · 권고 축)

**disposition.strings.ko.json 실제 값과 1:1** (label/action/tooltip/voice 키). `NONE`은 렌더하지 않음.

### MARK_OUT_OF_STOCK
- **label**: 공급사 품절 · **action(주 버튼)**: 품절 처리 권장
- **tooltip**: 공급사에 재고가 없는데 스토어에서는 아직 판매 중이에요. 주문이 들어오면 취소해야 하고, 취소가 쌓이면 스토어 등급이 깎여요. 스토어에서 품절 처리해두면 재입고될 때 바로 되살아나요.
- **voice**: 이랴➰ 지금 주문 들어오면 취소해야 해유! 품절로 바꿔두어유

### SUSPEND
- **label**: 장기 품절 · **action**: 판매중지 권장
- **tooltip**: 2주 넘게 품절이에요. 네이버는 오래 품절인 상품의 노출을 점점 낮춰서, 이대로 두면 순위가 조금씩 깎여요. 판매중지로 바꾸면 검색에서는 내려가지만 상품 주소와 리뷰·판매 이력은 그대로 남아요.
- **voice**: 빵야➰ 2주 넘게 품절이에유. 순위 깎이기 전에 잠깐 내려두어유, 리뷰는 안 사라져유

### RESOURCE (단절 + 자산 O)
- **label**: 공급처 단절 · **action**: 대체 소싱 권장
- **tooltip**: 공급사가 이 상품을 도매처에서 내렸어요. 판매 이력이 있어서 삭제하면 리뷰와 검색 순위가 함께 영구히 사라지고, 다시 올려도 리뷰 0인 새 상품이 돼요. 삭제 대신 판매중지 후 같은 상품에 다른 공급처를 연결하세요.
- **voice**: 빵야➰ 공급처가 사라졌어유! 근데 이 꽃은 팔린 적 있으니 지우지 말고 다른 공급처를 찾아주어유

### DELETE_SAFE (단절 + 자산 X)
- **label**: 공급처 단절 · **action**: 삭제 권장
- **tooltip**: 공급사가 이 상품을 도매처에서 내렸어요. 다시 들여올 수 없고 판매 이력도 없어서 지킬 리뷰나 순위가 없으니, 목록에서 삭제하는 게 깔끔해요.
- **voice**: 빵야➰ 이 꽃은 공급처가 사라졌어유. 삭제하고 새 꽃 찾아보아유

### NONE
- 렌더 안 함. severity='none'.

**보조 문자열**: `daysSuffix` = "일째" (품절 지속일 표기) · `sourcingLabel` = "공급처 단절".

## 3. 상태 ↔ 권고 동시 노출 예시 (모순 아님 검증)

| 케이스 | 배지(진단) | 주 액션(처방) | 정합 |
|---|---|---|---|
| 단절 + 자산O + 미조치 | 공급처 단절(빨강) | 대체 소싱 권장 | O |
| 단절 + 자산O + **이미 판매중지** | 공급처 단절(빨강) | (버튼 없음 · NONE) | O — 조치 완료, 사실은 유지 |
| 단절 + 자산X | 공급처 단절(주황) | 삭제 권장 | O |
| 장기품절(단절 아님) | 품절(주황) | 판매중지 권장 | O |
| 정상 | 판매중 | (없음) | O |

> disposition.ts:139가 SUSPENSION→NONE을 돌려주는 것과 정합. 배지는 사실을 유지하고, 버튼만 사라진다.

## 4. 문구 작성 규칙 (#262 · #73)
- 개발자 은어 금지("튜닝"·"방어" 등). 셀러 실무 용어 + 정원 컨셉 하이브리드.
- 코드 내부(타입명·변수명·주석)는 영어 유지 — 화면에 안 보임.
- 네이버 공식 용어(예: 카테고리명)는 원문 유지(검색 직결).
- "좀비"는 운영자 의도 컨셉이라 예외(은어 아님).
- 애매하면 후보 2~3개를 운영자에게 제시하고 확인.
