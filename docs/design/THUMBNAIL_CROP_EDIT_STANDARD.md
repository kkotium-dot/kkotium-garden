# 썸네일 크롭·이미지 편집 표준 (THUMBNAIL / CROP / EDIT STANDARD)

> 박제일: 2026-06-07 · 적용 범위: **전상품 상품등록** (단일 상품 아님)
> 목적: "상세페이지·사이트의 잘 찍힌 제품컷을 규격 썸네일로 추출·편집"하는 운영 표준.
> 이 문서는 운영자 지시의 영구 기준이다. 위반 크롭(배경·잡요소·글씨 포함)은 재작업 대상.

---

## 0. 한 줄 원칙
**썸네일 = 제품이 주인공인 규격컷.** 상세페이지 속 잘 찍힌 컷을 1:1로 타이트하게 크롭하고, 필요 시 글씨 제거·영역확장으로 다듬는다. 없는 걸 새로 만들지 말고, 좋은 자산을 정확히 추출한다.

---

## 1. 좋은 컷 기준 (GOOD-CUT CRITERIA)

| 항목 | 기준 |
|---|---|
| 주제 | 제품이 명확한 주인공, 프레임의 60~80% 차지, 중앙/안정 구도 |
| 배경 | 흰색·중립 OR 의도된 라이프스타일 맥락(가죽 차량 인테리어 등). 어수선함 금지 |
| 텍스트 | 책등·캡션·UI·워터마크 등 잡텍스트 0 (있으면 글씨 제거 편집). 제품 라벨 텍스트는 허용 |
| 규격 | **1:1 비율, ≥1000px**. 부족 시 업스케일 또는 캔버스 영역확장 |
| 대표(rep) | 깨끗한 단품(흰/중립) 또는 프리미엄 라이프스타일 단품 |
| 추가컷 | 라이프스타일·장착·선물세트·디테일 매크로 |

### 반려 컷 (REJECT) — 지난 실패 사례 기록
- 임의 배경 영역(가죽만 잔뜩), 부분 잘린 제품, 드로퍼팁만 매크로, 캡션 밴드 포함, 다물체 어수선 합성.
- → 이런 "쓸데없는 사항"이 들어가면 썸네일 부적합. 제품 중심 타이트 크롭으로 재작업.

## 2. 기준컷 레퍼런스 (명화 디퓨저, 운영자 제공 5컷)
상세페이지에서 이런 컷을 크롭해야 한다는 운영자 지정 기준:

| # | 컷 | 용도 | 편집 필요 |
|---|---|---|---|
| 1 | 대시보드 2병 장착 | 라이프스타일 추가컷 | 없음 |
| 2 | 4병 스타일 플랫레이(+식물·책) | 라이프스타일 추가컷 | **책 글씨 제거**(Barcelona/Lifestyle Creator's/DOR) |
| 3 | 가죽 도어 단품 장착(+올리브잎) | **대표 1순위**(프리미엄 단품) | 1:1 영역확장 가능 |
| 4 | 흰 접시 단품+리필 드로퍼 | **대표 후보**(클린 제품) | 1:1 영역확장 가능 |
| 5 | 기프트박스(Monet) | 선물 추가컷 | 없음 |

생산물(v2, /mnt/user-data/outputs/): v2_thumb_dashboard / flatlay / leather_single / whitedish_single / giftbox (1:1 1000px).

## 3. 3가지 작업 모드 (운영자 개입 모델)

| 모드 | 트리거 | 동작 | 도구 |
|---|---|---|---|
| A. 자동 크롭 | 상세 품질 양호 | 잘 찍힌 제품컷 자동 탐지 → 1:1 타이트 크롭 | 내장 크롭(Sharp) |
| B. 지정영역 크롭 | 운영자가 영역(박스) 지정 | 지정 좌표 그대로 크롭 | 내장 크롭 + 좌표 입력 UI |
| C. 이미지 편집 | 글씨/여백/배경 문제 | 글씨 제거·영역확장·배경정리 | **Adobe Firefly / Express MCP** |

### 편집 연산 정의 (C)
- **글씨 제거(inpaint)**: 생성형 채우기로 책등·캡션·로고 등 제거. (예: 기준컷 #2 책 글씨)
- **영역확장(outpaint)**: 비정사각 컷을 1:1로 — 흰 패딩 대신 배경을 자연 생성해 확장. (예: 504×492 → 1000² 깔끔)
- **배경정리/누끼**: 어수선 배경 → 흰/중립. (라인 B 또는 대표 클린화)

## 4. 2개 워크플로우 라인 (전상품)

| | 라인 A — 양질 자산 | 라인 B — 저품질 / 고상품성 |
|---|---|---|
| 트리거 | 상세·사이트 썸네일 품질 양호 | 품질 미달 OR 상품성 높아 차별화 필요 |
| 썸네일 | 상세페이지 직접 크롭(모드 A/B) + 필요시 편집(C) | 누끼+무드배경 / Firefly 생성·업스케일 |
| 상세 | 그대로 활용 + SEO 문구·이미지 보강 | Canva·Figma·Claude Design 빌드(리디자인) |
| 도구 | 내장 크롭 + Firefly(편집) | Firefly·Express·Canva·Figma·Claude Design |
| 개입점 | 프리뷰 갤러리에서 크롭 컷 택1 / 영역 지정 / 편집 승인 | 생성 후보 택1 + 디자인 승인 |

명화 = 라인 A (상세 양질 → 크롭 + 대표 컷 선택).

## 6. 개입점이 자연스럽게 (운영자 경험)
- 발행전 프리뷰 화면 = 단일 검수 지점. 여기서: 크롭 후보 갤러리 택1 / 영역 박스 지정 / "글씨 제거"·"1:1 확장" 버튼 / 생성 후보 택1.
- 운영자는 로그인·CAPTCHA·비가역 클릭·다운로드만 담당. 나머지(프롬프트·생성·선택·폴링)는 앱/Claude가 구동.

---

## 7. 불변 규칙
- 외부 무단 생성 금지 — 좋은 자산은 그대로/크롭 우선. 생성은 자산 빈약 시(라인 B).
- 명화 라벨 보존 — 글씨 제거는 잡텍스트만, 제품 라벨/명화 그림은 보존.
- 1:1·≥1000px 규격. 발행전 프리뷰 통과 없이 비가역 PUT 금지.
- 전상품 적용 — 신규 등록 상품마다 라인 판정 → 모드 A/B/C 라우팅.
