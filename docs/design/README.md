# docs/design/ — 디자인 / UX / IA 산출물

> 이 폴더는 꽃틔움 가든의 **시각 디자인 작업 전 단계** 또는 **설계 의사결정**의 영구 산출물을 담는다.

## 폴더 정책

- **docs/plan/** = 매 세션 정독 (상태/진행/세션 로그)
- **docs/research/** = 외부 리서치 보고서 (필요할 때만 grep/read)
- **docs/design/** = 디자인/UX/IA 설계서 ← *본 폴더*

세 폴더는 역할이 명확히 분리되며, 새 채팅 시작 시 docs/plan/만 정독하면 충분하다. docs/design/은 *디자인 작업 시점*에 정독.

## 현재 보관 문서

## 사용 패턴

### 디자인 작업 시작 시
1. `MASTER_UX_BLUEPRINT.md` 정독 — Part 8 (디자인 손볼 때 우선순위) Tier 1부터 진행
2. Design Tokens 정의 → 컴포넌트 라이브러리 → 마스코트 SVG 순
3. 작업 결과를 본 폴더에 `DESIGN_TOKENS.md`, `COMPONENT_LIBRARY.md` 등으로 누적

### IA / UX 의사결정 시
1. 블루프린트의 Part 3 (IA) + Part 4 (핵심 화면) 참고
2. 변경 결정은 본 문서 v2/v3로 진화시키되, 변경 사유는 **버전 이력** 섹션에 기록
3. 큰 의사결정은 별도 ADR 형식 문서로 분리 (예: `IA_DECISION_2026_05.md`)

### 신규 화면 추가 시
1. 블루프린트 Part 4의 *5요소 (JTBD / 정보 계층 / CTA / 자동화 / 효율 패턴)* 그대로 적용
2. 신규 화면이 기존 8개 화면과의 deep-link 연결 명시
3. 사용자 여정 4개에 새 화면이 어떻게 끼어드는지 추가 검증

## 새 문서 추가 가이드

### 파일명 규칙
- `MASTER_*.md` — 전체 가이드성 문서
- `DESIGN_TOKENS.md` — 색상/타이포/spacing 토큰
- `COMPONENT_LIBRARY.md` — 컴포넌트 명세
- `MASCOT_KKOTTI.md` — 꼬띠 마스코트 디자인 가이드
- `JOURNEY_<여정명>.md` — 특정 사용자 여정 상세
- `IA_DECISION_<YYYYMM>.md` — IA 변경 의사결정 기록

### 섹션 구조 권장
- **헤더**: 작성일 / 버전 / 대상 / 범위
- **본문**: Part 단위 구분, 각 Part는 정의 + 근거 + 함의
- **부록**: 표/매트릭스/체크리스트
- **버전 이력**: 변경 사유 + 일자
