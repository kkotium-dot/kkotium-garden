## 1. 생애주기 = 정원 흐름 (역할 명확)
```
발굴·대기            제작              재배·돌봄·부활
꿀통 창고  ──씨앗심기──▶ 온실아틀리에 ──발행──▶ 꽃밭 돌보기
(크롤링·미발행)         (씨앗심기·꽃단장)        (등록 상품 통합)
    ▲                                            │
    │                                       좀비 감지·수정
    └────────────── 씨앗심기(중앙 엔진) ◀─────────┘
```
- **꿀통 창고**: 크롤링한 상품·임시저장·작성중 = 아직 발행 안 한 상품 정리소. 크롤링 기능 유지.
- **온실아틀리에**: 씨앗심기(등록·수정)·꽃단장(디자인). 제작.
- **꽃밭 돌보기**: 발행된/스토어 연동 상품 통합 관리. 좀비 감지·SEO 튜닝·품절 대체·부활.
- 나눔 기준: **발행 여부**(미발행=꿀통창고·발행/연동=꽃밭돌보기). 같은 Product 테이블의 두 뷰.

## 2. 씨앗심기 = 중앙 편집 엔진 (용어 단순화·"3레이어" 폐기)
운영자 실제 흐름 그대로:
```
(꿀통창고 or 꽃밭돌보기) 상품 클릭
   ▼
상품 정보 화면 (인라인·답답하지 않게)
   · 연동 상품 = 네이버에서 가져올 수 있는 중요 정보 최대 표시
   · 좀비면 "왜 좀비인지" 한 줄 이유
   │ [수정] 버튼
   ▼
씨앗 심기로 이동 — 기존 정보가 그대로 입력된 상태
   · 그 상태에서 수정
   · [임시저장] → 꿀통창고에 '작성중'으로
   · [스토어 등록/재등록] → 발행준비 체크 → 네이버 반영(GO)
```
- 연동 상품 수정 시: 네이버 원본 값이 자동 채워지고, 바꾼 부분만 수정해서 재등록. (원본 대비 뭘 바꿨는지 시각 표시는 보조 — 별도 "레이어" 개념 아님.)

## 3. 꽃밭 돌보기 상세 (운영자 요청)
- 연동+앱등록 **구분 없이 하나의 등록 상품 목록**(#245).
- 수백 개 대비: 하단 **`< 1 2 3 4 … >` 페이지네이션** + **상품번호·상품명 검색창** + 정렬(튜닝 필요도·최신).
- **동기화 버튼**(주문관리처럼): 연동 상품이 실시간 아니면 수동 동기화(네이버 최신 상태 가져오기).
- 상품 클릭 → **인라인 정보 화면**(연동 상품은 네이버 기본 중요 정보 최대·자연스러운 구조) → [수정]→씨앗심기.
- 상태 필터 심플: **판매중 / 판매중지 / 좀비발견** (+ 필요시 품절).
- 연동 선택 화면(승인됨): 판매중/판매중지 외 **튜닝 필요도로 "키울 상품/방어/내릴 상품" 구분**.

## 4. 좀비 감지 + 부활 (꽃밭 돌보기 내)
- 조건(품절·장기 악성재고·마진 위기·SEO 미달) → **좀비 배지** + 디스코드("좀비 출현! 지금 부활 시 마진 00%·[수정 바로가기]").
- 좀비 상품 클릭 → 좀비 지수뿐 아니라 **"왜 좀비인지 한 줄"**("7일간 판매 0%"·"경쟁가 하락 마진 위기").
- 튜닝 필요도 = 자체 지수(네이버 공식점수 API 불가·#255). 우선순위: 품절+실적>마진위기>성장여력>악성재고.

## 5. 품절 시 대체 상품 (승인됨)
- 재고 0 순간 → 빨간 강조 "대체 상품 지정" 폼. 입력: ⓐ앱 상품 선택 ⓑ도매매 코드 ⓒ같은 카테고리 자동 추천.
- 지정 시 상세페이지 "이 상품 추천" 노출·기회손실 최소화.

## 6. 나의 추가 제안
- **꿀통 창고 탭 구분**: [크롤링 상품 / 작성중(임시저장)] 2탭 — 성격이 달라 섞이면 혼란. 심플 2분할.
- **다음 할 일 1버튼**(카드마다·#240)·**포커스 모드**(상품 클릭=전용 작업공간).
- **씨앗심기 진입 일관화**: 꿀통창고·꽃밭돌보기 어디서 [수정]을 눌러도 동일한 씨앗심기로(중앙 엔진).

## 7. 단계별 실행 (승인 후·라우트 보존)
- **P1 (명명·재편·저위험)**: garden-nav 라벨 변경(온실아틀리에·씨앗심기·꽃단장)·상품연동→꽃밭돌보기 분리·소싱+창고→꿀통창고 통합·필터 축소·모호 배지 제거.
- **P2 (꽃밭돌보기 코어)**: 통합 목록·검색·페이지네이션·동기화 버튼·인라인 정보 화면.
- **P3 (씨앗심기 연계)**: 수정→씨앗심기 기존정보 프리필·임시저장/재등록.
- **P4 (좀비·튜닝)**: 튜닝 필요도 지수·좀비 배지+한줄이유+디스코드·품절 대체 폼·연동선택 튜닝 구분.
- 각 P: tsc0/build0→브라우저 실측→배포→Desktop 검증.
