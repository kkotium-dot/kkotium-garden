## 3. 진단 — 왜 "뒤죽박죽"으로 느껴지나

1. **빈 데이터가 최대 면적을 차지** — 시장 분석 234px(28%)가 전부 0. 정보가 아니라 소음.
2. **중복** — 순마진/마진율이 2~3곳에 다른 문구로 반복. 어느 게 맞는지 헷갈림.
3. **라벨 부재** — 7블록 중 2개만 라벨. 나머지는 구분선 없이 연달아 붙어 눈이 쉴 곳이 없음.
4. **우선순위 없음** — "지금 뭘 해야 하나"가 안 보임. 좋은 정보(꿀통지수)와 나쁜 정보(미흡 항목)가 같은 무게로 나열.
5. **탭 3개 추가 예정** — 이 상태에서 그냥 추가하면 악화 확정.

## 4. 재설계 스펙

### 4-1. 탭 구조 (작업 F 이식분 수용)

```
┌─ [정보] [동기화] [반영] [품절대체] ─┐   ← PanelTabs (link 페이지 패턴 재사용)
│                                      │
│  (탭별 콘텐츠)                        │
└──────────────────────────────────────┘
```
- 기본 탭 = **[정보]**
- 탭 라벨은 셀러 실무 용어(#262). 개발 용어 금지.
- 좀비 상품이면 [품절대체] 탭에 빨간 점 표시(사유에 "대체상품 미설정"이 있을 때만)

### 4-2. [정보] 탭 — 섹션 순서 (급한 순 = 셀러 작업 순서)

| 순서 | 섹션 | 라벨 | 조건부 |
|---|---|---|---|
| 1 | **좀비 사유 + 꼬띠 한마디** | (라벨 없음·빨간 카드 자체가 신호) | 좀비일 때만 |
| 2 | **지금 할 일** | `지금 할 일` | 미흡 항목 있을 때만 |
| 3 | **핵심 지표** | `한눈에 보기` | 항상 |
| 4 | **가격** | `가격` | 항상 |
| 5 | **재고·배송** | `재고 · 배송` | 항상 |
| 6 | **네이버 정보** | `네이버 정보` | 연동 상품만 |
| 7 | **시장 분석** | `시장 분석` | **데이터 있을 때만** ★ |
| 8 | **꼬띠 한마디** | (라벨 없음) | 좀비 아닐 때만(1번과 중복 방지) |

### 4-3. 필수 규칙

**R1. 빈 데이터 섹션은 렌더하지 않는다**
- 시장 분석: 경쟁상품수 0 && 평균가격 0 이면 **섹션 자체를 숨김**
- "데이터 없음" 문구조차 넣지 말 것 — 자리만 차지함
- 이 규칙은 패널 전체에 적용(전상품 범용·#55)

**R2. 같은 사실은 한 번만**
- 순마진/마진율 표시는 **가격 섹션 한 곳**으로 통일
- 핵심 지표(#3)에는 꿀통지수·SEO만
- 좀비 사유의 "순마진 0.0% — 마진 위기"는 유지(사유 설명이므로 중복 아님)
- 꿀통지수 하위의 "순마진 0.0% — 수익성 위험 (공급가 재협상 필요)"는 **제거**(가격 섹션과 중복)

**R3. 모든 섹션에 라벨**
- 고아 텍스트(공급사명 "보배몰" 등)는 소속 섹션으로 편입
- 라벨 스타일 통일: 11px · 600 · #9CA3AF · uppercase 금지(한글이라 의미 없음)
- 섹션 간 여백 16px · 구분선은 필요한 곳만(과하면 더 복잡해짐)

**R4. 첫 화면(스크롤 0)에 핵심이 다 보여야 함**
- 목표: 좀비 사유 + 지금 할 일 + 핵심 지표 = 스크롤 없이
- 나머지는 스크롤 허용
- 현행 191px 숨음 → 재설계 후 재측정 필수

**R5. 폭**
- 현행 `w-[min(720px,50vw)] min-w-[420px]` 유지하되, 탭 3개 추가 후 재측정
- 1760px 화면에서 41% — 아이템스카우트/셀퍼 수준(35~45%)에 부합하므로 폭 자체는 유지 가능
- **문제는 폭이 아니라 내부 구조** — 폭을 더 늘리기 전에 R1~R4부터 적용할 것
