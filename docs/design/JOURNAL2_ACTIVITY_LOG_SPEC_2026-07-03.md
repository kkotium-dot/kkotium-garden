## 1. verify-first (#181 — Code 착수 전 실측)
================================================================
★Code 필수 — 신규 로깅 테이블 만들지 말고 **기존 이벤트 소스 집계**(#181):
- `asset_jobs` — 잡 생명주기(created/updated·job_type·status). JobLifecyclePanel이 이미 사용 → 소스 확인됨.
- `crawl_logs` — 소싱 이벤트(created_at·sourcing_status·product_id).
- `Product` — status 변경·updatedAt(발행/재고 등 상태 전이).
- 자산 생성 — storage.objects / asset_registry created_at(상품별 prefix).
- ★실측: 위 중 상품별(productId) + 시각(created_at) 으로 저비용 조회 가능한 소스를 확정. 신규 테이블은 최후.
