## 1. ★운영자가 디스코드 서버에서 해야 할 것 (운영 이벤트 알림 추가 시)
새 운영 알림(발행준비·부활·주문배송·마진)을 추가하려면 **채널+webhook을 운영자가 생성**해야 함:
1. 디스코드 서버에 새 채널 생성(권장명):
   - `#🚀발행-준비` (발행 가능 전환)
   - `#🧟부활-후보` (revival S/A 신규)
   - `#🛒주문-배송` (신규주문·발송·배송완료)
   - `#📉마진-경고` (마진 임계 이하)
   *(또는 기존 #📊운영-리포트 하나에 통합해 스레드로 나눠도 됨 — 채널 수 최소화 원할 시)*
2. 각 채널 → 채널설정 → 연동 → 웹후크 → 새 웹후크 → URL 복사.
3. **Vercel 환경변수 추가**: DISCORD_WEBHOOK_PUBLISH_READY·_REVIVAL·_ORDER_SHIP·_MARGIN_ALERT (Code가 env 키 확정해 전달).
4. 코드 username 수정(꼬뛲→꼬띠) 배포 후 → 알림 봇 이름 "꼬띠" 정상 확인(서버작업 아님·확인만).
5. 기존 5채널 유지. #🌸꼬띠-오늘추천 = 황금/니치/시즌 큐레이션 채널로 활용.
- ※채널명/이모지는 취향대로 변경 가능(코드는 webhook URL로만 전송·채널명 무관). 위는 권장.

## 2. 알림 고도화 아이디어 (운영자 요청분 + 추가)
- **행동 유발 알림**(#56): 각 알림에 "앱에서 열기" 딥링크(→/products/[id])·꼬띠 한마디(세계관 보이스).
- **알림 피로 방지(ADHD)**: 우선순위 티어 — 🔴긴급(주문·품절 실시간) / 🟡주의(마진·좀비) / 🟢정보(추천·리포트 다이제스트로 묶음). 저우선은 일일 다이제스트 1회로 배칭.
- **꼬띠 보이스**: "대표님, OO가 발행 준비를 마쳤어요! 🌷 지금 올리면 좋아요" (외계어 배제·#233).
- **주간 꼬띠 큐레이션**: 황금/니치/시즌 Top pick + 근거(네이버 검색↑·마진·경쟁).
- **네이버 시즌 캘린더 연동**: 시즌상품 추천을 네이버 쇼핑 시즌 이벤트(빼빼로·화이트데이·여름·명절 등) 기준으로(네이버 우선 원칙).

- 황금상품 = **네이버 검색량↑ + 네이버 수수료 반영 순마진↑ + 네이버 경쟁도↓** (category-score#249 재사용·전부 네이버 DataLab/수수료 기반).

- 시즌상품 = 네이버 쇼핑 시즌 트렌드 급상승.
- ★모든 최적화는 네이버 기준(쿠팡/해외는 후속 어댑터 확장·현재 기본=네이버).
