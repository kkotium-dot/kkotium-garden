# 상세페이지 제작 플레이북 (모든 상품 공통 SOP)

> 저장: 2026-06-03 | 권위 문서 (모든 상품 등록 시 적용되는 표준 작업 절차)

## 0. 적용 범위
꽃틔움 스토어에 등록하는 **모든 상품**. 도매처 원본 상세이미지를 그대로 쓰지 않고, 이 플레이북으로 재가공해 품질을 표준화한다. 썸네일을 Firefly·Express로 가공하듯 상세페이지도 동일 수준으로 격상한다.

---

## 1. 5단계 하이브리드 파이프라인 (도구 역할 분담)
각 도구를 "강점이 있는 단계"에만 투입한다. 한 도구로 다 하려 하지 않는다.

| 단계 | 작업 | 주력 도구 | 산출물 |
|---|---|---|---|
| **S1. 소재 확보** | 상품 사진 정리, (명화 상품) 퍼블릭 도메인 작품 확보 | 도매처 원본 + Met Open Access(CC0) | 원본 자산 |
| **S2. 이미지 생성·합성** | 배경 생성, 제품 합성, 배경 제거, 라이프스타일 컷 | **Adobe Firefly / Express** (유료=IP 면책) | 가공 이미지 |
| **S3. 기획·프로토타입** | 섹션 구성, 카피, 레이아웃을 인터랙티브 목업으로 검증 | **Claude Design + 아티팩트** (DESIGN.md 공급) | 클릭형 HTML 시안 |
| **S4. 정밀·양산** | 확정 시안을 860px 규격으로 조립·완성 | **Figma**(정밀) 또는 **Canva**(빠른 템플릿) | 860px 분할 이미지 |
| **S5. 업로드·SEO** | 스마트에디터 ONE 분할 업로드, 모바일 확인 | 네이버 스마트스토어 | 발행 |

**도구 선택 기준**: 정교한 컴포넌트·반복 양산 → Figma / 빠른 템플릿·즉시 수정 → Canva / 아이디어·구성 검증 → Claude Design.

## 2. 7섹션 구성 공식 (전환율 검증된 순서 — 모든 상품 공통)
순서는 고정, 내용은 상품별로 채운다.

1. **후킹 (첫 화면)** — 5초 내 이탈 방지. 감성 비주얼 + 한 줄 카피. (로딩 3초 초과 시 모바일 53% 이탈)
2. **가치 제안** — "왜 이 상품인가" 핵심 3가지만. 차별화 포인트 강조.
3. **상세 스펙** — 소재·크기·구성·기능. 미니멀 스펙 카드(DESIGN.md §5).
4. **사용법 / 활용 장면** — 3단계 일러스트 또는 라이프스타일 컷.
5. **신뢰 요소** — 포토리뷰·인증·출처표기. (명화 상품은 작품 캡션 = 진정성)
6. **CTA** — 대형 레드 버튼 + 초대형 카피. 모바일 폭 90%.
7. **공통 안내** — 배송·교환·반품 (상단/하단 공통 이미지 슬롯 = store_settings, 이미 구축됨).

## 3. 네이버 스마트스토어 규격 체크리스트 (발행 전 필수)
- [ ] 가로 860px (제작은 1000~1200px 후 축소)
- [ ] 이미지당 세로 5,000px 이하, 용량 20MB 이하, JPEG 퀄리티 90+ (배경 투명은 PNG)
- [ ] **분할 업로드** (통이미지 금지 — 로딩·SEO·가독성)
- [ ] 모바일 본문 16px+/행간 1.5, 버튼 48×48px+
- [ ] 대표 썸네일: 텍스트·워터마크·외곽선 없음 (검색 불이익 방지)
- [ ] 상세 설명 텍스트·태그·속성 채움 (적합도 SEO 기여)
- [ ] 업로드 후 **실제 스마트폰에서 육안 확인**

## 4. 카테고리별 적용 (DESIGN.md §6 톤 변주와 연동)
- **아로마/디퓨저**: 후킹=아트갤러리 무드, 스펙=향·지속력·발향방식·차량조화, 신뢰=성분 불검출·작품 출처.
- **전통/도어벨/소품**: 후킹=한국적 여백, 스펙=소재·수공·크기, 신뢰=정서적 스토리.
- **주방/생활**: 후킹=청결·실용, 스펙=기능·위생·용량, 신뢰=관리 편의·안전.
- **선물/기프트(공통 오버레이)**: 패키지·메시지카드·선물 장면 추가.

## 5. 명화·이미지 저작권 안전 규칙 (명화/아트 상품 필수)
- **사용 가능**: 사후 70년 경과 퍼블릭 도메인 작가 — 모네(1926)·고흐(1890)·클림트(1918)·드가(1917) 등. 한국 저작권법상 자유 상업 이용.
- **원본 출처**: **Met Open Access(CC0, 출처표기 의무 없음·상업 무제한)** 우선 사용. 반 고흐 미술관 등 "비상업 한정" 약관 사진본은 회피.
- **Firefly 생성 시**: 유료 플랜(IP 면책) 사용. 실존 브랜드/인물/캐릭터 프롬프트 금지. 업로드 레퍼런스·합성 결과물은 면책 제외임을 인지.
- **AI 얼굴 금지**: "얼굴 없는 인체 일부 전략" 하드 규칙 유지(기존 시스템 원칙).
- **캡션 표기**: 작가·작품명·연도·"Public Domain"을 상세페이지에 명시 (진정성 + 스토리텔링 + 분쟁 예방).

## 7. 전환 KPI & 개선 루프
- **측정**: 페이지 로딩 3초 이내, 첫 화면 이탈률, CTA 클릭률, 모바일/PC 전환율 격차.
- **A/B 단일변수 순서**: 후킹 이미지 → CTA 카피 → 버튼 색상 순. 한 번에 하나만.
- **정체 시**: 모바일 전환이 PC보다 현저히 낮으면 모바일 폰트·버튼 크기부터 재점검.

## 8. 앱 통합 로드맵 (향후 — 이 플레이북의 자동화)
현재는 수동 SOP. 다음 단계로 앱(꽃틔움 가든)에 통합 가능:
- **Phase A**: 이 플레이북의 7섹션 구조를 상세페이지 생성 템플릿으로 코드화 (상품 데이터 → 섹션 자동 배치).
- **Phase B**: DESIGN.md 토큰을 앱 디자인 시스템 상수로 반영 (썸네일·상세 자동 생성 일관성).
- **Phase C**: Firefly 합성 → Claude Design 프로토타입 → 860px export를 반자동 파이프라인으로.
- 트리거: 첫 상품(명화 디퓨저) 상세페이지를 이 플레이북으로 수동 완성·검증한 뒤 패턴을 코드화.
