## 1. 3단 + 이중 사이드바 (3-column + dual sidebar)

온실 아틀리에 데스크톱(lg+) 레이아웃:

```
[헤더(아틀리에 타이틀)]
[스테퍼: 썸네일 랩 → 상세 캔버스 → SEO 부스터 → 발행 검토]   (전폭)
┌────┬──────────┬───────────────────────┬───────────────┐
│레일│ 동적패널 │      개화 작업대        │  검색 생장    │
│w-16│  w-96    │     (flex-1 min-w-0)   │  관제탑       │
│아이│ (접힘O)  │                        │ clamp 24%     │
│콘탭│          │                        │               │
└────┴──────────┴───────────────────────┴───────────────┘
```

- **이중 사이드바**(좌): `w-16`(64px) 아이콘 레일 + `w-96`(384px) 동적 패널.
  - 레일 탭 3종: **창고**(에셋·도구함) / **배양실**(향·무드 배양, S2-A 골격) / **일지**(작업 기록, S2-A 골격).
  - 활성 아이콘 재클릭 = 패널 **접힘**(작업대가 폭을 회수). 다른 아이콘 클릭 = 해당 탭으로 전환 + 패널 열림.
  - 좁아지면 패널을 접어 레일만 어포던스로 유지(향후 S2-B 오버레이 정교화).
- **개화 작업대**(중앙): `flex-1`, 남는 폭 전부. 디바이스 토글 + 라이브 미리보기 + 단계 카드.
- **검색 생장 관제탑**(우): `clamp(280px, 24%, 360px)`. 발행 준비도 신호등.
- 모바일(<lg): 고정 분할 해제 → 사이드바 탭은 가로 셀렉터로, 영역은 세로 스택(스테퍼 → 사이드바 → 작업대 → 관제탑).

## 2. flex-fill 고정 뷰포트 (매직넘버 금지·#141)

- 셸 전체 트리는 뷰포트에 **바운드**: `layout.tsx` 최상위 `height:100vh; overflow:hidden`,
  컬럼/`<main>` 각 `flex:1; min-height:0`. → `<main>`이 단일 스크롤 컨테이너.
- 아틀리에는 `height:100%`(= main 콘텐츠 박스)로 **flex-fill**. `calc(100vh - 매직넘버px)` 금지.
  헤더/푸터 높이가 바뀌어도 어긋나지 않음.
- 일반 페이지(dashboard 등)는 `<main>` 안에서 그대로 세로 스크롤. **글로벌 main 변경 시 전 페이지 회귀 셀프체크 의무**(검증: 가로 overflow 0 + 콘텐츠 전폭).
- 푸터(SD-01 아랍어 감사문)는 **불가침** — 위치·내용 변경 금지.

## 3. 색 토큰 (CSS 변수·중앙 클렌징 #142)

`globals.css :root` 정규 토큰:

| 토큰 | 값 | 용도 |
|---|---|---|
| `--brand-red` | `#E62310` | 액션 / 활성 (CTA·활성 탭·강조, ~10%) |
| `--pink-soft` | `#FFCCEA` | 틴트 / 뱃지 / 장식 보더 (~15%) |
| `--cream` | `#FAF8F3` | 배경 캔버스 (레일·보조면) |

- 제네릭 pink 하드코드 대신 토큰 사용(한 번 수정 = 스튜디오 전체 리스킨).
- **신호등 의미색은 리터럴 유지**: emerald(양호)·amber(검수)·red(보강). 의미색은 토큰화하지 않음.
- 기존 `--gp-*`/`--color-*` 디자인 시스템 토큰은 유지(충돌 0). `--pink-soft`/`--cream`은
  각각 `--gp-pink-200`/`--gp-cream-50`과 동값의 의미 별칭.

## 4. 반응형 4종 하드닝 (전상품·#144·#62)

모든 작업 화면(studio/crawl/products·new/dashboard)에 적용:

1. **(a) 텍스트**: 넘칠 수 있는 텍스트는 `truncate`(말줄임) 또는 `break-words`(긴 단어 줄바꿈).
2. **(b) 그리드 트랙**: 모든 그리드는 `minmax(0, 1fr)`. bare `1fr`/`repeat(N,1fr)` 금지
   (bare `fr`은 `min-width:auto`=min-content → 넓은 자식이 트랙을 가로로 폭발시킴; 실측 11506px).
   *Tailwind `grid-cols-*`는 이미 `minmax(0,1fr)` 내장 — 인라인 `gridTemplateColumns`만 교정 대상.*
3. **(c) flex/grid 자식**: `min-width:0`(=`min-w-0`) — 그래야 (a)(b)가 실제로 동작.
4. **(d) 인라인 display 금지**(#140): 반응형 표시 토글(`lg:hidden`/`lg:flex` 등)은 **클래스로만**.
   인라인 `style.display`가 Tailwind 반응형 유틸을 무력화함.

검증: 1440 / 1024 / 375 폭에서 가로 overflow 0(`scrollWidth === clientWidth`).

## 5. 신호등 (관제탑)

- 3단계: 양호(emerald) / 검수 권장(amber) / 보강 필요(red).
- 관제탑은 **발행 게이트의 시각화**일 뿐 — 새 점수 엔진 아님(#132). 개화도 0~100은 기존 준비도 판정 기반.

## 6. 꼬띠 카피 (마스코트 안내·#73)

- 꼬띠 = 튤립 마스코트, 공식 안내자(SD-08). 단계별 한 줄 가이드(말풍선) — 텍스트 벽 금지, 직관 우선.
- 한글 우선 라벨(음차 금지). 코드=영어, 표면=한글. 정착 외래어(누끼/합성/SEO/AI/ZIP/Firefly)는 유지.
- JSX 이모지 금지 → Lucide 아이콘만(#3-1).
