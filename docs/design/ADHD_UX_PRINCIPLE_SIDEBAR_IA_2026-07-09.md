## 1. ★ADHD 최적화 UX 원칙 (#233·전 UI/UX 적용)
================================================================
"빠른 선택·집중"을 위한 8대 규칙 — 앱의 모든 화면·컴포넌트에 적용:
1. **단일 초점(Single Focus)**: 한 화면에 primary 액션 1개만 시각 지배. 나머지는 secondary/명도 낮춤.
2. **점진적 공개(Progressive Disclosure)**: 지금 필요한 것만 노출·나머지는 접기/다음 단계. 한 번에 결정 ≤3.
3. **결정 부하 최소화**: 선택지 줄이기(힉의 법칙)·기본값 제공·"추천" 우선 표시. 빈 화면엔 "다음 1액션" 명시.
4. **명확한 다음 행동(Clear Next Action)**: 모든 화면이 "지금 뭘 하면 되는지" 한 문장으로. CTA 라벨은 동사+대상.
5. **일관·예측 가능**: 같은 기능=같은 위치·같은 색·같은 패턴. 놀라움 제거.
6. **기능 우선 라벨(Function-First)**: 은유는 보조·기능명이 주(예 "씨앗 심기(상품 등록)"). 아이콘+텍스트 병기.
7. **시각 청킹(Chunking·Cowan 4)**: 관련 항목 그룹화·그룹당 ≤4~5·여백으로 분리. 한 화면 정보 덩어리 ≤6.
8. **모션·장식 절제**: 자동재생/무한루프 금지·prefers-reduced-motion·주의 분산 요소 최소(#217/#228).
→ 판단 나침반: "이 화면을 3초 봤을 때 '지금 뭘 할지'가 즉시 보이는가?" (squint test + 3초 규칙)

## 2. 사이드바 IA 재감사 (현황)
================================================================
현재 ~20항목·5그룹·은유 위주 이름:
- **홈**: 정원일지(대시보드)·시장분석·성장·소싱·관제탑 (4)
- **화단·상품**: 꿀통 꽃나들이·씨앗 심기·정원 창고·검색 조련사·온실 아틀리에·좀비 부활소 (6)
- **수확·주문**: 주문 관리·상품 연동·인서트 카드 (3)
- **벌통·고객**: 카카오 채널 (1)
- **설정·도구**: 거래처 명단·배송 레시피·공급사 열쇠방·네이버 기본값·라이프 자산·… (6+)

**문제(ADHD 관점)**: 항목 과다(20)·은유만으론 기능 불명(좀비 부활소=?)·판단 부하. 유료 SaaS는 7±2 상위 메뉴.

## 3. 재편 제안 (합치기/빼기/명확화)
================================================================
**A. 기능 우선 라벨 + 은유 보조** (전 항목):
- 씨앗 심기 → "상품 등록(씨앗 심기)" · 온실 아틀리에 → "상세 디자인(아틀리에)" · 정원 창고 → "상품 창고" · 검색 조련사 → "SEO 진단(검색 조련사)" · 좀비 부활소 → "품절 대체(부활소)" · 꿀통 꽃나들이 → 기능 확인 후 명확화 · 인서트 카드 → 기능 확인 후.

**B. 병합 후보:**
| 병합 | 근거 |
|---|---|
| 씨앗 심기 + 온실 아틀리에 + 상품 창고 | 상품 생성 파이프라인 = "상품 만들기" 단일 진입(내부 단계 탭) |
| 상품 연동(수확·주문) → 화단·상품으로 이동 | 상품 sync는 주문 아닌 상품 도메인 |
| 거래처 명단 + 공급사 열쇠방 | 둘 다 공급사 관리 = "공급사 관리" 병합 |
| 배송 레시피 + 네이버 기본값 + 라이프 자산 | 설정류 → "설정" 하위 탭으로 |

**C. 그룹 재편(7±2 상위):**
```
홈           : 대시보드 · 시장분석 · 성장·소싱 · 관제탑
상품         : 상품 만들기(등록·디자인·창고) · 상품 목록 · 상품 연동 · SEO 진단 · 품절 대체
주문         : 주문 관리
고객         : 카카오 채널
공급사       : 공급사 관리(명단+열쇠방)
설정         : 배송·네이버 기본값·자산 등(탭)
```
→ 상위 그룹 6·각 그룹 항목 ≤5. 은유+기능 병기.
