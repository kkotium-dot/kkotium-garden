## 0. v5 결과 (운영자 확정)
- ★운영자: "대시보드 전체가 정원으로 보이는 컨셉 제대로 구현됨." → **v5 = 컨셉 방향 확정본.**
- 실측(PDF): 하늘/구름/잔디 언덕/울타리 배경·화단 화분 KPI·정원 팻말(오늘 할 일)·정원 시설물(온실/우체통/망루)·중앙 수익 분수대·꼬띠. #238 과감한 전환 성공.

## 1. 남은 문제 (운영자 지적 — v6 개선 대상)
1. **숫자 가독성 부족**: 정원 배경(연녹) 위 숫자가 대비 약하고 화분 카드가 배경에 묻힘.
2. **강조 폰트 과함**: 헤드라인·매출 숫자 등 굵은 디스플레이 폰트 과다.

## 2. Desktop 추가 발견 (수석 디자이너)
3. 분수대 매출과 KPI 3개 시각 위계 비슷 → 매출 1순위 강화·KPI 물러나게.
4. KPI 화분이 배경에 묻힘 → 옅은 화이트 배경+그림자로 "떠 있게".
5. 오늘 할 일 팻말↔분수대 세로 여백 큼 → 리듬 조여 밀도 최적화.
6. 영문 라벨(new orders 등)이 숫자보다 시선 끎 → 라벨 명도/크기↓·숫자 주인공.
7. 데이터 일관성 → 전 숫자 tabular·단위(건/원/점) 작게 분리 통일.

## 3. v6 개선 방향 (감성 유지·가독성만 향상)
- **유지(불변)**: v5의 정원 풍경 전체(하늘/잔디/화단/시설물/분수대/꼬띠/울타리)·팔레트·손그림.
- **개선**:
  · 모든 데이터 카드(KPI 화분·분수대 매출·요약)에 **반투명 화이트 패널**(rgba(255,255,255,.82) + backdrop-blur 약간)을 숫자 뒤에 깔아 정원 위에서도 또렷.
  · **숫자 = 깔끔한 산세리프(데이터 폰트)·크게·고대비 다크(#3A2E28)**·tabular. 라벨/단위는 작게·명도 낮춰 분리.
  · **강조 디스플레이 폰트는 타이틀(오늘의 정원/섹션명)에만**·매출 숫자는 산세리프 볼드로(디스플레이 아님)·굵기 한 단계↓.
  · 매출(분수대)을 최상위 위계로 크게·KPI 화분은 명도 살짝↓로 2순위.
  · 세로 여백 조여 1뷰포트 밀도 균형.
- ADHD 3초룰: 분수대 매출 + 오늘 할 일이 즉시 눈에·숫자 즉시 판독.

## 4. 절차
1. Claude Design v5 → v6 가독성 개선 프롬프트(감성 유지·데이터 패널/폰트만) → PDF/스샷 확인.
2. 확정 → standalone HTML → Desktop 토큰 추출(색·폰트·패널 규격·그림자) → Code handoff(#237 §4).
3. Code 구현: globals.css 토큰 + 대시보드 컴포넌트(반투명 패널·숫자 산세리프·라벨 절제)·실데이터 바인딩·저위험 단계·prod 검증.

## 작업 유의사항/원칙 (신규)
- **#239** 감성 강한 배경(정원 풍경 등) 위 데이터는 "반투명 화이트/뉴트럴 패널 + 고대비 산세리프 숫자 + 라벨 절제"로 가독성 확보. 브랜드 디스플레이(강조) 폰트는 타이틀/헤드라인에만·숫자와 데이터엔 깔끔한 산세리프. 감성(배경/일러스트)과 가독성(데이터 레이어)을 "레이어 분리"해 둘 다 살림 — 감성 때문에 데이터가 묻히면 실무 대시보드로서 실패. 전 데이터 표기는 tabular·단위 분리로 일관.
