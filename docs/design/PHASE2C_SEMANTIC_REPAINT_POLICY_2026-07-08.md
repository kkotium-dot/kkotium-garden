# Phase 2c 시맨틱 리페인트 — 정정 정책 (택소노미 보존) (2026-07-08)

Authoring: DESKTOP(정책 확정). 구현=Code. 권위: 본 문서 + KKOTIUM_BRAND_PALETTE §1 + Code 착수 전 발견(택소노미 붕괴 리스크). 원칙 #221/#224/#62. 비게이트.
계기: Code Phase 2c 착수 중 발견 — 시맨틱 헥스 상당수가 일반 semantic이 아닌 의도적 다색 도메인 택소노미(orders 8색·관제탑 STATUS/OVERALL/MODE/LINE/TIER·꿀통 등급). 플랫 4토큰 매핑 시 상태 구분 붕괴.

================================================================
## 1. 정정 매핑 정책 (확정)
================================================================
**A. 토큰화 (일반 단일의미 시맨틱만):**
- 저장 완료·성공 토스트 → --success/-bg
- 에러 메시지·실패 → --danger/-bg
- 경고·주의 배너 → --warning/-bg
- 정보 안내 배너 → --info/-bg
- 단순 정상/실패 배지·재고정상 등 이분 상태 → 해당 토큰
- 컨텍스트별: 배경틴트→-bg · 보더→토큰 · 소형 텍스트→잉크 유지(대비)

**B. 보존 (도메인 다상태 택소노미·N>4 상태·4토큰 금지):**
- 주문 상태(결제대기/완료/배송중/배송완료/구매확정/교환/취소/반품 — 8색)
- 관제탑 STATUS(done/in_progress/pending/blocked/awaiting_human/none)·OVERALL(risk/attention/caution/ok/none)·MODE·LINE·TIER·CATEGORY
- 꿀통 등급(S/A/B/C/D)·라이프사이클 스테이지
- → 이 색들은 "의미 구분"이 목적이라 그대로 유지(트렌디 통일은 별도 과제·§3).

================================================================
## 2. 페이지군 순서 (안전 우선)
================================================================
1. **일반 시맨틱 밀집 페이지 먼저**: settings 폼(저장/에러)·각 페이지 toast/배너·단순 배지.
2. **택소노미 밀집 페이지(orders·관제탑)는 나중·극도 주의**: 해당 페이지에서도 택소노미 색은 건드리지 않고, 확인된 일반 시맨틱 인스턴스만 토큰화.
3. 페이지군별 push → Desktop 페이지별 검증(상태 구분 보존·대비·브랜드 충돌0).

================================================================
## 3. 별도 과제 (플래그·Phase 2c 아님)
================================================================
- **트렌디 택소노미 팔레트**: 도메인 다상태 색(주문상태·관제탑 등)을 트렌디하게 통일하려면, N색 상태 팔레트를 응집성+구분성+온브랜드로 "설계"하는 별도 디자인 작업 필요(단순 토큰 치환 아님). 운영자가 "상태 색도 트렌디하게" 원하면 별도 리서치/설계로.

================================================================
## 작업 유의사항/원칙 (신규)
================================================================
- **#224** 시맨틱 토큰화는 **의도적 다상태 도메인 택소노미(N>4 상태)를 보존** — 일반 단일의미 시맨틱만 4토큰(success/warning/danger/info)으로 수렴. 상태 구분이 목적인 색을 4토큰으로 뭉개면 정보 손실(#62 systemic이되 의미 보존 우선). 다상태 색의 트렌디 통일은 별도 설계 과제.
