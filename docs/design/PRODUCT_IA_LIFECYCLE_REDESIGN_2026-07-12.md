# 상품 IA 전면 재설계 — 라이프사이클 파이프라인 (승인 대기 브리핑) (2026-07-12)

Authoring: DESKTOP(IA 실측+운영자 요청+Gemini 참고 종합). 원칙 #55/#56/#232/#233/#245/#251. 상태: ★브리핑·운영자 승인 후 단계별 구현. 미구현.

## 0. 진단 — 근본 문제
- 상품 하나의 생애가 **7개 메뉴(소싱·등록·디자인·창고·연동·SEO·품절대체)에 파편화** → "이 상품 지금 어디서 만지지?" 반복 = ADHD 인지과부하.
- 모든 기능 페이지에 **모든 상품이 다 나옴**(역할 구분 없음).
- 필터 탭 과다(정원창고 6~10개)·리스트 라인 안 맞음·가독성 저하.
- "품절 대체" 라벨이 실제 기능(좀비부활소 리셋/튜닝)과 불일치.
- 연동 상품 정보 빈약·품절대체 정보 입력 미숙.

## 1. 핵심 재설계 — 상품 라이프사이클 3단계 파이프라인
기능(feature) 나열 → **워크플로우 단계(stage)** 로 재편. 각 단계 = 정원 은유 + 그 단계 상품만 등장(역할 기반 버킷).

| 단계 | 메뉴(정원 은유) | 하는 일 | 등장 상품(버킷) |
|---|---|---|---|
| ① 발굴 | **꿀통 꽃나들이**(소싱) | 팔 신상품 발굴 | 소싱 후보(앱 밖) |
| ② 가꾸기 | **씨앗심기**(등록+디자인+창고 통합) | 앱에서 제작·편집·보관 | APP_CREATED·편집중·미등록 |
| ③ 부활 | **좀비부활소**(SEO진단+품절대체 통합) | 이미 네이버 등록 상품 진단·리셋·튜닝·부활 | IMPORTED(네이버 등록)·부활 대상 |

- **연동(Import)** = 별도 메뉴가 아니라 **③ 좀비부활소의 "네이버 상품 가져오기" 진입점**(#244: 부활 대상 선별 import→튜닝→재업로드).
- **SEO 진단**은 독립 메뉴 폐지 → ③ 좀비부활소의 핵심 진단 기능으로 흡수(운영자 요청). ②에서도 상품명 진단은 인라인 배지로 제공(작성 중 도움).

→ 결과: 좌측 메뉴 상품 그룹이 **7개→3개(소싱/씨앗심기/좀비부활소)**. 역할 명확·오가기 제거.

## 2. 운영자 요청 11건 대응
1. "품절 대체" 라벨 오류 → **"좀비부활소"**(리셋·튜닝) 정명. sub 은유 유지.
2. SEO진단+품절대체 통합 → ③ 좀비부활소 단일 기능.
3. 소싱/창고/연동 기준 애매 → 라이프사이클 버킷으로 역할 확정(위 표). 연동=부활소 진입점.
4. 리스트 가독성/라인 → 표→**카드 그리드**(썸네일·상품명·상태·다음 액션 1개)·정렬 통일(연동 카드화 패턴 확장·#233).
5. 필터 탭 과다(10개) → **핵심 3개 + 나머지 드롭다운 접기**. 정원창고: [전체 / 작성중 / 발행대기] 상단·그 외(네이버판매중·등록대기·품절 등)는 "상태 더보기" 드롭다운.
6. 부활소=네이버 등록 상품만 → 역할 버킷(IMPORTED만). ②는 미등록/앱생성만. 각 기능 자기 상품만.
7. 연동 페이지네이션 → 하단 **`< 1 2 3 4 … >` 번호 페이지** 추가.
8. 연동 상품 정보 빈약 → 연동 시 네이버 원본 필드 최대 수집 + 부족분 표시(무엇을 채워야 하는지 #240).
9·10. 품절대체·연동 상품 관리 흐름 → **씨앗심기 편집 화면에서 3-레이어 관리**:
   - [네이버 원본 정보] 읽기전용(공급사/네이버 원본)
   - [업데이트할 정보] 앱 개선분(상품명·이미지·가격 = 재업로드 대상)
   - [품절대체 상품 정보] 이 상품 품절 시 대체 상품 지정·연결
   - 탭/아코디언으로 분절(스크롤 압박 차단·#233).
11. 배지 숫자 의미: sourcing=소싱후보수·draft=작성중수·zombie=부활대상수·orders=신규주문. → **행동 유발되는 것만 유지**: orders(신규주문)·zombie(부활 시급)만. **sourcing·draft 배지 제거**(모호·피로 유발).

## 3. 나의 추가 아이디어 (시니어)
- **"다음 할 일" 액션 버튼**: 각 상품 카드에 앱이 짚어주는 다음 행동 1개(이미지 없음→디자인·발행준비 완료→발행·부활 대상→튜닝). "뭐 하지?"를 앱이 대신 판단(#240·행동 유도).
- **상품 = 단일 작업 공간(포커스 모드)**: 상품 클릭→그 상품의 모든 작업(편집·디자인·SEO·발행)을 한 맥락(풀스크린/사이드패널)에서. 메뉴 오가기 제거(Gemini 포커스 모드 반영·All-in-one은 회피).
- **관제탑 = 상품 아닌 "오늘 할 일" 허브**: 개별 기능 카운트는 관제탑/대시보드에 모으고, 사이드바 배지는 최소화(피로↓).
- **정원 은유 정합**: 발굴(꿀통)→가꾸기(씨앗·온실)→부활(좀비부활소). 라이프사이클이 은유와 자연 일치.

## 4. 단계별 실행 계획 (승인 후·리스크 분리)
- **P1 (라벨·배지·정리·저위험)**: 품절대체→좀비부활소 리네임(라우트 보존)·sourcing/draft 배지 제거·필터 탭 3+드롭다운 축소. 즉시 체감·회귀 낮음.
- **P2 (통합)**: SEO진단 메뉴 폐지→좀비부활소 흡수·연동을 부활소 진입점으로·역할 버킷(각 기능 자기 상품만).
- **P3 (리스트 UX)**: 전 목록 카드 그리드 통일·정렬·다음 액션 버튼·연동 페이지네이션.
- **P4 (3-레이어 편집)**: 씨앗심기 편집에 네이버원본/업데이트/품절대체 레이어·품절대체 상품 지정 구조화.
- 각 P: tsc0/build0→브라우저 실측→배포→Desktop 검증. 라우트 리네임 0(리다이렉트 불요) 원칙 유지.

## 작업 유의사항/원칙 (신규)
- **#254** 상품 관리 IA는 기능(feature)별 분리가 아니라 **상품 라이프사이클 단계(발굴→가꾸기→부활)** 로 구성 — 각 단계는 자기 역할의 상품만 보여주고(역할 버킷·#245), 상품 클릭 시 그 상품의 모든 작업을 한 맥락(포커스 모드)에 모아 메뉴 오가기를 제거(#233 ADHD). 필터 탭·배지는 행동 유발되는 최소만 노출(나머지 드롭다운/관제탑). 목록은 표 나열 대신 카드+다음 액션 1개(행동 유도). 대규모 IA 변경은 라벨/정리(저위험)→통합→UX→데이터 순 단계 분리·라우트 보존.
