# 꽃틔움 KKOTIUM — 하이브리드 컨셉 프리셋 시스템 (CONCEPT_PRESET_SYSTEM.md)

> 저장: 2026-06-03 | 권위 문서 (모든 상품의 디자인 컨셉을 결정하는 핵심 엔진 명세)
> 짝 문서: KKOTIUM_DESIGN_SYSTEM.md (브랜드 코어 토큰) + DETAIL_PAGE_PLAYBOOK.md (제작 SOP)
> 핵심 원칙: 고정 코어(꽃틔움다움)는 절대 불변, 가변 레이어(상품 컨셉)는 카테고리 성격에 맞춰 강도 조절.
> 해결하는 문제: "모든 상품을 한 톤(예: 내추럴 미니멀)으로 밀면 셀렉트샵 큐레이션 가치가 죽는다."

---

## 0. 한 문장 요약
**브랜드 코어는 고정하고, 상품 컨셉은 3단계 강도(Level 1~3)로 갈아끼우는 프리셋 시스템.**
상품 등록 시 카테고리 → 프리셋 자동 추천 → 최소 수정으로 완성. 디자인을 매번 0에서 고민하지 않는다.

---

## 1. 3계층 토큰 구조 (업계 표준: primitive → semantic → component)
| 계층 | 정의 | 꽃틔움 예시 | 변동성 |
|---|---|---|---|
| **Primitive** | 맥락 없는 원시값 (색 팔레트 풀, 스페이싱 스케일) | 전체 색 풀, 4px 베이스 간격, Pretendard 패밀리 | 리브랜딩 때만 |
| **Semantic** | 의미·용도 (테마 전환이 일어나는 곳) | `--color-surface`, `--color-accent`, `--copy-tone` | **프리셋별 가변** |
| **Component** | 컴포넌트별 사용 | 섹션 배경, 헤드라인 스타일, CTA 버튼 | semantic 참조 |

핵심: **컴포넌트는 원시 색(hex)을 절대 모른다.** CTA 버튼은 `--color-action-primary`만 참조하고, 그 값이 어스톤이냐 단청 적색이냐는 활성 프리셋이 결정한다. (= 테마 교체가 "수백 개 수정"이 아니라 "한 줌 교체"가 되는 이유)

---

## 2. 고정 코어 vs 가변 레이어 경계 (절대 기준)
| | 고정 코어 (절대 불변) | 가변 (프리셋별) |
|---|---|---|
| 컬러 | 꽃틔움 시그니처(Brand Red #E62310 / Pink #FFCCEA) | surface·accent·text 무드 컬러 |
| 타이포 | Pretendard 패밀리, 본문 16px+ | 헤드라인 세리프/산세리프 선택, 여백 밀도 |
| 구조 | 로고 영역, 가격/CTA 블록, 리뷰 블록, 그리드 베이스 | 섹션 순서·히어로 형식(Level 3만) |
| 마스코트 | 꼬띠 (변형 금지) | 등장 빈도·톤 |
| **SEO** | **상품명 50자, 대표이미지 1:1 화이트배경, 카테고리 정확도 — 전 상품 동일** | (SEO는 프리셋과 완전 독립 = 직교) |

규칙: 어떤 프리셋을 써도 위 "고정 코어"는 절대 안 바뀐다. 이게 "꽃틔움다움" + 네이버 신뢰도를 지킨다.

---

## 3. 가변 강도 3단계 (시스템의 심장)
| 단계 | 변하는 것 | 고정되는 것 | 적용 카테고리 |
|---|---|---|---|
| **Level 1 (절제)** | 팔레트(accent·surface)만 | 레이아웃·카피톤·이미지스타일·타이포 | 주방/생활, 디지털/가전 |
| **Level 2 (중간)** | 팔레트 + 카피톤 + 이미지 필터/프레임 + 여백 밀도 | 레이아웃 구조·타이포 시스템 | 패션잡화, 반려동물 |
| **Level 3 (강한)** | 팔레트 + 카피톤 + 이미지 + 여백 + 레이아웃 구조(섹션순서·히어로) | 로고·시그니처색·SEO·그리드 베이스만 | 향/아로마, 선물/기프트, 전통/공예 |

원리: **감성 소비재 = 강한 가변(Level 3), 실용 소비재 = 절제된 가변(Level 1).** 상품이 요구하는 만큼만 변주한다.

## 4. 카테고리별 프리셋 정의 (무드 기준 명명 — 29CM 방식)
| 프리셋 | Level | 팔레트 무드 | 카피 톤 | 이미지 스타일 | 예시 상품 |
|---|---|---|---|---|---|
| **aroma** (향·아로마) | 3 | 세이지·스톤·웜크림·테라코타 (자연 차분) | 감성 내러티브 (향→장면 번역) | 연출컷·여백·무광 | 명화 차량용 디퓨저 |
| **gift** (선물·기프트) | 3 | 웜 파스텔·골드·앰버 | 설렘·따뜻함 | 언박싱·리본·플로럴 | 기프트 공통 |
| **tradition** (전통·공예) | 3 | 오방색·단청·한지·먹 톤 | 정서·이야기 | 한지 질감·여백의 미 | 달항아리 도어벨 |
| **kitchen** (주방·생활) | 1 | 청결 화이트·블루 | 기능 불릿·명료 | 누끼·스펙컷 | 아이스트레이 |
| **pet** (반려) | 2 | 친근 웜톤 | 신뢰·친근 | 라이프스타일컷 | 반려용품 |

각 프리셋은 6요소를 정의: ①팔레트 ②폰트페어링(브랜드 범위 내) ③여백 밀도 ④이미지 스타일 ⑤카피 톤 ⑥레이아웃 변형(Level 3만).

## 5. 천편일률 방지 — 커스터마이징 슬롯
프리셋이 너무 경직되면 "공장 양산"처럼 보인다. 해법: 프리셋당 **자유 변수 2~3개만** 허용.
- 허용 슬롯: accent 색 1개 오버라이드 / 히어로 카피 자유입력 / 대표 무드이미지 교체
- 잠금(슬롯 제외): 로고, 시그니처색, 가격/CTA 블록, SEO 필드
- 원칙: "엄격한 유연성(Rigid Flexibility)" — 가드레일은 평소 안 보이고, 선 넘을 때만 부딪힌다.

## 6. 앱 구현 명세 (Next.js + Tailwind + Supabase)
### 6-A. 데이터 모델 (Supabase products 확장)
```
concept_preset    text   default 'kitchen'   -- 'aroma'|'gift'|'tradition'|'kitchen'|'pet'
preset_intensity  text   default 'l1'        -- 'l1'|'l2'|'l3'
preset_overrides  jsonb                       -- {accent:"#...", hero_copy:"..."} 커스터마이징 슬롯
-- SEO 필드는 프리셋과 분리된 고정 컬럼 (seo_title, naver_category_id, attributes 등)
```

### 6-B. CSS 변수 + data-attribute (런타임 테마)
```
:root { --brand-signature:#E62310; --font-brand:"Pretendard"; }   /* 고정 코어 */
[data-preset="aroma"]    { --surface:#f3efe7; --accent:#76864c; --text:#3a352e; }
[data-preset="tradition"]{ --surface:#f4ece0; --accent:#9b2d30; --text:#1c1c1c; }
[data-preset="kitchen"]  { --surface:#ffffff; --accent:#2f6fb0; --text:#1a1a1a; }
```
렌더: `<article data-preset={preset} data-intensity={level}>` → 하위 전체 상속.

### 6-C. 컴포넌트 variant (CVA)
7섹션 컴포넌트를 preset × intensity 2축 variant로 코드화. 타입 안전 + 재사용.

### 6-D. 점진 전략 (과잉엔지니어링 금지)
- 프리셋 3개 이하 → 수동 CSS 변수로 충분.
- 프리셋 5개+ → Style Dictionary(JSON 토큰 → CSS/Tailwind 자동변환) 도입.
- 토큰 계층 2~3단계면 충분. 4단계 이상 금지(혼란·디버깅 비용).

## 7. SEO 직교 원칙 (디자인이 검색을 해치지 않게)
디자인 컨셉(상세페이지 내부 비주얼)과 SEO(상품명·대표이미지·카테고리·속성)를 **데이터 모델에서 분리**.
- 프리셋 = 상세페이지 body 비주얼만 제어.
- SEO 필드 = 프리셋 무관, 전 상품 동일 검증: 상품명 50자 린터 / 대표이미지 1:1 화이트배경·홍보문구 금지 가드 / 카테고리 정확도.
- 대표이미지는 항상 "깔끔한 배경 + 본품만"(컨셉 무관). 무드는 상세페이지(가로 860px) 안에서만 펼친다.
- 네이버 2024-10-28 대표이미지 기준 강화: 외곽선·도형·홍보문구("무료배송"·"1위") 삽입 시 제재.

## 8. 상품 등록 워크플로우 (프리셋 자동 적용)
1. 상품 등록 → 카테고리 선택
2. 카테고리 → **프리셋 + Level 자동 추천** (aroma=L3, kitchen=L1 등)
3. 프리셋 기본값으로 7섹션 자동 조립
4. 커스터마이징 슬롯 2~3개만 손봐서 완성
5. SEO 가드 자동 검증 (프리셋 독립)
6. 발행 게이트 → 대표 승인 → 발행

ROI: "디자인 결정 시간" 제거. 일관성(브랜드 신뢰=네이버 신뢰도) + 생산속도(상품 수=노출 기회) 동시 확보.

---

## 9. 의사결정 임계값
- 카테고리 전환율이 평균(~2%) 미달 → 해당 프리셋 Level 한 단계 상향(차별화 강화).
- 상세페이지 제작 상품당 2시간 초과 → 커스터마이징 슬롯 축소, 프리셋 기본값 강화.
- 브랜드 일관성 혼란 신호 → 가변 강도 한 단계 하향, 고정 코어 비중 확대.
- 프리셋 5개+ → Style Dictionary 자동화 도입.

---

## 10. 앱 통합 로드맵
- **Phase A (즉시)**: 토큰 경계 확정 + globals.css에 :root(코어)+[data-preset](가변) 정의 + 프리셋 4개(aroma/gift/tradition/kitchen).
- **Phase B (단기)**: 7섹션 CVA variant 코드화 + Supabase 컬럼(concept_preset/intensity/overrides) 추가 + SEO 가드 린터 내장.
- **Phase C (중기)**: Style Dictionary 파이프라인 + 프리셋별 A/B 전환율 측정 → Level 데이터 기반 조정.

트리거: 명화 디퓨저(aroma L3)를 첫 레퍼런스로 완성·검증 → 프리셋 패턴 확정 → 코드화.
