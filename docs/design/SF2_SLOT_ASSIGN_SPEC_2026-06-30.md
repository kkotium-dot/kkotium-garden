# SF-2 — 슬롯 배정 + 영속 스펙 (2026-06-30)

Authoring: DESKTOP. 권위: SLOT_FUNNEL_BOARD_SPEC_2026-06-30.md(v1.1) §3 SF-2 상세화. 구현=Code. 원칙 #55/#62/#181/#46.
전제: SF-1(읽기 보드·DetailAssemblyBoard) DONE·prod검증. 운영자 Q2=**(a) detail_images 순서 재사용**(신규 컬럼 0) 승인 · Q3=진행 · Q4=in-app MVP.

================================================================
## 0. 계약 실측 (verify-first #181)
================================================================
- `Product.detail_images` = **`string[]`**(상세페이지 이미지 URL 평면 배열). jsonb.
- 쓰기/읽기 공유처(현재):
  1. 씨앗심기 serialize: `detail_images: v.detailImages`(string[]) — product-form-mapping.ts
  2. 씨앗심기 hydrate: `dto.detail_images.join(',')`
  3. buildDetailContent: detail_images를 순서대로 detailContent HTML로 emit(발행)
- → **계약은 "순서 있는 URL 평면 배열"**. SF-2는 이 계약을 **바꾸지 않고 재사용**(zero-ripple).

================================================================
## 1. 설계 결정 (Q2=a 충실 — 평면 순서 영속)
================================================================
- **섹션 = 편집 모델(in-session)** · **detail_images = 평면 순서 출력**.
- SF-2 보드에서 자산을 7섹션 슬롯에 배정 → **저장 시 섹션 순서대로 평탄화**(hook→value→spec→usage→trust→cta; **notice 제외**=store_settings 핀) → `detail_images: string[]`로 영속(기존 계약 그대로).
- buildDetailContent·씨앗심기 **무변경** → #62 리더 일괄수정 불요·마이그레이션 0·발행 배관 그대로.
- ★MVP 한계(명시): **섹션 배정 자체는 영속 안 됨**(순서만). 재오픈 시 보드는 detail_images를 **"현재 조립됨(순서)" 스트립**으로 복원 표시 + 7섹션은 재구성 작업면. 진짜 섹션 복원이 필요하면 = SF-3에서 경량 slot_assignment 필드(Q2-b 승격)로.

================================================================
## 2. 구현 (DetailAssemblyBoard 확장 — 읽기→배정)
================================================================
- **자산 트레이(좌)**: 기존 SF-1 그대로(8스테이지 그룹).
- **섹션 슬롯(우)**: 각 섹션에 자산 **클릭-추가/드래그-드롭**(MVP는 클릭-추가가 단순·견고). 슬롯 내 자산 순서 변경(위/아래) + 제거.
- **현재 조립됨 스트립**: detail_images(영속본)를 순서대로 썸네일 표시 — 진실원천 가시화.
- **저장**: 섹션 배정 → 평탄화 → autosave(C-1 패턴 재사용·debounce·silent·savingRef 직렬화) → PUT detail_images. 명시 저장 버튼 불요(자동).
- **재오픈**: detail_images → "현재 조립됨" 복원. (섹션 슬롯은 빈 작업면 또는 stage 기반 best-effort 제안 — MVP는 빈 작업면 + 안내.)
- 발행 무관(#46) — 조립≠발행. detail_images 갱신은 가역(발행 PUT은 별도 GO).

================================================================
## 3. 검증 기준 (Desktop·prod·#45)
================================================================
1. 배양실+상세 스텝 보드에서 자산을 섹션에 배정 → **autosave 칩**(저장됨) 확인.
2. GET /api/products/{id} → **detail_images가 섹션 순서대로 갱신**(평면 string[]) 확인.
3. 재오픈(?product=&tab=image) → "현재 조립됨" 스트립이 저장 순서로 복원.
4. 씨앗심기(?edit=) 상세 이미지 영역이 **여전히 동일 detail_images 표시**(계약 무파손) — #62 교차 확인.
5. 콘솔 0 · ErrorBoundary 폴백 미발동 · 테스트 변경 정리(#177, 명화는 실데이터라 신중—테스트는 격리 상품 권장).

## 4. 작업 유의사항/원칙 (신규)
================================================================
- **#184** 공유 컬럼(detail_images) 재사용 기능은 **계약 shape를 바꾸지 말 것** — 바꾸면 전 리더(씨앗심기 serialize/hydrate·buildDetailContent) 일괄수정 필요(#62 리스크). SF-2는 평면 순서 유지로 ripple 0.
- **#185** "순서 재사용"형 영속은 구조(섹션) 복원을 보장하지 않음 — 구조 복원이 요구되면 별도 경량 필드로 승격(아키텍처 결정은 운영자 락).
- 두 입력 surface(씨앗심기 dropzone ↔ SF-2 보드)가 같은 detail_images를 쓰므로, **마지막 저장이 승**(평면 순서). 워크플로 권장: 씨앗심기에서 초기 상세이미지 → 스튜디오 SF-2에서 순서·구성 정교화.
