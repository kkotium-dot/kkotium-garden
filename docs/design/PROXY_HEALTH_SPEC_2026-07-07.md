# PROXY-HEALTH — 네이버 연동 헬스체크 + 관제탑 경보 (2026-07-07)

Authoring: DESKTOP. 구현=Code. 승인: 운영자 명시 승인(2026-07-07). 권위: 본 문서 + naverRequest/NaverApiError classifier + 대시보드 알림 서피스(ORDER-QUEUE-1 패턴). 원칙 #62/#82/#181/#56.
배경: 네이버 프록시(홈 Mac Tailscale Funnel)가 세션 중 조용히 다운(ECONNRESET) → 주문·상품 동기화 전부 멈췄으나 앱은 무표시. **프록시 = 모든 네이버 연동의 단일 장애점(SPOF)**.

================================================================
## 0. 목적 (왜)
================================================================
네이버 연동(프록시/토큰/IP) 장애를 **주기·on-demand로 감지해 관제탑에 경보 카드로 표면화** → 1인 셀러가 "왜 동기화가 안 되지?"를 겪기 전에 앱이 먼저 알림. 조용한 실패 → 가시적 상태로.

================================================================
## 1. 헬스체크 엔드포인트
================================================================
- **GET `/api/naver/health`** — 경량 네이버 호출 1회(예: `products/search` size=1, 또는 최경량 계정/채널 조회)로 프록시+토큰+IP 생존 확인. 짧은 타임아웃.
- 반환: `{ healthy:boolean, errorClass?, message?, checkedAt }`.
- **errorClass 분류**(기존 NaverApiError classifier 재사용·#181): 
  - `PROXY_DOWN`(ECONNRESET/타임아웃 = 프록시 불통) 
  - `IP_NOT_ALLOWED`(국내 IP 아님) 
  - `AUTH_SIGN_INVALID`(토큰/서명) 
  - `RATE_LIMIT`(429) 
  - `NAVER_ERROR`(기타).
- **캐시**: 결과 60초 캐시(레이트리밋·부하 방지). 강제 재검사 `?fresh=1`.

================================================================
## 2. 관제탑 경보 (표면화)
================================================================
- 대시보드 **시스템 상태 배너/카드**(받은편지함 상단 또는 관제탑) — healthy=false일 때만 노출(정상 시 미표시·노이즈 0).
- 문구(errorClass별·#3-1 i18n):
  - PROXY_DOWN: "네이버 연동 끊김 — 홈 프록시(맥/Tailscale) 확인 필요"
  - IP_NOT_ALLOWED: "네이버 연동 차단 — 국내 IP 재등록 필요"
  - AUTH_SIGN_INVALID: "네이버 인증 만료 — 토큰/키 확인"
  - RATE_LIMIT: "네이버 호출 한도 — 잠시 후 자동 복구"
- checkedAt 상대시각 표시. 비강제·경보성(#56).
- ★엔티티: 시스템 전역 상태 → 상품 C-9 매트릭스 아님, 대시보드 서피스(#195 연장).

================================================================
## 3. 트리거/주기
================================================================
- **on-demand**: 대시보드 로드 시 `/api/naver/health`(60초 캐시) → 끊김이면 배너. (Vercel cron daily 상한이라 실시간성은 on-demand가 유리·#194.)
- (옵션) 기존 cron(order-sync/daily)이 프록시 실패를 감지하면 상태 저장 → 대시보드가 읽기. 신규 로깅 테이블 없이 경량 상태(메모리/기존 테이블 1행)로.

================================================================
## 4. 검증
================================================================
- Code(라이브): 프록시 정상 시 healthy=true·프록시 다운 시 PROXY_DOWN 분류 확인. tsc0/build0.
- Desktop(prod): **현재 프록시 다운 상태가 마침 테스트 케이스** — health가 PROXY_DOWN 반환·대시보드 배너 노출 확인. 복구 후 healthy=true·배너 소멸 확인.

================================================================
## 작업 유의사항/원칙 (신규)
================================================================
- **#204** 외부 의존성 SPOF(프록시/외부 API)는 **헬스체크 + 가시적 경보 필수** — 조용한 실패는 1인 셀러에게 치명(동기화 멈춤을 모름). 감지→표면화→가이드 3단.
- errorClass별 구체 가이드로 "무엇을 고쳐야 하는지"까지 안내(단순 "오류" 금지).
- 정상 시 미표시(노이즈 0)·비강제(#56).
