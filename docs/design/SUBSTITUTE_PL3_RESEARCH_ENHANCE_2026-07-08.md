Authoring: DESKTOP. 구현=Code. 권위: STOCKOUT_SUBSTITUTE_RESEARCH_2026-07-08 + SUBSTITUTE_STOCKOUT_SPEC + PL3_PRICE_STOCK_PUSH_SPEC. 원칙 #56/#211/#3-1. 비게이트(문구·체크리스트·기본값).
목적: 리서치 실전로직을 이미 빌드된 SUBSTITUTE + 대기중 PL-3에 얹어 "실무 사고 방지(페널티·순위리셋·불일치 클레임)" 수준으로.

## 1. SUBSTITUTE 강화 (빌드됨 → 개선 얹기)
================================================================
1. **전환 전 확인 체크리스트 팝업**(대체상품 전환/발행 직전): 
   "□ 같은 카테고리인가요? □ 상품명 핵심키워드를 유지했나요? □ 대표이미지를 크게 바꾸지 않았나요? □ 동급·상위호환인가요?" 
   — 미체크 시 "순위 리셋·상품불일치 신고 위험" 경고(#211). Lucide·i18n(#3-1).
2. **품절 카드 단계별 액션 가이드**(substitute_ready 카드 확장): 품절 감지 시 "①옵션 일부 품절 ②재입고/재소싱 ③7~10일이면 예약구매 전환 ④대체상품 전환" 순서 안내(랭킹 영향 적은 순).
3. **임계 기본값 10**(현재 5 → 리서치 기본값 10 or 5일치). 사용자 조정 가능. 툴팁: "재고 10개 또는 5일치 판매량 권장".
4. **페널티 안내**(품절 카드): "주문 취소 시 판매관리 페널티 2점(선물하기 3점) — 취소보다 옵션 품절처리/예약전환 권장" + "최신 기준은 스토어센터 확인" 링크(하드코딩 금지·#211).

================================================================
## 2. PL-3 강화 (빌드 시 얹기)
================================================================
1. **"네이버 자동반영 안 됨" 경고**(재고/가격 push UI): PL-3는 dry-run+GO로 실제 PUT하지만, 수동 반영 특성상 "반영 후 스토어센터에서 실제 노출 확인" 안내.
2. **재고 반영 5단계 체크리스트**(재고 push 확인 모달): "□ 판매가·할인가 □ 옵션별 재고 □ 마진(역마진?) □ 배송비·반품비 □ 반영 후 노출확인".
3. **역마진 경고**: 가격 push 시 supplierPrice 대비 salePrice가 마진 임계 이하면 "역마진/저마진 경고".
4. 재고 push 시 "네이버 실재고 덮어씀 — 진행중 주문 차감분 확인"(기존 스펙 유지).

## 3. 별건 정리 (Code 발견 반영)
================================================================
- `product_alternatives` 테이블 prod 부재 → 기존 AlternativeProductPanel(가격카탈로그) 저장 무효. 
- 결정: (a) AlternativeProductPanel을 substitute_info 기반으로 통합/대체, or (b) product_alternatives 테이블 생성. 
- 권고: **substitute_info로 일원화**(중복 방지·#55) → AlternativeProductPanel의 유효 필드(공급가/판매가 카탈로그)를 substitute_info 스키마에 흡수하거나 제거. 운영자 확인 후.
