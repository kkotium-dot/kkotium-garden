# KPI 시계열 backend — 일별 집계 엔드포인트 (2026-07-08)

Authoring: DESKTOP. 구현=Code. 권위: 본 문서 + DASHBOARD_SHELL_REDESIGN_RESEARCH(KPI 큰숫자+델타+스파크라인) + Order 스키마 실측. 원칙 #82/#45/#181. 비게이트.
목적: KPI 4카드의 델타(전일 대비)+스파크라인 실구현. 현재 stats API 시계열 없어 "큰숫자만"이던 것 완성.

================================================================
## 0. 실측 (Supabase·#181)
================================================================
- Order: payment_date(timestamp)·totalAmount(int)·totalPrice(int)·status(text) 존재.
- 데이터: 24건·payment_date 전건·2026-04-07~2026-07-05·17 distinct days. → 일별 aggregate 가능(신규 daily_metrics 테이블 불필요·Order 직접 GROUP BY).

================================================================
## 1. 엔드포인트
================================================================
`GET /api/dashboard/metrics-timeseries?days=14`
- SQL: `SELECT payment_date::date d, SUM("totalAmount") rev, COUNT(*) cnt FROM "Order" WHERE payment_date >= now()-interval 'N days' GROUP BY d ORDER BY d`.
- 응답: `{ series:[{date, revenue, orderCount}], filledDays:N }` — 누락일 0 채움(연속 N일).
- ★정직(#82): 실 데이터만. 주문 없는 날 = revenue 0(실값·가짜 아님). 데이터 없으면 빈 series+honest 표시.
- guarded: Order 조회 실패 시 success:false(가짜 시계열 금지).

================================================================
## 2. KPI 카드 소비 (KpiStrip)
================================================================
- **오늘 매출**: 오늘 payment_date sum + **델타**(전일 대비 ↑↓% or ₩차) + **스파크라인**(최근 14일 일별 revenue).
- **신규 주문**: 오늘 orderCount + 델타 + 스파크라인(일별 cnt).
- 정산 예정·품절 경보: 시계열 무관(현행 유지).
- ★단, 오늘 매출/주문은 로컬 Order 기반이라 **네이버 다운과 무관하게 실값**(0 포함). 기존 "—"(네이버 degrade)와 구분: 로컬 집계는 항상 실값, 네이버 의존分만 degrade.
- 스파크라인=경량 인라인 SVG(라이브러리 0)·색 --brand-red or --brand-pink-mid·델타 상승=--success/하락=--danger(트렌디 시맨틱).
