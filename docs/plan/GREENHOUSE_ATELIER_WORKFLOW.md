## 0. 핵심 철학 (대표님 교정 반영)

> 각 엔진은 **결정자가 아니라 제안자**다.
> 자동화가 결정을 대신하는 게 아니라, **각 단계 결과를 대표님이 컨펌하며 흐른다.**
> 디자이너의 눈이 매 길목에 살아있다.
> "자동화가 감각을 대체하는 게 아니라, 감각을 펼칠 시간을 벌어준다."

**중요 — 분기 기준 폐기**:
이전 "길 A/B 자동 분기" 설계는 폐기. 디렉팅은 사전 분기가 아니라
*각 엔진 결과를 보고 대표님이 컨펌*하는 구조. 컨펌 없이 다음 단계로 넘어가지 않음.

대표님이 짚은 사실: **네이버 상세페이지는 HTML 직접 등록 가능**
→ 클로드 디자인 HTML 출력물을 마무리 단계에서 그대로 활용.

## 1. 3층 구조 (운영 골격)

```
┌──────────────────────────────────────────────────┐
│ 층 1 · 온실 곳간 (Asset Library)                  │
│   Firefly 이미지 / 디자인 프리셋 / 검증된 블록     │
│   = 한 번 만들고 쌓아 재사용 (매번 새로 X)         │
├──────────────────────────────────────────────────┤
│ 층 2 · 온실 아틀리에 (Production — 5 컨펌 게이트)  │
│   뉘집씨앗인가 → 화단스케치 → 광합성샤워 →         │
│   꽃까꿍 → 비밀정원입장                            │
│   각 단계 = 엔진 제안 + 대표님 맞춤 컨펌 게이트     │
├──────────────────────────────────────────────────┤
│ 층 3 · 꽃틔움 결 맞추기 (Sync Layer)              │
│   클로드 디자인 프리셋 ↔ 자체 엔진 토큰 일치       │
└──────────────────────────────────────────────────┘
```

## 2. 층 1 · 온실 곳간 (Asset Library)

대표님 원안에 없던 핵심 개념. *매번 새로 만들지 않고 쌓아 재사용.*
비유: 매번 장 보는 게 아니라 잘 채운 냉장고에서 꺼내 쓴다.

| 자산 | 만드는 곳 | 재사용 방식 |
|---|---|---|
| 합성 이미지 (라이프스타일 컷) | 어도비 Firefly | 카테고리/계절 태그로 분류, 30일 cooldown 후 재사용 |
| 꽃틔움 디자인 프리셋 | 클로드 디자인 (1회 등록) | 모든 시안에 자동 상속 |
| 검증된 블록 구조 | 아티팩트 → 자체 엔진 | 잘 먹힌 조합을 컴포넌트로 축적 |

→ 기존 자산 활용: lifestyle-picker.ts (30일 cooldown), art-director (Firefly 프롬프트)

## 3. 층 2 · 온실 아틀리에 — 5개 컨펌 게이트 (핵심)

각 엔진은 *제안만* 한다 → 대표님이 컨펌 → 다음 단계로.
**중요**: 컨펌 액션은 게이트마다 다르다. 각 엔진이 내놓는 결과물의
성격(판정값 / 구조 / 소재 / 완성본 / 마무리)에 맞춰 *승인·수정 도구*가 다름.
비유: 검수대 위 물건이 다르니(진단서/설계도/재료/완성품/포장), 손에 쥐는
수정 도구도 다르다.

공통 베이스 액션: **승인 (다음으로)** / **이 단계만 다시 (엔진 재실행)**
각 게이트 고유 수정 액션은 아래 표.

### 게이트 1 · 뉘 집 씨앗인가 (Seed Interview)
```
엔진: 진단 8축(CTI) + 전략역할(꿀통/니치/트렌드/레드오션) + 정체성 키워드
결과물 성격: 판정값 (이 상품의 정체)
산출 예: "니치 / 정체성='달항아리 도어벨' / 미적 점수 7"
```
| 대표님 컨펌 | 수정 액션 |
|---|---|
| 승인 | 판정 확정 → 화단 스케치로 |
| 수정 | 전략역할 변경(니치→꿀통) · 정체성 키워드 교체 · 점수 조정 |
| 다시 | 진단 엔진 재실행 |

### 게이트 2 · 화단 스케치 (Stem Sketch)
```
엔진: 아트디렉터 + 모듈 조립 — 역할에 맞는 블록 순서 자동 구성
결과물 성격: 구조 (블록 배치도)
산출 예: 후킹 → 페인포인트 → USP → 라이프스타일 → FAQ → CTA
도구: 아티팩트로 구조·텍스트 흐름 시각화
```
| 대표님 컨펌 | 수정 액션 |
|---|---|
| 승인 | 구성·순서 확정 → 광합성 샤워로 |
| 수정 | 블록 추가/제거 · 순서 드래그 · 블록 켜고 끄기 |
| 다시 | 조립 엔진 재실행 (다른 구성안) |

### 게이트 3 · 광합성 샤워 (Fire-fly Shower)
```
엔진: AI 이미지(Firefly 허브 + Nano Banana Pro) + Groq 카피 + 프롬프트 자산
결과물 성격: 소재 (각 블록 채울 이미지+문구 후보)
산출 예: 블록별 이미지 + 헤드라인/본문 카피
```

**7단계 이미지 워크플로우 (AI_IMAGE_PROMPT_STRATEGY_2026_05.md 기반)**
```
1. 실촬영/공식실사 (상품 본체)        ← 표시광고법 회피 절대원칙
2. 누끼          (어도비 익스프레스)   ← 대표님 검증: 익스프레스 내장으로 충분
3. 배경/씬 생성  (Nano Banana Pro)     ← 합성 자연스러움 1위 (대표님 외주검증)
4. 톤·색 보정    (파이어플라이 편집)    ← 대표님 검증: 자체 편집 나쁘지않음
5. 한글 헤더     (익스프레스 별도합성)  ← 썸네일엔 텍스트 X (네이버 정책)
6. 검수 게이트                          ← 실물일치/가상인물/어뷰징/AI메타 4체크
7. 등록후 30일 CTR/CVR 회수 → 프롬프트 점수 갱신
```
도구 수렴: Firefly Pro $19.99/월 단일 허브
  = 생성 + 누끼 + 편집 + 파트너모델(Nano Banana Pro/GPT Image) 무제한.
  외주 누끼도구 불필요 (대표님 검증). 1인 셀러 비용구조 최적.

**모델 선택 가이드 (상품 컨셉별)**
| 컨셉 | 권장 모델 | 이유 |
|---|---|---|
| 저관여·감성(디퓨저/캔들/데코) | Nano Banana Pro · Firefly5 | 라이프스타일씬·자연광 |
| 고관여·기능성(주방/청소/수납) | GPT Image 2 · Firefly5 | 디테일컷·매크로 |
| 가격高(프리미엄) | Firefly5(안전) · NanoBananaPro | 미니멀·럭셔리 |
| 가격低(대중잡화) | NanoBananaPro · FLUX/Ideogram | 컬러풀·텍스트 |
| 한글 텍스트 이미지 | Nano Banana Pro | 한글렌더링 1위 |

**프롬프트 자산화 연동**: 사용한 프롬프트는 prompt_assets 테이블에 저장.
  대표님이 수정해 더 좋은 결과 도출 시 그 프롬프트가 자산이 됨.
  intent 선택 → 모델별 프롬프트 자동변환 (모델 어댑터). 상세: 섹션 9.

**법적 안전 3원칙**
  ① 상품 본체는 실촬영·배경만 AI (표시광고법)
  ② 셀러 = 이용자 (AI기본법 의무 없음)
  ③ 꽃틔움 = 인공지능사업자 (약관 사전고지)

| 대표님 컨펌 | 수정 액션 |
|---|---|
| 승인 | 소재 확정 → 꽃 까꿍으로 |
| 수정 | 이미지 재생성/교체(Firefly/NanoBanana) · 프롬프트 수정 · 카피 다시 · 곳간 다른 컷 |
| 다시 | 소재 일괄 재생성 |

### 게이트 4 · 꽃 까꿍 (Final Bloom)
```
도구: 클로드 디자인 — 게이트 1~3 결과를 입력해 1차 완성본 생성
       꽃틔움 프리셋 자동 적용
결과물 성격: 완성본 (디자인된 상세페이지 HTML)
산출 예: 꽃틔움다운 1차 완성 상세페이지
```
| 대표님 컨펌 | 수정 액션 |
|---|---|
| 승인 | 디자인 확정 → 비밀 정원 입장으로 |
| 수정 | 슬라이더 조정(톤·여백·색) · 직접 편집 · 이 블록만 다시 |
| 다시 | 클로드 디자인 1차 완성본 재생성 |

### 게이트 5 · 비밀 정원 입장 (Atelier Showcase)
```
도구: 클로드 코드 — 상황에 맞는 확장자로 마무리 + 네이버 정책 자동 체크
결과물 성격: 마무리 (업로드 직전 결과물)
산출 옵션: HTML / 이미지 / 컴포넌트
```
| 대표님 컨펌 | 수정 액션 |
|---|---|
| 승인 | 업로드 확정 (정책 체크 통과 후) |
| 수정 | 확장자 선택(HTML/이미지) · 출구 선택: 네이버 직접 등록 / 자체 엔진 축적 / 추가 작업 회귀 |
| 다시 | 마무리 재처리 |

## 4. 층 3 · 꽃틔움 결 맞추기 (Sync Layer)

클로드 디자인에서 정한 꽃틔움 프리셋(색·폰트·여백)을
클로드 코드로 자체 엔진 디자인 토큰과 동일하게 맞춤.

```
클로드 디자인 프리셋 (본점 인테리어 색)
        │  클로드 코드가 토큰 추출·동기화
        ▼
자체 엔진 디자인 토큰 (분점이 같은 페인트 코드로)
```

효과: 게이트 5에서 HTML 직접 등록한 상세페이지와,
자체 엔진이 대량 생산한 상세페이지가 *형제처럼 일관된 꽃틔움다움.*

## 5. 도구별 역할 (조사 근거)

| 도구 | 역할 | 근거 |
|---|---|---|
| 어도비 Firefly | 합성 AI 이미지 (실사+감성배경) | 클로드 디자인 *이미지 생성 불가* |
| 아티팩트 | 구조·텍스트 흐름 설계 (게이트 2) | 빠른 구조 탐색 |
| 클로드 디자인 | 1차 완성본 + 프리셋 (게이트 4) | HTML 코드 출력, Max5 구독 보유 |
| 클로드 코드 | 마무리·이식·동기화 (게이트 5) | Handoff 번들 = 토큰+구조+의도 |
| 자체 엔진 (Satori+Sharp) | 검증된 류의 대량 자동 생산 | 클로드 디자인 API 없음 |

클로드 디자인 활용 원칙:
- 매일 호출 X (주간 한도) — 게이트 4 디렉팅에만
- 이미지 생성 기대 X — Firefly가 담당
- 한국어 폰트 명시 (Pretendard CDN/파일)
- 출력은 HTML로 받아 활용
