# 이미지·SEO 전략 엔진 — 전체 시스템 설계 (2026-06-07)

> 대표 비전: 양질 공급사 자료는 그대로 활용(사이트 썸네일/상세페이지 as-is), 미흡하면 상세페이지에서 크롭/보강/신규, SEO ROI는 네이버 기준 채움, 부족분만 문구/이미지 생성. 상황별 융통 + 운영자 개입점이 앱에서 자연스럽게.

## A. 현재 상태 정직 진단 (Desktop 실측)

| 구성 | 상태 |
|---|---|
| 품질 분류기(작업2) | ✅ production (6지표 0~100 + 3모드) |
| thumb-crop(작업5) | ✅ 작동 (상세 1:1 크롭·1000px·OCR가드). 단 대표이미지 적용 루프 미완 |
| seo-text/attributes/etc/enqueue | ✅ production |
| 명화 발행 준비도 | ✅ dryRun A/84 (origin·재질·색상·태그·향4·안전번호 반영) |
| ★ 관제탑 ↔ 네이버 routes 동기 | ❌ 미동기 — asset_jobs와 routes가 별개. 명화 A/84인데 matrix risk/blocked/nextAction null |
| ★ 이미지 전략 T0(그대로-대표) | ❌ 부재 — 품질 3모드만, "as-is" 티어 없음 |
| ★ crop→대표이미지 적용 | ❌ 미완 — preview만, mainImage set 루프 없음 |
| ★ nextAction 개입점 | ❌ null — 운영자 "다음 할 일" 미surfacing |

## B. 목표 — 이미지 전략 5티어 (cheapest-first)

| 티어 | 조건 | 동작 | 개입점 |
|---|---|---|---|
| T0 그대로-대표 | 공급사 대표 ≥1000·단색·텍스트0·단품 | site thumb as-is | 자동 / 경계 1클릭 |
| T1 상세크롭 | 대표 미흡 + 상세 양질 단품영역(≥1000) | thumb-crop | 영역지정·승인 |
| T2 보강 | 단품 있으나 배경/품질 보강 | 누끼+클린배경 | 승인 |
| T3 신규 | 소스 빈약 | 누끼+AI배경(B안) | 승인 |
| 상세페이지 | 양질→그대로+SEO문구 / 빈약→빌드 | as-is or build | 부족분만 생성 |

분류기는 공급사 **대표(site thumb)** 와 **상세페이지** 를 각각 평가해 티어 추천. 소스별 분기.

## C. SEO ROI 채움 (네이버 기준) — routes 존재

카테고리 · 속성(enum 재질/색상 등) · 상품명(50자·중복0) · 태그10(태그사전·제한어 회피) · 정보고시(안전확인 신고번호·생활화학제품).

## D. 운영자 개입점 (자연스럽게)
- 경계 점수(40~70) · 비가역 직전(네이버 PUT) · 크롭 영역지정 · 4번째 옵션 등 판단 → 관제탑 nextAction 1클릭 승인/조정.

## E. Code 시공 트랙 (병행, 누락 0)
1. 관제탑 SoT 통합: matrix publish track = 네이버 update dryRun 준비도(grade/score/missingRequired). image track = 이미지전략 티어. 단일 진실원.
2. nextAction 엔진: 상품별 다음할일 산출 + 1클릭 노출.
3. 이미지 전략 T0 추가: 공급사 대표 품질평가 → USE_AS_IS 추천. 소스별(대표 vs 상세) 분기.
4. crop→대표 적용 루프: thumb-crop confirm 시 mainImage set.
5. 상세 as-is + SEO 문구/이미지 보강: 양질 상세 그대로, 부족분만 생성.

## F. 명화 즉시 경로 (시스템과 별개로 진행 가능)
- 대표이미지: 4종합성을 T0(그대로) 수용 → 발행 가능 / 단품 원하면 풀해상 화보(1000px+) 필요.
- 정보고시 HB 표시 검증(dryRun preview에 notice 노출 = Code) → 대표 GO → PUT(비가역).
